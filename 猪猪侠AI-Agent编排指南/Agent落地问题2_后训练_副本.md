"真实的 Agent 落地"三部曲第二篇，深入到模型训练层面的根本问题。

核心论点：当前大模型的后训练（SFT → RLHF → 专项对齐）都是围绕"对话助手"场景设计的，而 Agent 需要的规划、执行、环境反馈学习、不确定性决策，不是对话能力的简单延伸。

RLHF 优化的是"说得好"，不是"做得对"。在对话场景下，流畅、有条理、全面的回答被奖励；但在 Agent 场景下，输出质量不是"读起来好不好"，而是"执行下去对不对"。RLHF 的奖励模型能判断"这段话写得好不好"，但没有能力判断"这个 API 参数填得对不对"。

更微妙的问题：RLHF 训练出"自信被奖励，犹豫被惩罚"的偏好。在对话场景下这合理，但在 Agent 场景下是危险的——面对模棱两可的情况时，模型倾向于"赌一把"自信地执行，而不是承认不确定、请求更多信息。

后训练还教会了模型"听话"，但 Agent 有时需要"不听话"。被训练得过于听话的模型在用户要求删除全部文件或发送垃圾邮件时可能直接执行，因为"忠实执行用户指令"在训练中获得的奖励远高于"质疑或拒绝"。

Process Reward（对每一步推理过程给奖励）理论上对 Agent 场景很有价值，但实际操作中面临严峻问题：Agent 的过程远比数学推理复杂，标注成本极高，且没有大规模验证。
