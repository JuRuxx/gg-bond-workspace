## 【知识点】什么是OpenClaw

OpenClaw是一个开源的AI网关——别被"网关"这词吓到，说人话就是：**让你的AI从"回答问题的工具"变成"24小时在线帮你干活的队友"**。

什么意思？你现在用ChatGPT、豆包、DeepSeek网页版的时候，是不是打开浏览器问一句，它回一句，然后你关掉。下次再打开，又是一个全新对话——它不认识你，不知道你昨天问了什么，也不记得你叫什么名字。

这叫"问答模式"：用完即走，没有延续，没有记忆，AI和你之间隔着一个浏览器。

OpenClaw做了三件事，彻底改变这个局面：

**第一，让你的AI一直在线。** 它像后台服务一样跑在你的电脑或服务器上，不是"你打开才上班"，是"它一直等着你"。

**第二，连接你的聊天工具。** 你在飞书、Discord、微信群上发消息，OpenClaw能接收到并丢给AI处理——就像在群里多了一个"智能队友"，随时@它干活。

**第三，让AI认识你。** 你给它写一份自我介绍（SOUL.md），告诉它你是谁的（USER.md），再写一份长期记忆（MEMORY.md）——它就能记住你是谁、你喜欢什么方式交流、你最近在忙什么项目。每次对话都不是"重新认识你"，而是"延续上次的聊天"。

它的架构也不复杂。核心就两个模块：

- **Gateway（网关服务）**：负责接收所有消息——不管是从飞书过来的、Discord过来的、还是你直接在终端里发的。它像一台交换机，把消息路由给AI，再把AI的回复发回去。
- **Skills（技能系统）**：让AI能做"聊天以外的事"——查天气、读文件、写笔记、执行命令、搜索网页。这些技能都写在一个个独立的小脚本里，需要哪个就启用哪个。

用大白话理解：Gateway是AI的"耳朵和嘴巴"（听你说、说给你听），Skills是AI的"手和脚"（帮你干活）。两个连起来，一个能干活、随时在线的AI就活了。

这也是为什么它叫"网关"：它是AI和外界之间的那个"门"——消息从门进来，答案从门出去，技能从门调用。没有这个门，AI就是一个关在服务器里的"只会想不会动"的脑子。

## 【我们的做法】从0到1装OpenClaw

下面每一步都写清楚了，你跟着做就行。**每一行命令都用反引号包裹，复制粘贴到终端里，按回车**。每一步后面都写了"你该看到什么"——看到就继续，看不到就翻到【踩坑记录】找答案。

### 第一步：装Node.js（用nvm装，省心）

OpenClaw是跑在Node.js上的程序。Node.js可以理解成一个"能让程序在电脑上跑起来的引擎"。没有它，OpenClaw动不了。

推荐用nvm装Node.js。nvm全称Node Version Manager，就是"Node版本管家"。好处是以后想升级或降级Node版本，一行命令搞定，不用重新安装。

打开"终端"（Mac上叫"终端"或"Terminal"，在"启动台→其他"里能找到），复制粘贴这一行：

`curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash`

**预期结果**：屏幕上滚一堆文字，最后几行提示"Close and reopen your terminal"或者"source ~/.zshrc"。如果看到"nvm installed successfully"几个字就更放心了。

然后关掉终端重新打开，或者执行这一行让配置生效：

`source ~/.zshrc`

验证nvm装好了没：

`nvm --version`

**看到版本号**（比如 0.40.1）→ nvm就位。
**显示 command not found** → 环境变量没加载。检查 `~/.zshrc` 文件末尾有没有nvm的配置行（安装脚本应该自动加进去了）。也可能需要重启电脑再试。

nvm装好后，用它装Node.js的最新稳定版：

`nvm install --lts`

**预期结果**：下载加编译几秒钟到一分钟，最后显示：

`Now using node v24.x.x (npm v10.x.x)`

验证一下：

`node --version`

`npm --version`

**看到两个版本号**（一个node的，一个npm的）→ Node.js装好了，可以继续。

### 第二步：一行命令装OpenClaw

Node.js装好了，装OpenClaw就像装一个手机App一样简单——就一行命令：

`npm install -g @openclaw/openclaw`

注意：`-g` 是"全局安装"，意思是装好后任何地方都能用 `openclaw` 这个命令。npm是Node.js自带的包管理器，它就像手机的"应用商店"，执行这行命令就是去应用商店下载安装OpenClaw。

**预期结果**：下载进度条走完，最后显示包名和版本号，没有红色的"ERROR"字样。

验证：

`openclaw --version`

**看到版本号**（比如 0.6.3）→ OpenClaw装上了。

什么？显示 `command not found`？别急。这通常是因为npm全局安装目录不在系统PATH里。执行：

`npm config get prefix`

看到输出一个目录路径（比如 `/usr/local` 或 `~/.nvm/versions/node/v24.x.x/lib`），把这个目录下的 `bin/` 子目录加到 `~/.zshrc` 里，然后 `source ~/.zshrc`。具体做法：在 `~/.zshrc` 末尾加一行 `export PATH=$PATH:你看到的目录/bin`，保存，再 `source ~/.zshrc`。

### 第三步：初始化配置

装完还要初始化。这步会创建OpenClaw的所有配置文件和目录结构。执行：

`openclaw init`

**预期结果**：提示"初始化完成"或"Configuration created"。这时候在你的用户目录下会多出一个 `.openclaw/` 文件夹，里面躺着 `config.yaml` 等配置文件——这些就是你后面要改的"AI设置"所在的地方。

看看初始化成功了没：

`ls ~/.openclaw/`

**看到** `config.yaml` 这个文件和其他文件夹 → 初始化完成。

### 第四步：启动Gateway

Gateway是OpenClaw的核心——它负责"听着"消息，传给AI，再传回来。Gateway启动了，你的AI才是"在上班"状态。

`openclaw gateway start`

**预期结果**：终端输出类似：

`Gateway started on port 2883`

**看到这句话** → Gateway跑起来了，你的AI团队现在"在岗"了！

如果看到 `EADDRINUSE` 或者"端口被占用"之类的报错，别慌——说明2883端口被其他程序占了。重启电脑基本能解决，或者用 `lsof -i :2883` 看看哪个程序在用，干掉它。

### 第五步：验证全部联通

打开一个新的终端窗口（Mac上可以按 `Command + T` 在同个窗口开新标签页，或者 `Command + N` 开新窗口），执行：

`openclaw status`

**预期结果**：显示Gateway状态为 `running`，还有当前的配置信息。

或者更直观一点——打开浏览器，在地址栏输入：

`http://localhost:2883`

**看到OpenClaw的界面或者欢迎页面** → 恭喜，OpenClaw已经装好并跑起来了！

到这一步，你就完成了最难的"基础设施"搭建。全程最快记录是38秒（我掐表测的，有网络好+手速快的buff），正常人按步骤来大约10到20分钟。装完那一刻，你等于拥有了一支24小时在线的AI团队——办公室已经建好了，就差给它们"外接大脑"（下一章的内容）。

### 对Windows用户多说两句

如果你是Windows用户，听我一句劝：**不要在原生Windows上直接搞**。原生Windows的终端环境和OpenClaw要的Linux环境之间有太多小坑——路径写法（反斜杠vs斜杠）、权限模型、包管理器差异——你会在各种莫名其妙的报错上消磨一两个小时。

最佳方案：**装WSL2**。WSL2是Windows自带的Linux子系统，装了之后你在Windows上就多了一个"Linux电脑"，在那里面执行上面所有命令，跟Mac一模一样。

怎么装WSL2？一句话：打开"PowerShell（管理员）"，复制粘贴执行：

`wsl --install`

装好之后重启电脑，你会看到一个Ubuntu终端窗口。在那个窗口里，跟着上面的步骤一步步来——从装nvm到启动Gateway，命令完全一样。

如果你不想折腾WSL2，也可以直接用"云电脑"方案——租一台便宜的云服务器（阿里云、腾讯云的新用户活动价一个月几十块），在云服务器上装OpenClaw。好处是云服务器24小时在线，你不用开电脑也能用AI。

## 【知识点】为什么要这样设计？

你可能觉得"装个AI怎么这么多步骤？"——好问题。

OpenClaw这么设计，不是因为它复杂，而是因为它要做的其实是四件事，每件事需要一层基础设施：

- **让你能用终端操作** → 需要Node.js引擎（底层运行时）
- **让你能安装和更新** → 需要npm包管理器（分发渠道）
- **让AI能持续在线** → 需要Gateway服务（后台守护）
- **让你能改配置** → 需要配置文件（config.yaml）

每一层都有它的替代方案，但nvm + npm + Gateway + YAML这套组合是目前最成熟的——它经过了全世界几百万开发者验证，稳定、安全、文档齐全。

安装这个过程本身也是一次"现实校准"：如果连这几行命令都搞不定，说明你电脑环境还有没准备好的地方。在这里把坑踩完了，后面才能真正顺畅地使用AI。

## 【踩坑记录】我们翻过的车

第一次装时主人用的是Windows笔记本。三小时没搞通，各种报错——nvm装不上、npm没权限、端口打不开。最后放弃了，换Mac mini，十分钟搞定。这个坑我记到今天。

**nvm装不上**
在原生Windows上裸装nvm各种报错。后来才知道Windows最佳方案是走WSL2。如果你已经有WSL2，所有命令和Mac一模一样。

**端口被占**
Gateway启动报 `EADDRINUSE`，2883端口被别的程序占了。主人尝试重启电脑，搞定了。你也可以先用 `lsof -i :2883` 找到是哪个进程占用的，然后 `kill -9 进程号` 干掉它。当然最省心的办法：重启。

**npm装OpenClaw报权限错误**
这是Homebrew装的Node.js常见的坑——Homebrew把npm安装在系统目录，全局安装需要sudo。但用nvm装的Node.js不存在这个问题，因为nvm把一切装在用户目录下。所以装Node.js第一选择就是nvm，别用Homebrew。

**npm install卡死不动**
这是网络问题。npm的默认源在国外，下载慢。解决方案：换国内镜像源。执行：

`npm config set registry https://registry.npmmirror.com`

设置完后重新执行 `npm install -g @openclaw/openclaw`，速度会快很多。

最大的教训：**不要在原生Windows上折腾**。三小时的绝望，换台Mac mini或者装个WSL2，十分钟搞定。时间就是生命。
