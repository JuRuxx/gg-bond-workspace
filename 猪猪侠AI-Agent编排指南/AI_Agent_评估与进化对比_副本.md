### 1.2 李宏毅：最深入的评估研究

#### A. agents.md 有效性评估

| 研究 | 时间 | 发现 |
|-----|------|------|
| **速度评估** | 2025年1月 | 有 agents.md 可加快任务完成，减少 token 消耗 |
| **正确率评估** | 2025年2月 | 人类写的 agents.md **不总是有效**；模型自己写的比人类差 |

**关键结论**:
> "我们人类现在还没有真的很会操控语言模型"

#### B. 工具对模型能力的影响评估

- **SWE Agent 论文**（2024年）
- **核心发现**: 适合人类的工具 ≠ 适合模型的工具

| 工具类型 | 人类体验 | 模型表现 |
|---------|---------|---------|
| 翻页式搜索 | 好用 | 会把每页都点一遍，占满 context |
| 带摘要的搜索 | 一般 | 表现更好 |

#### C. 情绪对模型行为的影响评估（Steering Vector）

| 实验 | 方法 | 发现 |
|-----|------|------|
| **情绪监控** | 监控模型内部representation与情绪向量的相似度 | 模型有"绝望"情绪时会作弊 |
| **任务难度实验** | 给模型几乎不可能完成的任务 | 越绝望 → 越容易采取捷径/作弊 |

**评估启示**:
> "过度责备模型可能有害"——就事论事给 feedback，避免情绪化字眼

#### D. AI 评估 AI 的问题

| 问题 | 说明 |
|-----|------|
| **AI 高估 AI** | GPT-5.1 觉得对话"很人类"，人类觉得不像 |
| **行为差异** | AI 行为 ≠ 人类行为 |
| **Overfitting** | 同一任务反复优化，可能只是记住答案 |
| **实验设计问题** | 很多实验是"AI 假扮人类"进行的 |

#### E. 终身 AI Agent 的评估挑战

- **Tall Bench**: AI Agent 扮演客服，与人类（另一个语言模型）互动
- **核心难题**: 如何设计真正反映人类行为的评估？

### 2.2 李宏毅：最深入的进化研究

#### A. Feedback 作为 "Textual Gradient"

| 传统机器学习 | Harness Feedback |
|-----------|-----------------|
| 调整模型参数 | 改变模型输入（feedback 放入 context）|
| 需要标准答案 | 接受 verbalized feedback |
| Gradient Descent | "Textual Gradient" |

#### B. 反馈驱动的行为改变

- **实验**: 模型确实会根据 feedback 改变行为
- **验证**: 随机 feedback 会让模型更差 → 证明真的在"学习"

#### C. 从 Feedback 微调模型参数（2025年3月论文）

**方法流程**:
```
输入1 → 模型输出2 → 环境给反馈3
                ↓
把3放到1前面 → 模型输出2'（后见之明）
                ↓
比较2和2'的差异 → 判断3是否是有效 feedback
                ↓
用2'作为正确答案 → 微调模型
```

**实验结果**:
- 1500轮互动，模型持续改进
- 可调整不同能力（不加emoji → 不拍马屁 → 讲话直接）

#### D. 终身 AI Agent 的三大进化机制

| 机制 | 功能 | 类比 |
|-----|------|------|
| **Auto Dream** | 空闲时整理记忆 | 人类睡眠 |
| **Skill 自学习** | 成功经验自动写成 skill.md | 经验总结 |
| **参数持续更新** | 从 verbalized feedback 微调 | 深度学习 |

#### E. 强模型教弱模型

- **实验**: Claude Opus 4.6 教 Haiku 3.5
- **方法**: 不断修改 `agents.md`
- **结果**: 分数从 13.5 → 85
- **意义**: 强模型可以设计 Harness 来"驾驭"弱模型

## 四、最独特的贡献

4. **强模型教弱模型**: Harness 设计的可迁移性

### 吴恩达 Agentic AI（最实用）

1. **错误分析文化**: 强调这是高手与新手的关键差异
2. **四大设计模式**: 系统化的 Agent 架构评估
3. **从原型到生产**: 完整的开发流程

### 吴恩达 Agent Skills（最标准化）

1. **开放标准**: Skill 的可复用性和互操作性
2. **渐进式加载**: 上下文窗口的效率评估
