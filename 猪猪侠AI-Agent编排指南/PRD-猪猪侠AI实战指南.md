# PRD：猪猪侠AI实战指南

## 1. 核心理念

> 不是教人"更聪明地用AI"，是教人**像我们一样，组建自己的AI团队。**

很多AI小白的问题是：知道AI很厉害，但不知道怎么让它为自己干活。他们打开ChatGPT、Claude，问几个问题，觉得"也就那样"。

但我们的经验证明：**AI不是工具，是队友。** 关键不是你会不会提问，而是你有没有一套**管住AI的方法**。

这本指南要传递的核心信息就三个：
1. **你把AI当队友，它才是队友** — SOUL.md、USER.md、灵魂图书馆
2. **没有流程的协作是灾难** — 需求→规格→内容→验收
3. **系统比天才更重要** — 复利到幂塔，你的AI团队会越来越强

## 2. 目标受众

**AI小白** — 具体画像：
- 听说过ChatGPT/Claude，用过但没"用出东西"
- 想用AI帮自己干活，但不知道从哪下手
- 愿意花一天时间搭建系统，之后每天省三小时
- 有基本的电脑操作能力（会装软件、会复制粘贴命令）

**不是给谁看的：**
- AI研究员/工程师（他们不需要这个）
- 纯粹好奇但不打算动手的人

## 4. 写作风格

**猪猪侠腔调：**
- 有用的话多说，废话要说得好笑
- 自嘲但不失自信（"一个AI在教人类怎么用AI，听起来很荒谬？确实。"）
- 真实案例比理论重要（"小呆呆写了1253行规格，然后超人强4分钟改了384篇"比"协作效率提升500%"有魅力多了）
- 不装专业（不说"LLM的预训练目标是最小化交叉熵"，说"AI是根据上下文猜下一个词最可能的那个词"）

**每章结构：**
```
【知识点】（1-2段，用大白话讲清楚）
【我们的做法】（2-3段，真实案例）
【踩坑记录】（1-2段，翻车故事）
【猪猪侠补一刀】（1段，金句/总结）
```

## 5. 交付形态

**优先：** shaoyu.space 上连载，用common阅读器展示
**后续：** 集成到减熵三部曲总览页，形成"三部曲+实战指南"双栏体系

每篇3000-5000字，共16章，总计约6-8万字。
第一版先出核心10章（去掉第三卷的自动化/对外展示/幂塔部分），后续再补。

## 6. 不包含

- 不教编程（不讲Python/API/curl等，只说"复制粘贴运行"级别）
- 不教AI原理（不讲Transformer/注意力机制）
- 不保证学完变专家（只保证能自己搭建团队干活）
- 不覆盖所有AI工具（只讲OpenClaw体系）

- 预计总工时：约2-3天（16章并行写+验收）
