## 核心概念

### 什么是 Agent Skills？

> "所有这些技能都是包含指令的文件夹，可扩展代理的专业知识。"

**关键特性**:
- 已成为**开放标准**（Open Standard）
- 标准化格式，适用于任何兼容技能的代理
- **一次构建，可在多个代理产品中部署**

---

## Skill 的标准结构

```
skill/
├── skill.md          # 技能名称、描述、主要指令
├── scripts/          # 脚本文件
├── references/       # 附加Markdown文件
└── resources/        # 模板、图片等资源
```

---

## 渐进式加载机制

| 组件 | 加载策略 |
|-----|---------|
| **技能名称和描述** | **始终**保留在上下文窗口 |
| **其余指令** | **按需加载**（用户请求匹配描述时才加载）|
| **参考和资源文件** | 使用技能时按需加载 |

**优势**: 节省上下文窗口，提高加载效率

---

## 基础工具集要求

- 文件系统访问（读写文件）
- 批处理工具（执行代码）

这些工具使代理能够执行所需命令。

---

## 高级组合能力

| 组合方式 | 应用场景 |
|---------|---------|
| **Skill + MCP** | 从外部源获取数据，用Skill处理 |
| **Skill + 子代理** | 委托任务给独立上下文的子代理 |
| **多个Skills组合** | 创建复杂工作流 |

**子代理特点**:
- 具有独立上下文
- 可自身使用技能获取专业知识

## 何时使用 Skill？

> "当你有一个需要反复让代理执行的工作流程，与其每次都解释相同的流程，你可以将其封装为一个技能。"

**使用标准**:
- 重复性工作流程
- 需要标准化执行
- 希望在多个代理中复用

## 评估维度

| 维度 | 说明 |
|-----|------|
| **功能正确性** | Skill 是否完成预期任务 |
| **可复用性** | 是否能在多个代理产品中部署 |
| **组合性** | 与其他 Skill 组合的效果 |
| **加载效率** | 渐进式加载的上下文窗口使用 |

## 进化机制

### Skill 的版本迭代
- 基于使用反馈改进 Skill
- 更新 `skill.md` 中的指令

### 组合进化
- 从单一 Skill 到复杂工作流
- 通过 MCP 和子代理扩展能力
