## 摘要

这是 Anthropic 技术团队成员关于 Claude Code 的实战教学分享。内容覆盖了从安装配置到高级工作流的完整使用指南，核心围绕三个层次展开：新手如何入门（代码库问答）、进阶如何提升效率（规划先行+上下文管理）、以及高级用户的并行工作流。演讲者强调 Claude Code 的本质不是一个简单的代码补全工具，而是一个"全自主代理"，能在任意 IDE 和终端中无缝运行，通过极少的工具组合完成复杂的开发任务。

## 详细解读

### Claude Code 是什么

Claude Code 是一个全自主的 AI 编程代理，不是传统的逐行代码补全工具。它能一次性编写完整函数和文件、修复 bug，并且**无缝适配任何开发环境**——VS Code、Xcode、JetBrains 系列 IDE，无论是本地、远程 SSH 还是 tmux 都能运行。

Anthropic 团队选择做命令行工具而非 IDE 插件有两个原因：一是团队内部使用各种 IDE，终端是唯一共同的选择；二是他们预见模型能力进化极快，不应该在 UI 上层架构上过度投入，否则很快过时。

### 新手入门：从代码库问答开始

对初学者最强烈的建议是**先别急着让它改代码，先问问题**。这是 Anthropic 培训新人的第一课——技术入职从过去的两三周缩短到了两三天。

可以问的问题包括：
- 这段代码是怎么用的？怎么实例化这个对象？
- 分析 Git 历史记录（某个函数为什么有 15 个参数？谁加的？关联了哪些 issue？）
- 查阅 GitHub issue 和相关背景

Claude Code **不做任何代码索引**，代码不会上传到远程，不用于模型训练，完全本地存储，零配置即可使用。

### 核心工作流：规划先行

让 AI 代理写代码前，**先让它做头脑风暴或规划方案**。不要一上来就丢给它一个 3000 行的功能——不需要开任何特殊模式，只要告诉它"写代码前先规划，等我批准再动手"就这么简单。

这种"先思考再行动"的方式能大幅提高输出质量。

### 上下文管理：给得越多干得越好

Claude Code 表现与上下文充分度直接相关。多层级上下文引入机制：

- **CLAUDE.md**（项目根目录）：每次会话自动加载，建议提交到版本控制，团队共享
- **子目录中的 CLAUDE.md**：只在对应目录中运行时自动加载
- **本地 .claude/** 文件：个人使用，不提交
- **企业级全局配置**：适用于所有项目

内容可以包括常用 bash 命令、代码风格指南、MCP 工具配置、架构决策等。关键原则是**保持简短**，太长会消耗上下文且没实际价值。

### 工具集成：MCP 与 Bash

Claude Code 支持两类外部工具：
1. **Bash 工具**（CLI 命令行工具）
2. **MCP 工具**（Model Context Protocol）

只需告诉它工具的存在，它就能自行摸索使用方法。最佳实践是将团队常用工具链配置到 CLAUDE.md 中，一次设置全员复用。Anthropic 内部的做法是在代码库中提交 MCP 配置文件，所有工程师直接调用 Puppeteer 进行端到端测试。

### 自我迭代：给反馈工具让它越做越好

一个关键技巧是**给 Claude Code 一个能检验自己成果的工具**，它就会自动迭代优化。比如：
- 编写单元测试让它验证
- 用 Puppeteer 或 iOS 模拟器截图让它对比设计稿

只要给它反馈机制，迭代两三次后效果会显著提升。

### 实用快捷键与命令

- **Shift + Tab**：进入自动接受编辑模式
- **ESC**：中断当前操作，可以要求重做部分修改
- **按两次 ESC**：回溯历史
- **Control + R**：查看完整上下文输出
- **`#` 加文字**：让 Claude Code 记住某件事，跨会话记忆
- **`/install github`**：安装 GitHub 应用
- **`/memory`**：查看/编辑所有加载的记忆文件
- **`!` 加命令**：在 bash 模式下运行并同步到上下文

### 命令行 SDK

`claude -p` 参数启用 SDK 模式，可以：
- 传入提示词和允许使用的工具
- 自定义输出格式（普通 JSON 或流式 JSON）
- 通过管道与其他命令组合（如 `git status | claude -p`）
- 在 CI 流水线中作为 UNIX 工具使用

### 高级用法：并行工作流

重度用户通常通过 SSH 配合 tmux 隧道接入，检出同一仓库的多个副本并行运行多个 Claude Code 会话。也可以利用 Git worktree 实现操作隔离。可以同时开启任意数量的会话来并行处理多项任务。

### 安全与权限

Bash 命令的安全性是最棘手的问题。Claude Code 采用多层级权限系统：
- 部分命令设为只读，通过静态分析判断哪些命令可以安全组合执行
- 支持命令的白名单/黑名单
- 企业策略可以强制执行限制（如禁止访问特定 URL），员工无法覆盖

### 内部使用情况

Anthropic 内部约 80% 的技术人员每天使用 Claude Code，包括工程师和研究人员（使用 Notebook 等工具的也同样使用）。这是产品经过内部高强度打磨的证明。
