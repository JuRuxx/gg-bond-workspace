## 第二部分：Skill 的结构与规范

### 第3节：Skill 的标准文件结构

```
skill/
├── skill.md              # 核心文件（必需）
│   ├── 技能名称
│   ├── 描述
│   └── 主要指令
├── scripts/              # 脚本文件
│   └── *.py, *.js, etc.
├── references/           # 附加Markdown文件
│   └── *.md
└── resources/            # 资源文件
    ├── templates/        # 模板
    ├── images/           # 图片
    └── examples/         # 示例
```

### 第4节：skill.md 详解

**必需字段**:
```markdown
# Skill Name
## Description
简要描述技能的功能和用途

## Instructions
详细的使用指令和步骤

## Parameters
- param1: 参数1说明
- param2: 参数2说明
```

### 第5节：渐进式披露机制（Lazy Loading）

| 组件 | 加载策略 | 目的 |
|-----|---------|------|
| **名称和描述** | 始终保留在上下文 | 让代理知道技能存在 |
| **主要指令** | 匹配时加载 | 节省上下文窗口 |
| **参考文件** | 使用时加载 | 避免信息过载 |
| **资源文件** | 按需加载 | 优化性能 |

**工作流程**:
```
用户请求 → 匹配技能描述 → 加载skill.md → 执行指令 → 按需加载资源
```

## 第四部分：高级组合能力

### 第8节：Skill + MCP（Model Context Protocol）

**MCP 的作用**: 连接外部数据源

**工作流程**:
```
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 外部数据 │ → │   MCP   │ → │  Skill  │
│  源     │    │ 连接器  │    │ 处理数据 │
└─────────┘    └─────────┘    └─────────┘
```

**应用场景**:
- 从数据库获取数据，用 Skill 分析
- 从 API 获取信息，用 Skill 整理
- 从文件系统读取，用 Skill 处理

### 第9节：Skill + 子代理（Sub-agent）

**子代理特点**:
- 具有独立上下文
- 可自身使用 Skill 获取专业知识
- 任务委托后并行执行

**架构示例**:
```
主代理（Manager）
    ├── 子代理A（使用Skill X）
    ├── 子代理B（使用Skill Y）
    └── 子代理C（使用Skill Z）
        ↓
    汇总结果
```

### 第10节：多 Skill 组合

**组合策略**:
| 策略 | 说明 | 示例 |
|-----|------|------|
| **顺序执行** | 按顺序调用多个Skill | 研究 → 写作 → 编辑 |
| **条件执行** | 根据结果选择Skill | 如果是数据任务用A，否则用B |
| **并行执行** | 同时调用多个Skill | 同时生成文本和图片 |

## 第六部分：最佳实践

### 第44节：何时使用 Skill？

> "假设你有一个需要反复让代理执行的工作流程，与其每次都解释相同的流程，你可以将其封装为一个技能。"

**使用标准**:
| 场景 | 是否使用Skill |
|-----|-------------|
| 一次性任务 | ❌ 不需要 |
| 重复性工作流程 | ✅ 必须封装 |
| 需要标准化执行 | ✅ 推荐封装 |
| 跨项目复用 | ✅ 必须封装 |
| 复杂多步骤任务 | ✅ 建议封装 |

### 第45节：Skill 设计原则

| 原则 | 说明 |
|-----|------|
| **单一职责** | 每个Skill只做一件事 |
| **清晰描述** | 描述准确，便于匹配 |
| **完整指令** | 步骤清晰，无歧义 |
| **可测试** | 提供测试用例 |
| **可扩展** | 易于添加新功能 |

### 第46节：版本管理与迭代

**版本控制**:
```yaml
# skill.md
version: "1.2.0"
changelog:
  - "1.2.0: 添加批量处理功能"
  - "1.1.0: 优化错误处理"
  - "1.0.0: 初始版本"
```

**迭代流程**:
```
使用Skill → 收集反馈 → 识别改进点 → 更新Skill → 重新部署
```
