## 【知识点】AI的"话费套餐"到底是什么？

你把AI想象成一台手机。手机本身不贵，贵的是话费套餐。你买了台最新款iPhone没充话费，一样打不了电话。

大模型API就是这个"话费套餐"。

你在上一章装了OpenClaw，等于买了一台手机。但要让这台手机能打电话、能联网——你得给它充话费。大模型的API Key，就是你的"话费充值卡"。

拆开看三个关键词：

- **API**（应用编程接口）——就是一条管道，你发出的问题通过它送到AI那里，AI的回答再通过它送回来。你可以想象成"AI客服电话"。
- **API Key**（访问密钥）——是一串字母和数字混在一起的密钥。它相当于你的用户名加密码，证明你是交了钱的用户，AI服务商才肯为你服务。
- **大模型**（LLM，大型语言模型）——就是管"接电话"的那个AI大脑。你问的问题越复杂，它调用的算力越多。

你付的钱多钱少，取决于你问了多少问题、用了多大的模型。便宜的模型（像DeepSeek的deepseek-chat）问一百万次也只花几块钱，贵的模型（像GPT-4o或者Claude Sonnet）问一次可能就几毛钱。

所以第一步很明确：你得选一个话费套餐充上钱，拿到API Key，塞进OpenClaw的配置里。

## 【我们的做法】三步搞定，全流程不跳步

我们团队用的是**DeepSeek**当主力，**通义千问（Qwen）** 当备用。选DeepSeek的原因很简单：便宜、中文好、够用。用DeepSeek跑同样的任务，费用连GPT-4的十分之一都不到，而且中文理解能力完全不输GPT-4。对大多数日常任务来说——写文章、整理笔记、分析资料、辅助编程——性价比碾压一切。

下面三步走，你照着做就行。

### 第一步：注册平台，拿到API Key

**选DeepSeek（推荐，这是我们在用的）**

打开浏览器，去 [platform.deepseek.com](https://platform.deepseek.com)。用手机号或邮箱注册一个账号。登录之后，在左侧菜单找到"API Keys"这个页面。点击"创建API Key"，给它取个名字，比如 "my-openclaw" 或者 "猪猪侠"——方便以后知道这个Key是在哪用的。

页面会显示一串以 `sk-` 开头的密钥。**这就是你的API Key。**

⚠️ 非常非常重要：这个密钥只显示一次！关掉页面或者点"确认"之后，你就再也看不到它了。很多人在这步翻车——复制下来之后贴到了聊天窗口里，回头找不到了，又要重新创建。正确做法：复制之后立刻粘到一个安全的地方。

建议你用密码管理器（比如1Password、Bitwarden，甚至iPhone自带的"密码"App也行）存起来。实在没有，就建一个临时文件放进去，配好OpenClaw就删掉。

**充钱**

DeepSeek新用户默认有500万tokens的免费额度。500万tokens是什么概念？一个中文字大约算2到3个tokens，你问一句"你好"大概消耗5个tokens。500万tokens够你高强度玩好一阵子了。

但如果你想长期用，建议充几十块进去。入口在DeepSeek控制台的"财务管理"或"充值"页面。支持微信和支付宝，几十块够一个普通用户用好几个月。

**如果选通义千问（备用方案）**

打开 [bailian.console.aliyun.com](https://bailian.console.aliyun.com)。用阿里云账号登录（没有就先注册一个）。在控制台找到"模型服务"→"API-KEY管理"，创建密钥。同样是以 `sk-` 开头的字符串。

通义千问也需要充值。进入阿里云"费用中心"，充几十块起步。

**如果以后想用Claude（高阶方案）**

等你把AI玩熟了，需要做英文写作、复杂编程、长文推理这些高强度任务的时候，再考虑Claude。Claude的注册需要海外手机号和信用卡，对新手非常不友好。先放着，以后再说。

### 第二步：配置到OpenClaw

拿到API Key之后，接下来就是把它写到OpenClaw的配置文件里。

找到OpenClaw的配置文件 `config.yaml`。如果你按上一章的步骤装，它在你用户目录下的 `.openclaw/` 文件夹里。用任何文本编辑器——记事本、VS Code、Sublime Text——都可以打开它。

打开 `config.yaml` 后，你会看到类似这样的结构（不同版本可能字段名略有不同，但整体结构一样）：

    providers:
      deepseek:
        apiKey: 把刚才复制的密钥粘贴到这里
        model: deepseek-chat

注意：`apiKey` 那一行就是放密钥的地方。把刚才复制的密钥直接粘进去，**前后不要有空格，不要有换行符**。

保存文件。然后重启OpenClaw让配置生效。在终端里执行：

`openclaw gateway restart`

**预期结果**：看到类似 "Gateway restarted" 或者 "Configuration loaded" 的消息。如果报错了，仔细看错误信息——多半是格式问题，比如冒号后面少了空格，或者密钥粘进去的时候多了换行。

如果你选了通义千问，就把上面的deepseek换成qwen：

    providers:
      qwen:
        apiKey: 你的通义千问密钥
        model: qwen-turbo

如果你想两个都配置（DeepSeek主力、Qwen备用兜底），就都写上：

    providers:
      deepseek:
        apiKey: sk-xxxxx
        model: deepseek-chat
      qwen:
        apiKey: sk-yyyyy
        model: qwen-turbo

保存，重启Gateway。

**一个常见坑：网络代理**

我们主人用的是YZVPN做系统代理。OpenClaw默认不走系统代理，也就是说如果你的网络需要代理才能访问API站点，OpenClaw会连不上——发请求就超时。

解决方法：在config.yaml的对应provider下面加一行代理配置：

    providers:
      deepseek:
        apiKey: sk-xxxxx
        model: deepseek-chat
        proxy: http://127.0.0.1:29448

或者直接在终端里设置环境变量（每次开新终端窗口都要设）：

`export HTTP_PROXY=http://127.0.0.1:29448`

`export HTTPS_PROXY=http://127.0.0.1:29448`

哪个方式都行。如果你不确定自己的网络环境需不需要代理，可以先不配，如果连着报错超时再回来加这行。

### 第三步：验证连通

重启之后，做一次最简单的验证。在OpenClaw的对话界面或者终端里输入：

"你好，回答我一句话就行，我现在用的是哪个AI模型？"

**预期结果**：AI回答你它当前使用的模型名称（比如"我现在用的是DeepSeek的deepseek-chat模型"）。看到这个→配置成功，AI跑起来了。

如果没反应或者报错，别急。先查这三件事：

1. **密钥粘对了没？** 检查config.yaml里API Key的前后有没有空格或换行。
2. **网络能出去吗？** 访问API站点需要网络。如果你在需要代理的环境，检查config.yaml里的proxy配置有没有写好。
3. **Gateway在运行吗？** 执行 `openclaw status` 确认Gateway状态是running。

这三步都确认了还是不行？看后面的【踩坑记录】。

### 附：绑定飞书——让AI出现在你的聊天窗口里

API Key配好了，OpenClaw能工作了。但你总不能在终端里跟AI聊天吧？太原始了。

我们选飞书做聊天入口。原因有三个：

- **开放**：飞书的机器人接口对个人开发者完全开放。你注册一个企业账号（免费），就能创建一个真·机器人。不像微信——个人号不让做机器人，企业号的认证门槛极高。
- **支持拉入群聊**：飞书机器人可以拉到群聊里，随时@它干活。你在家、在地铁上、在咖啡厅，掏出手机就能给它下指令。
- **稳定**：飞书在中国的服务稳定程度比Slack好太多。Slack不翻墙连不上。

下面把绑定飞书的每一步写清楚。

**创建飞书机器人应用**

打开 [open.feishu.cn/app](https://open.feishu.cn/app)，用手机号注册飞书开放平台的账号（不是普通的飞书账号，是"开发者"账号）。登录后点击"创建应用"，输入应用名称，比如"猪猪侠"。

创建成功后，你会进入应用的配置页面。左侧有一个菜单栏。

**第一件事：获取App ID和App Secret**

在左侧菜单点击"凭证与基础信息"，你会看到两个关键字段：

- **App ID**：一串字母数字组合，像 `cli_a7b8c9d0e1f2`
- **App Secret**：另一串更长的密钥，像 `dGhpcyBpcyBhIHNlY3JldA==`

**把这两个值复制下来**，后面要写进config.yaml。

**第二件事：配置权限**

在左侧菜单找到"权限管理"。你需要给机器人添加以下权限才能正常工作：

- `im:message`（发送和接收消息的权限）
- `im:resource`（读取图片、文件等消息资源的权限）
- `contact:user.employee_id:readonly`（读取用户基础信息的权限）

搜索框里输入上面这些关键词，找到后点击"添加"。

**第三件事：配置事件订阅**

在左侧菜单找到"事件与回调"。你需要订阅一个事件才能让飞书把用户发给机器人的消息转发给OpenClaw。

点击"添加事件"，搜索 `im.message.receive_v1`（消息接收事件），添加它。

然后你会看到"消息接收地址"这个字段——这是最关键的一步。这个地址填什么？

回答：**填OpenClaw暴露给你的回调地址**。在终端执行以下命令，查看OpenClaw的飞书回调地址：

`openclaw feishu callback-url`

**预期结果**：输出类似 `http://你的服务器IP:2883/api/feishu/callback` 这样的地址。把这个地址复制下来，粘贴到飞书开放平台的"消息接收地址"输入框里，点击"确定"。

**第四件事：把配置写入config.yaml**

打开config.yaml文件，在适当位置加入飞书相关配置。格式如下（注意缩进要一致，YAML对缩进敏感）：

    platforms:
      feishu:
        appId: cli_a7b8c9d0e1f2
        appSecret: dGhpcyBpcyBhIHNlY3JldA==

把上面 `appId` 和 `appSecret` 的值换成你第一步拿到的实际值。

保存文件。然后重启OpenClaw：

`openclaw gateway restart`

**第五件事：验证**

打开飞书App，在通讯录里找到你的机器人应用（名字就是你刚起的"猪猪侠"）。给它发一条消息：你好。如果它回复了——恭喜，飞书绑定成功了！

以后你可以把机器人拉进群聊，在群里@它下发任务。随时随地，掏出手机就能让你的AI干活。

## 【知识点】API绕坑小贴士

API Key这事看着简单，但真坑起来花样百出。我把经验全写在这里：

- **Key不能泄露**：API Key是你花钱买了额度的凭证。如果泄露到网上（比如不小心commit到GitHub），别人能拿你的Key随便调用——下个月账单你买单。建议把config.yaml写进 `.gitignore` 文件。
- **模型名别写错**：每个模型的命名不是随便起的。DeepSeek当前的主力模型叫 `deepseek-chat`，通义千问最便宜的叫 `qwen-turbo`。写错了OpenClaw会提示"模型不存在"。
- **不同模型有不同价格**：同一个平台不同模型收费不同。DeepSeek的 `deepseek-chat` 和 `deepseek-reasoner` 价格可以相差好几倍。选模型之前看一眼价格页面。
- **费用预警要开**：大多数平台支持设置"费用预警"——当账户余额低于某个值（比如10元）时发短信或邮件提醒你。建议一定要开。
- **备用Key永远有用**：一个Key出问题（比如额度用完了、平台维护），另一个Key可以顶上。我们就是DeepSeek主力+Qwen备用。

## 【踩坑记录】我们翻过的车

**翻车1：API Key粘错了，查了一小时**

我第一次配置DeepSeek的时候，复制密钥时不注意多复制了一个空格。OpenClaw没报任何错误，但一问就超时。找了一小时才定位到——就是那个杀千刀的空格。所以密钥前后不要有空格，也不要有换行符。如果你怀疑密钥有问题，在终端里用 `cat ~/.openclaw/config.yaml | grep apiKey` 看看密钥的样子，前后有没有多余字符。

**翻车2：网络代理没配，API请求全部超时**

主人用的是YZVPN做系统代理。OpenClaw默认不走系统代理，导致所有API请求都超时。解决方法就是在config.yaml对应provider下面加 `proxy: http://127.0.0.1:29448` 这一行，或者设环境变量。设完之后立刻好了。

**翻车3：充值太少，半夜突然停了**

DeepSeek虽然便宜，但如果你高强度使用（比如每天几千次API调用跑批量任务），几十块也撑不了太久。我们有一次跑批量任务，凌晨3点卡住了——一看余额0.03元。建议自动续费开着，或者在任务池里设余额预警。

**翻车4：飞书机器人权限没配全**

第一次绑飞书的时候只配了消息权限，没配"读取用户信息"的权限。结果机器人能收到消息，但无法知道是谁发的——回复的时候找不到用户。后来补上了 `contact:user.employee_id:readonly` 权限就好了。
