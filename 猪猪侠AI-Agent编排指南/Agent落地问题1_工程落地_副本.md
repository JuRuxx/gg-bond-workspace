"真实的 Agent 落地"三部曲第一篇，聚焦工程层面的真实挑战。

文章首先区分了三种"Agent"：Workflow 型（预定义 DAG，目前落地最多）、ReAct 型（模型自己决定下一步做么）、Multi-Agent 型（多 Agent 协作）。关键洞察是：绝大多数声称做第二或第三种的公司，最终交付的其实是第一种——因为在生产环境中，不可控的东西就是不可控的，而 SLA 不允许不可控。

ReAct 循环工程化面临的难题远超论文描述。循环终止问题：模型什么时候该停？模型自己判断可能太早（任务没完成）或太晚（白烧钱）。循环退化问题：随着上下文越来越长，模型在后续步骤中变得重复、犹豫甚至矛盾。错误放大问题：第 N 步的错误通过上下文传递到 N+1 步，像滚雪球一样越滚越大。

工具调用是另一个重灾区。参数幻觉：模型自信地编造 customer_id 或日期。JSON 输出脆弱性：格式错误、类型不匹配、字段缺失。工具返回值处理：信息过载（50 个字段塞进上下文干扰模型注意力）和信息不足（只返回 success 但模型需要知道更多）。

经过无数次线上事故后，大多数团队走向一个务实的中间地带：用 Workflow 搭骨架，在关键节点用 Agent 做决策。但这方案的上限也很明显——只能处理设计好的场景。
