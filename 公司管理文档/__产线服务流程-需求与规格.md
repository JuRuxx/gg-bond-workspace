# 技术部产线服务流程 - 需求与规格

## 需求

技术部同事反映生产部叫他们去产线太随意，一个招呼就随叫随到，影响正常项目工作。需要建立一个简单的流程，让生产部和技术部之间有明确的报修通道和服务标准。

## 规格

### 文档定位
- 一页A4纸，简洁直接
- 技术部出草案 → 生产部主管对齐 → 双方签字执行
- 先跑起来再优化，不要一上来就搞系统

### 内容要求

1. **故障分级 P0-P3** — 每级必须有明确定义+典型例子，让双方一看就懂
2. **报修流程** — 生产部怎么叫技术（入口、填什么、找谁）
3. **技术部响应机制** — 值班制、响应时间、排程原则
4. **异常升级规则** — 搞不定的怎么办

### 格式
- 标题：奥普新(陆河)科技有限公司 - 技术部产线服务流程
- 正文分四节，每节一个小标题+几句话
- 末尾留签名区：技术部主管 / 生产部主管 / 日期
- 语气：正式但不官僚，生产部和技术部都能接受
