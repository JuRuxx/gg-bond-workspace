## 一、本章定位

- **所属书籍**：《AI助手实战手册》（《猪猪侠 AI Agent 编排实战指南》配套副本）
- **章节编号**：第一章
- **章节标题**：装好你的AI副手——Claude 桌面客户端 + CC-Switch 极简安装
- **培训时长**：约30分钟
- **培训对象**：自动化设备厂文职人员（4人，完全零基础）
- **学员设备**：混合Windows / macOS，部分机器可能老旧
- **前置知识**：不需要任何编程基础，不需要命令行经验
- **核心工具链**：
  - **Claude 桌面客户端**（主界面）— 带图形界面，直接打字跟AI对话
  - **CC-Switch**（配置工具）— 解决网络访问问题，可配置接入各类AI模型
  - **不涉及**：终端命令行、Node.js、Git、VSCode

## 二、目标与验收标准

> 一句话目标：让学员**把AI装上并跑通一次**，建立"我真的能学会"的信心。

### 2.1 硬性验收（必须达成）

| 序号 | 验收项 | 判断标准 |
|------|--------|----------|
| 1 | Node.js 安装成功 | 打开终端/命令提示符，输入 `node -v` 能显示版本号（≥18.x） |
| 2 | Claude Code CLI 安装成功 | 输入 `npx @anthropic-ai/claude-cmd --version` 能显示版本号 |
| 3 | API Key 配置成功 | 执行 `claude-cmd` 后不报"API key missing"错误 |
| 4 | 第一次对话成功 | 学员让AI做一件简单工作（如"把'hello'改成大写"），A正确输出了结果 |
| 5 | 确认数据本地处理 | 演示/说明：任务数据仅在本地流转，不发送到外部服务器（Claude API请求除外） |

### 2.2 软性验收（尽量做到）

| 序号 | 验收项 | 判断标准 |
|------|--------|----------|
| 1 | 学员在30分钟内全部完成安装 | 4人都能在培训课时内跑通 |
| 2 | 学员理解"终端"是什么 | 能说出"终端就是给电脑打字让它干活的地方" |
| 3 | 学员知道终端里哪些操作是安全的 | 会识别"以 $ 或 > 开头的是要输入的命令" |
| 4 | 学员体验成功后露出笑容 | 嗯，这个靠观察 |

### 模块四：安装 CC-Switch（约8分钟）

**内容要求：**
- 区分 Windows 和 Mac 两条路径，分步写
- 打开浏览器，访问 CC-Switch GitHub Releases 页面
- 下载对应系统的安装包（Windows → .msi / Mac → .dmg）
- 双击安装，一路下一步
- **安装完成后**，打开 CC-Switch
- 配置 DeepSeek 或其他国内可用模型：
  - 去 DeepSeek 开放平台注册账号
  - 创建 API Key
  - 在 CC-Switch 中添加 DeepSeek Provider，粘贴 API Key
  - 选择模型（DeepSeek V4 Pro / 其他）
  - 测试连接是否正常

**交付要求：**
- 给出两种操作系统的**完整分步指南**（编号1–N）
- 给出 DeepSeek API Key 获取的完整截图流程（从注册到复制Key）
- 给出 CC-Switch 内配置的截图（添加Provider → 填入Key → 测试连接）
- ⚠️ **安全警告**：API Key = 钱，不要分享、不要截图发给别人
- ⚠️ **费用说明**：DeepSeek V4 Pro 百万上下文，价格实惠，首次使用一般免费额度足够
- 避雷：
  - API Key 关闭对话框后不可再查看，一定要复制保存
  - Mac 首次打开 CC-Switch 可能提示"无法验证开发者"，需要去"系统设置→隐私与安全性→仍要打开"
  - 如果测试连接失败，检查 API Key 是否正确复制

### 模块七：关于数据安全（约2分钟）

**内容要求：**
- 说明架构：你的文字→Claude桌面客户端→（经过CC-Switch）→AI服务器→回传到你的屏幕
- 强调：你发给AI的那段话会经过网络，但AI公司不会拿你的数据去训练
- 一句话总结："只有你发给AI的那句话出去了，你电脑上的文件不会"
- CC-Switch 的作用之一就是中间做代理，数据流转可控

**交付要求：**
- 不要承诺"100%不上传"，那是假的
- 实话实说：如果你输入机密数据，它会经过网络
- 给出规避建议：不要粘贴完整合同内容，只粘贴需要改写/翻译的那几行

### 模块八：本章小结 + 常见问题排查（约2分钟）

**内容要求：**
- 安装三步走回顾：装Node.js → 装Claude Code → 配API Key
- 常见卡壳点对照表

**交付要求：**

| 问题现象 | 解决方案 |
|----------|----------|
| `node -v` 提示"不是内部或外部命令" | 没装成功或没重启，重装.msi并勾选"Add to PATH" |
| `npx` 报"execution policy" | 以管理员身份打开终端，运行 `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` |
| `npx` 下载卡住很久不动 | 检查网络连接；尝试断网重连后再试 |
| 输入API Key后报401/unauthorized | Key无效或过期，重新生成一个 |
| 忘记API Key生成页在哪 | [console.anthropic.com](https://console.anthropic.com) 登录 → API Keys |
| 终端窗口不知道怎么操作 | 只需要：输入命令 → 按回车 → 看结果 |

## 五、风险点

### 5.1 环境风险

| 风险 | 概率 | 应对方案 |
|------|------|----------|
| Windows 电脑太旧（Win7） | 中 | 说明仅支持 Win10+ / macOS 11+，老旧设备建议IT协助升级或用Mac |
| Windows 用户权限不够（不能安装软件） | 中 | 提前跟IT部门沟通，确保权限开放；或用便携版(nvm)绕过 |
| Mac 电脑太旧（macOS < 11） | 低 | Node.js 18+ 要求 macOS 11+，老系统只能降级装Node 16 |
| 32位操作系统 | 低 | Node.js 已停止提供32位版本，遇到则不可行 |
| 公司网络限制（npx下载被墙或走代理问题） | 中 | 提前准备Node.js离线安装包 + 在可上网的机器上先 `npm cache` |
| 硬盘空间不足 | 低 | Node.js安装包只有30-50MB，影响不大 |

### 5.2 学员心理风险

| 风险 | 应对方案 |
|------|----------|
| 看到终端黑窗口就害怕 | 语气要轻松："它就是个高级点的微信输入框" |
| 输入命令时手抖打错 | 提供完全可复制的命令文本（加一键复制按钮），错误时不批评 |
| 看不懂报错信息 | 学员手册里附一个"常见报错中文翻译对照表" |
| 觉得自己"学不会" | 模块一就要给安全感，每一步都配上截图，让学员跟着做就行 |

### 5.3 教学节奏风险

| 风险 | 应对方案 |
|------|----------|
| 有人快有人慢 | 快的学员可以帮慢的（提前设计"帮帮环节"） |
| 超时30分钟 | 如果过半人卡在安装，把"验证对话"延到下次课 |
| 某设备死活装不上 | 准备一台预装好的备用电脑，让学员先操作体验 |
