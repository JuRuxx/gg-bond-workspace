# 逆向工程基础

> *从自动化厂的角度看逆向工程——寄存器就是PLC的D寄存器、PE结构就是设备的BOM表、反调试就是安全光幕。*

---

## 第一章：x86 汇编基础

### 1.1 为什么要学汇编？

**汇编是CPU唯一真正懂的语言。**

C / Python / C# 写出来的高级代码，编译后会变成机器码（一串 0101），**汇编就是机器码的人类可读版本**。

当你在逆向一个工业软件时：
1. 用反汇编工具（IDA Pro / Ghidra）打开一个 .exe
2. 看到的不是高级语言的代码，是一行一行的 **汇编指令**
3. 只有看懂这些汇编，才能找到 `CheckDog()` 函数的验证逻辑在哪

**所以汇编是逆向工程的看家本领，绕不过去。**

> **🖥️ 类比：** 汇编之于逆向，相当于梯形图之于PLC维修工——你可能平时用高级语言写上位机，但真要查故障的时候，最终还是要看懂梯形图。

### 1.2 寄存器——CPU 的「随身笔记本」

CPU 有**寄存器**，相当于它手边最常用的"小本本"，读写速度比内存快几百倍。

#### 为什么要有寄存器？

| 层级 | 类比 | 速度 | 大小 |
|:----|:-----|:-----|:-----|
| 硬盘 | 厂区的巨大仓库 | 最慢 | TB级 |
| 内存 | 工位旁边的物料架 | 中等 | GB级 |
| **寄存器** | **工人工作台上的工具** | **最快** | **字节级** |

**原因很简单**：CPU 执行一条指令如果每次都要去内存取数据，等于工人每拧一颗螺丝都要跑去仓库拿——太慢了。所以 CPU 把手边最常用的数据放在寄存器里。

#### 通用寄存器（32位 x86）

```
EAX  — 累加器 — 算术运算结果放这，★函数返回值也放这★
EBX  — 基址寄存器 — 偶尔存地址
ECX  — 计数器 — 循环次数放这（for(i=0;i<n;i++) 的 i）
EDX  — 数据寄存器 — 配合 EAX 做乘法/除法

ESI  — 源索引
EDI  — 目标索引
EBP  — 基址指针 — 指向当前栈帧底部
ESP  — 栈指针 — 指向当前栈顶 ★★★★★
EIP  — 指令指针 — 指向下一条要执行的指令 ★★★
```

> **🖥️ 类比：** 寄存器就像你们厂PLC的 **D寄存器（数据寄存器）**——临时存放数值的工作单元。`EAX` 像点胶机的胶量传感器，每次运算结果都记在这。

#### 32位 → 64位 的名字变化

```
32位 → 64位
────────────────
EAX  → RAX（R = Register，64位）
EBX  → RBX
ECX  → RCX
...
EIP  → RIP
```

**地址长度也不同**：
- 32位：地址 4 字节，范围 `0x00000000` – `0xFFFFFFFF`
- 64位：地址 8 字节，范围 `0x0000000000000000` – `0xFFFFFFFFFFFFFFFF`

#### 32位寄存器的内部拆解

```asm
EAX（32位）= 高16位 + 低16位
   ├── AX（低16位）= AH（高8位）+ AL（低8位）
   └── 可以单独访问

示例：
MOV AL, 0xFF      ; 只改 EAX 的最低8位
MOV AH, 0x01      ; 只改 EAX 的第8-15位
MOV AX, 0xFFFF    ; 只改 EAX 的低16位
MOV EAX, 0x12345678 ; 改整个32位
```

#### 标志寄存器（EFLAGS）— CPU 的「红绿灯」

CPU 做运算时会自动记录结果的特征：

| 标志 | 全名 | 什么时候=1 | 类比例子 |
|:----|:-----|:----------|:---------|
| **ZF** | 零标志 | 计算结果为 0 时 | 良品率=100%时亮灯 |
| **SF** | 符号标志 | 结果为负数时 | 产量=负数时报警 |
| **CF** | 进位标志 | 加法进位或减法借位 | 计数器溢出时亮灯 |
| **OF** | 溢出标志 | 有符号数溢出时 | 传感器超量程时报警 |

**为什么逆向要看标志位？**

因为 `CMP EAX, EBX` 这条指令**不存结果到任何寄存器**，它只**影响标志位**。然后下一条 `JE 标签` 就**看 ZF** 来决定跳不跳。

### 1.3 常用指令

以下10条指令，够看懂80%的反汇编代码。

#### ① MOV — 数据搬砖

**格式**：`MOV 目标, 源`

```asm
MOV EAX, 1234         ; 把数值 1234 放入 EAX
MOV EBX, EAX          ; 把 EAX 的值复制到 EBX
MOV ECX, [123456]     ; 把内存地址 123456 处的值读入 ECX（加[]表示取地址里的值）
MOV [EAX], EDX        ; 把 EDX 的值写入 EAX 指向的内存地址
```

> **🖥️ 类比：** `MOV` 就像 PLC 的 `MOV D100 K1234` 指令。

#### ② ADD / SUB — 加减法

```asm
ADD EAX, 10      ; EAX = EAX + 10
SUB EBX, ECX     ; EBX = EBX - ECX
```

#### ③ PUSH / POP — 压栈 / 出栈

**栈 = 一种 LIFO（后进先出）的数据结构**，是函数调用的核心。

```asm
PUSH EAX    ; 把 EAX 的值压到栈顶（ESP 自动减4）
POP  EBX    ; 从栈顶弹出数据到 EBX（ESP 自动加4）
```

**栈的工作过程**：
```
初始状态：ESP → [   栈顶    ]
                 [          ]
                 [          ]

PUSH EAX 后：
         ESP → [  EAX值  ]  ← 新压入
                 [          ]
                 [          ]

POP EBX 后：
         ESP → [   栈顶    ]  ← EAX值被弹出到EBX
                 [          ]
                 [          ]
```

> **🖥️ 类比：** 就像你们产线的 **栈式料架**——后放上去的磁规先被拿走（LIFO）。

#### ④ CALL / RET — 函数调用

```asm
CALL 0x401000      ; 1. 把下一条指令地址压栈
                   ; 2. 跳转到 0x401000 执行
                   ; 相当于 C 语言的 func()

RET                ; 从栈顶弹出地址，跳回去继续执行
                   ; 相当于 C 语言的 return
```

**函数调用的完整流程**：
```
; 调用方
PUSH 参数3          ; 从右到左压栈
PUSH 参数2
PUSH 参数1
CALL 函数地址        ; 内部：PUSH 返回地址 + JMP

; 被调用方
PUSH EBP            ; 保存上一个函数的 EBP
MOV  EBP, ESP       ; 设置自己的栈帧基址
SUB  ESP, 64        ; 分配64字节局部变量空间
...                 ; 函数体
MOV  ESP, EBP       ; 恢复栈顶
POP  EBP            ; 恢复上一个函数的 EBP
RET                 ; 弹出返回地址并跳回
```

#### ⑤ CMP / JMP / Jcc — 比较和跳转

```asm
CMP EAX, EBX       ; 比较 EAX 和 EBX（做减法，只影响标志位）
JE  标签           ; 如果相等（ZF=1），跳转
JNE 标签           ; 如果不相等（ZF=0），跳转
JG  标签           ; 如果大于
JL  标签           ; 如果小于
JMP 标签           ; 无条件跳转（相当于 goto）
```

**对应的高级语言**：
```c
if (eax == ebx) {
    goto 标签;   // JE
}
```

**这是逆向中最关键的地方！** 加密狗验证、注册码验证，最后都是通过 `CMP` + `JE/JNE` 来决定"过"还是"不过"。

#### ⑥ XOR — 异或（常见清零）

```asm
XOR EAX, EAX       ; EAX = 0（任何数异或自身 = 0）
                   ; 比 MOV EAX, 0 更快
                   ; 逆向中看到这个基本就是清零
```

**XOR 的性质**：
```c
a ^ a = 0          // 自己异或自己 = 0
a ^ 0 = a          // 异或0不变
a ^ b ^ b = a      // 两次异或恢复原值（用于加密）
```

#### ⑦ LEA — 取地址（不是取值）

```asm
LEA EAX, [EBP-8]   ; 把 EBP-8 这个地址值放入 EAX
                   ; 相当于 C 语言的 &（取地址运算符）
```

**LEA 与 MOV 的区别**：
```asm
MOV EAX, [EBP-8]   ; 取 EBP-8 地址里存的值
LEA EAX, [EBP-8]   ; 取 EBP-8 这个地址本身
```

### 1.4 栈与函数调用约定

#### 调用约定（Calling Convention）

**调用约定 = 函数之间关于"怎么传参数、谁负责清理栈"的约定。**

| 约定名 | 参数传递方式 | 谁清理栈 | 最常见于 |
|:-------|:------------|:---------|:---------|
| **stdcall** | 从右到左压栈 | 被调用方 | Windows API |
| **cdecl** | 从右到左压栈 | 调用方 | C/C++ 默认 |
| **fastcall** | 前两个用 ECX/EDX，其余压栈 | 被调用方 | 性能敏感代码 |
| **thiscall** | this 指针放 ECX | 被调用方 | C++ 成员函数 |

**最重要的规则（逆向黄金法则）**：
> **函数返回值永远在 EAX 中。**

所以当你看到 `CALL CheckDog` 后面跟着 `MOV [EBP-4], EAX`，就是在把验证结果存到局部变量里。EAX = 0 → 没狗，EAX ≠ 0 → 有狗。

### 1.5 实战：代码解读

```asm
; 函数开场
PUSH EBP
MOV  EBP, ESP
SUB  ESP, 0x10           ; 分配16字节局部变量

; 调用加密狗验证
CALL DWORD PTR [401200]  ; 调用 CheckDog()
MOV  [EBP-4], EAX        ; 返回值存到局部变量
CMP  DWORD PTR [EBP-4], 0 ; 比较是否为0
JE   SHORT fail           ; 如果=0（没狗），跳到失败处理

; 有狗，正常运行
PUSH 0
CALL MessageBoxA          ; 显示"欢迎使用"

fail:
; 没狗，弹错误
PUSH 字符串地址_无狗       ; 文本
PUSH 字符串地址_标题       ; 标题
CALL MessageBoxA           ; 显示"请插入加密狗"
MOV  ESP, EBP
POP  EBP
RET
```

**翻译成 C 语言**：
```c
int has_dog = CheckDog();
if (has_dog == 0) {
    MessageBoxA(NULL, "请插入加密狗", "错误", 0);
    // 程序退出或限制功能
} else {
    MessageBoxA(NULL, "欢迎使用", "提示", 0);
    // 正常运行
}
```

**破解思路**：把 `JE fail` 改成 `JMP fail`（永远不去失败处理）或 `NOP` 掉，或者把 `CMP` 改成 `XOR EAX, EAX` 让返回值永远等于0后反转逻辑。

### 1.6 本章小结

| 知识点 | 要点 |
|:------|:-----|
| **寄存器** | EAX（返回值）、ESP（栈顶）、EIP（指令指针）、ZF（比较结果） |
| **栈** | LIFO，函数调用的基础 |
| **CALL/RET** | CALL = 压返回地址 + 跳转，RET = 弹出地址 + 跳回 |
| **CMP/Jcc** | 比较 + 条件跳转，**条件跳转是逆向要关注的重点** |
| **调用约定** | 返回值在 EAX |
| **逆向口诀** | 看到 CMP → 找 JE/JNE → 那就是验证逻辑的关键跳转 |
