1. 权威研究机构报告（IDC、Gartner、弗若斯特沙利文）
2. 上市公司财报和招股书
3. 行业协会数据
4. 权威媒体报道（36氪、虎嗅、晚点）
5. 咨询公司报告（麦肯锡、BCG）

**数据验证**：

## 二、行业研究维度清单（按优先级）

### 必选维度（每次研究必做）
1. **市场规模与增长**：当前规模、历史增长、预测增长
2. **产业链分析**：上下游、关键环节、价值链分布
3. **竞争格局**：头部玩家、市场份额、竞争维度
4. **技术趋势**：当前主流、未来方向、技术壁垒
5. **风险挑战**：技术风险、市场风险、政策风险

### 重要维度（根据研究目的选择）
6. **政策环境**：支持政策、监管政策、政策趋势
7. **投资热点**：投资规模、投资机构、投资逻辑
8. **未来展望**：短期、中期、长期趋势
9. **机会策略**：机会识别、进入策略、关键成功因素

### 可选维度（深度研究时补充）
10. **国际市场**：海外市场规模、竞争格局、进入策略
11. **细分领域**：各细分领域的详细分析
12. **用户调研**：用户需求、用户行为、用户痛点
13. **商业模式**：行业内主流商业模式分析

## 三、不同行业类型的研究重点

### 新兴行业（如AI Agent、Web3）
**重点关注**：
- 技术路线（是否确定？）
- 商业化路径（如何变现？）
- 市场教育成本（用户接受度）
- 政策风险（监管态度）
- 投资泡沫（估值合理性）

**警惕信号**：
- 技术路线不统一
- 商业化困难
- 过度依赖融资
- 政策不明确

---

### 成熟行业（如电商、SaaS）
**重点关注**：
- 市场集中度（是否垄断？）
- 增长率（是否放缓？）
- 利润率（是否下降？）
- 创新方向（第二曲线）
- 国际化机会（出海）

**警惕信号**：
- 增长停滞
- 价格战激烈
- 利润率下滑
- 创新乏力

---

### 周期性行业（如半导体、新能源）
**重点关注**：
- 周期位置（上行/下行）
- 供需关系（产能利用率）
- 技术迭代（代际更替）
- 政策影响（补贴、关税）
- 库存水平（库存周转率）

**警惕信号**：
- 产能过剩
- 库存高企
- 价格下跌
- 需求疲软

---

### 监管驱动行业（如医疗、教育）
**重点关注**：
- 政策变化（利好/利空）
- 准入门槛（牌照、资质）
- 合规成本（合规要求）
- 政策趋势（收紧/放松）
- 政策风险（黑天鹅）

**警惕信号**：
- 监管收紧
- 政策不确定
- 合规成本高
- 政策依赖度高

## 四、行业泡沫检测清单

### 投资泡沫信号
- [ ] 近1年投资金额同比增长>200%
- [ ] 平均估值同比增长>100%
- [ ] 早期项目估值过高（天使轮>1亿）
- [ ] 投资机构跟风（FOMO情绪）
- [ ] 媒体报道过度正面

### 市场泡沫信号
- [ ] 市场预测过于乐观（CAGR>50%持续5年）
- [ ] 进入者过多（1年内新增>100家）
- [ ] 同质化严重（产品差异化小）
- [ ] 价格战激烈（毛利率<20%）
- [ ] 客户获取成本高（CAC/LTV>1）

### 技术泡沫信号
- [ ] 技术概念炒作（过度宣传）
- [ ] 技术落地困难（Demo多、产品少）
- [ ] 技术路线不统一（标准未定）
- [ ] 技术壁垒低（容易被复制）
- [ ] 技术迭代过快（投入难回收）

### 商业模式泡沫信号
- [ ] 盈利模式不清晰
- [ ] 商业化困难（收入增长慢）
- [ ] 规模化难（边际成本高）
- [ ] 客户付费意愿低
- [ ] 过度依赖补贴

## 五、行业研究常用分析模型

### 1. 波特五力模型
**用途**：分析行业竞争强度
- 现有竞争者：竞争强度
- 潜在进入者：进入壁垒
- 替代品威胁：替代风险
- 供应商议价能力：强弱
- 客户议价能力：强弱

### 2. PEST分析
**用途**：分析宏观环境
- Political（政治）：政策环境
- Economic（经济）：经济环境
- Social（社会）：社会文化
- Technological（技术）：技术环境

### 3. 行业生命周期
**用途**：判断行业发展阶段
- 萌芽期：技术不成熟、市场小
- 成长期：技术成熟、市场快速增长
- 成熟期：市场饱和、竞争激烈
- 衰退期：市场萎缩、企业退出

### 4. 价值链分析
**用途**：分析产业链价值分布
- 识别高价值环节
- 识别高壁垒环节
- 识别机会点

### 5. TAM-SAM-SOM模型
**用途**：评估市场规模
- TAM（Total Addressable Market）：总可寻址市场
- SAM（Serviceable Addressable Market）：可服务市场
- SOM（Serviceable Obtainable Market）：可获得市场

## 七、行业研究效率提升技巧

### 搜索技巧
1. **优先找行业报告**：搜索"[行业名称] 行业报告 2025"
2. **关注权威机构**：IDC、Gartner、弗若斯特沙利文、艾瑞咨询
3. **看上市公司招股书**：行业分析章节通常很详细
4. **关注行业媒体**：36氪、虎嗅、晚点LatePost
5. **参加行业会议**：演讲PPT通常有干货

### 分析技巧
1. **数据交叉验证**：同一数据至少2个来源
2. **区分事实和观点**：数据是事实，预测是观点
3. **关注边际变化**：不是绝对值，而是变化趋势
4. **第一性原理**：从需求出发，不被表象迷惑
5. **对比分析**：与相似行业、海外市场对比

### 写作技巧
1. **数据可视化**：用图表展示市场规模、增长率、份额
2. **产业链图**：用流程图展示上下游关系
3. **竞争格局图**：用矩阵图展示玩家定位
4. **时间轴**：用时间轴展示发展历程
5. **案例穿插**：用具体案例说明抽象概念

## 八、常见陷阱与规避

### 陷阱1：数据口径不一致
**表现**：不同来源的市场规模数据差异大
**规避**：明确统计口径，注明数据来源，交叉验证

### 陷阱2：预测过于乐观
**表现**：CAGR预测>50%持续5年以上
**规避**：对比历史数据，质疑预测假设，关注制约因素

### 陷阱3：忽视政策风险
**表现**：只看到政策支持，忽视监管风险
**规避**：分析政策趋势，评估政策变化影响

### 陷阱4：技术决定论
**表现**：认为技术好就一定能成功
**规避**：分析商业模式、市场需求、竞争格局

### 陷阱5：忽视周期性
**表现**：把周期性增长当成趋势性增长
**规避**：分析行业周期，判断周期位置

## 十、数据保留规范

> 学术级研究要求每条结论可回溯、可验证。所有原始数据和中间产物必须完整保留。

### 10.1 数据保留目录结构

```
${RESEARCH_DIR}/
├── industry/
│   ├── [行业名称]-PRD-*.md
│   ├── [行业名称]-SPEC-*.md
│   ├── [行业名称]-深度研究报告-*.md
│   ├── [行业名称]-REVIEW-*.md
│   ├── 源数据/
│   │   ├── [行业名称]-行业报告-引用.md
│   │   ├── [行业名称]-权威数据-摘录.md
│   │   └── ...
│   ├── search-records/
│   │   ├── [行业名称]-搜索记录.md
│   │   └── ...
│   └── data-sources/
│       ├── [行业名称]-数据来源表.md
│       └── ...
└── ...
```

### 10.2 搜索记录规范

每次搜索必须保留以下信息：

| 字段 | 说明 |
|:---|:---|
| **搜索轮次** | 第几轮搜索（基础/深度/验证） |
| **搜索关键词** | 实际使用的查询语句 |
| **搜索时间** | 搜索执行的日期时间 |
| **搜索结果摘要** | 返回的关键结果 |
| **引用URL** | 使用的信息源链接 |
| **数据使用** | 是否写入报告、用于哪个章节 |
| **验证状态** | 该数据是否经过交叉验证 |

### 10.3 数据来源表规范

每条写入报告的数据，必须记录：

| 字段 | 说明 |
|:---|:---|
| **数据描述** | 具体数据项 |
| **原始来源** | 数据最初出处 |
| **引用URL/文件** | 可回溯的具体链接或本地文件路径 |
| **获取时间** | 数据获取的日期 |
| **报告章节** | 该数据出现在报告的哪个章节 |
| **验证方式** | 是否经过交叉验证，验证结果 |
| **备注** | 数据备注（如口径说明、误差范围） |

### 10.4 数据保留原则

1. **不删减**：原始搜索记录和中间分析文件不因"报告完成"而删除
2. **可回溯**：报告中的每个结论和数据点，必须能追溯到原始来源
3. **统一格式**：搜索记录和来源表使用统一模板，便于后续检索
4. **版本保留**：搜索记录和来源表随报告版本更新，不覆盖旧版

## 十二、使用说明

### 快速开始
1. 复制PRD模板，填写研究目标和核心问题
2. 复制SPEC模板，根据研究目的选择章节
3. 按搜索策略执行研究，生成CONTENT
4. 复制REVIEW模板，验收研究质量

### 进阶使用
1. 根据行业类型选择"不同行业类型的研究重点"
2. 使用"行业泡沫检测清单"评估投资热度
3. 使用"行业研究常用分析模型"深入分析
4. 参考"常见陷阱与规避"避免研究盲区

### 持续迭代
1. 每次研究后在REVIEW中总结经验
2. 根据经验优化研究框架
3. 更新版本号，记录变更历史
