## 主人指示 - 任务处理流程

**时间**：14:44  
**内容**：主人明确了任务处理原则

### 核心规则
- **简单任务**：猪猪侠自己直接搞定，不用派子Agent
- **复杂任务**：派小呆呆/超人强/迷糊老师，按四步流程走：
  1. **写需求**（明确目标）
  2. **写规格**（定义标准）
  3. **写内容**（执行落地）
  4. **验收**（质量检查）

### 判断标准
- 简单 = 目标清晰、标准明确、工作量小
- 复杂 = 需要分析、需要批量处理、需要多步骤协作

### 实际案例
今天 Obsidian 整理就是典型复杂任务：
- 小呆呆：写标准文档（1253行规格）
- 超人强：批量执行（4分钟改384篇）
- 迷糊老师：验收质量（发现 Pulse 问题）
- 超人强返工：精修问题文件

## 主人指示 - 复杂任务必须记录需求规格验收

**时间**：15:49  
**内容**：主人要求调用子agent处理复杂任务时，必须记录需求、规格、验收

### 新增规则（已写入 RULES.md）
**触发条件**：判断需要调用子 agent 处理复杂任务时

**必须记录三样东西**：
1. **需求**（任务开始前）：主人要什么？目标是什么？做到什么程度算完？
2. **规格**（任务开始前）：怎么做？有什么约束？输出格式是什么？验收标准是什么？
3. **验收**（任务结束后）：实际产出是否符合规格？哪些通过？哪些返工？

**记录位置**：
- 短期：当天的 `memory/YYYY-MM-DD.md` 日记里
- 长期：如果这个任务的经验值得沉淀，提炼到 Obsidian 库或 MEMORY.md

**核心原则**：
- 简单任务简单搞，不用走四步
- 复杂任务必须走四步，而且需求和规格要存下来
- 如果需求和规格写得好，不用注入 Soul 也能搞定，就别为了幂塔而幂塔
- 方法论是工具，不是宗教
