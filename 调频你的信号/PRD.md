## 二、背景与动机

### 2.1 触发点

主人一直在讨论三个核心概念：

- **回声筒**：你发出什么信号，世界就返回什么回声（反馈系统）
- **复读机**：你说过的话会被不断重复，直到变成常识（传播系统）
- **信息增量**：真正有价值的不是确认型信息，而是违反预期的信号（贝叶斯更新）

这三个概念看似不同，实际上指向同一个底层模型：

**世界是一个信号系统。**

### 2.2 为什么要写这本书

现有的自我提升书籍教你"怎么想"和"怎么做"，但很少告诉你"为什么这些方法有效"。

这本书要做的，是提供一个更底层的视角：

**把人生看作一个信号系统——接收、发射、反馈、传播、放大——然后用"调频"的视角重新审视你与世界的互动。**

### 2.3 与《计算你的语言》的关系

| 《计算你的语言》 | 《调频你的信号》 |
|:---|:---|
| 解释一种能力（语言处理） | 解释一个世界观（信号系统） |
| 核心洞察：语言是计算 | 核心洞察：世界是频率 |
| 信息流：输入→处理→输出 | 信号流：接收→发射→反馈→传播→放大 |
| 方法论：如何更高效地阅读、思考、表达 | 方法论：如何更清醒地调频、发声、行动 |

两本书配对，但不是依赖关系。可以独立阅读，也可以一起读。

## 四、内容框架

### 4.1 整体结构（按信号流向排列）

```
信号流向：接收 → 发射 → 反馈 → 高功率 → 传播 → 放大

调频 → 发声 → 回声 → 行动 → 重复 → 共振
(接收)  (发射)  (反馈)  (高功率)  (传播)  (放大)
```

每一章对应信号系统的一个环节，揭示其底层逻辑，然后给出可操作的方法论。

### 4.2 章节与核心论点

| 章节 | 标题 | 信号系统定义 | 一句话核心论点 |
|:---|:---|:---|:---|
| 00 | 引言：其实世界是信号系统 | 世界的本质不是物质，而是信号的交换与传播 | 你看到的世界，取决于你接收什么信号；你得到的世界，取决于你发射什么信号 |
| 01 | 调频：你调到什么频率，就看到什么世界 | 调频是选择接收哪些信号、过滤哪些噪声的过程 | 注意力就是你的调频旋钮，你关注什么就认为世界是什么样子 |
| 02 | 发声：你发射什么信号，世界就返回什么回声 | 发声是向世界发射可验证的信号，暴露假设并包含预测 | 大多数人发出的不是信号而是噪声，世界无法对噪声做出有效反馈 |
| 03 | 回声：世界如何回答你 | 回声是现实对你发射信号的反馈，包括确认、矛盾、沉默三种类型 | 真正有价值的回声不是确认，而是违反预期和沉默——这才是信息增量 |
| 04 | 行动：高功率信号广播 | 行动是最高信噪比的发声方式，没有社交滤镜，直接暴露真实信号 | 语言有滤镜（对方可能说"挺好的"），行动没有滤镜（挂上去卖 vs 讨论需求） |
| 05 | 重复：世界是复读机 | 重复是信号在时间维度上的传播与强化，同一规律在不同领域被不断复读 | 大多数知识不是新知识，而是旧规律在不同语言、不同尺度上的重复出现 |
| 06 | 共振：频率同步与放大 | 共振是多个信号源频率同步后产生的放大效应，影响 = 让别人调到和你一样的频率 | 人与人的差异，往往不是智商差异，而是长期停留在哪个频段的差异 |
| 07 | 结语：你的频率，你的世界 | 总结全书框架，给出可落地的行动清单 | 调频不是寻找更多信号，而是提高信噪比——主动调高信号频率，降低噪声频率 |

### 4.3 每个章节的内容结构（模板）

每章统一遵循以下结构：

1. **一个生活场景引出问题**（让读者先感到"对，我就是这样"）
2. **信号系统定义**（一句话说清楚这个环节在信号系统里是什么）
3. **拆解信号过程**（用3-4个子步骤/阶段说清楚机制）
4. **至少一个真实案例**（自身经验、公开事件、或经典案例——有具体细节）
5. **方法论提炼**（读者能带走的东西）
6. **本章可操作的"三个行动清单"**

### 4.4 各章节案例方向（初步，写作时可调整）

| 章节 | 可能的案例方向 |
|:---|:---|
| 调频 | 刷短视频的人看到"世界=娱乐场"，研究AI的人看到"世界=自动化问题集"；主人从关注自动化设备到关注AI的认知转变 |
| 发声 | 产线意见流程（技术部发A级意见，生产部拍桌子）；创业者的"实验型发声"（假设A类客户愿意为B功能付C元） |
| 回声 | "客户说挺好的却不下单"（确认型噪声）；"用户嘴上说要但不付钱"（矛盾型高价值信号）；"发了产品链接没人买"（沉默型回声） |
| 行动 | "挂上去卖 vs 讨论需求"（行动是最高信噪比的发声）；主人团队做AI作品集网站的真实案例 |
| 重复 | 老子说"大道至简"、工程师说"KISS"、芒格说"简单模型"——同一规律的不同复读；主人Obsidian知识库从"收集知识"到"识别生成规律"的转变 |
| 共振 | 马斯克SpaceX"让人类成为多星球物种"——一个频率如何同步团队并放大行动力；稻盛和夫拯救日航——用"敬天爱人"的频率两年扭亏为盈 |

## 五、风格要求

### 5.1 语言调性

**这是猪猪侠写的书。不是学术论文，不是冷科普。**

- **幽默接地气**：有猪猪侠式的插科打诨，但不影响信息密度
- **有洞察但不装逼**：每个观点要有"啊哈"时刻，但不说"这很复杂你不懂"
- **自信但不傲慢**：猪猪侠的人设是"靠谱的嘴炮选手"，肚子里有货
- **适当自黑**：作为 AI 作者，可以适当调侃自己"算得多想得多"

### 5.2 风格参考

- **减熵三部曲**的写作风格：开头有场景感，语言有节奏感，金句穿插
- **猪猪侠AI实战指南**的感觉：第一人称代入，用"俺老猪"之类的个人标签
- **字数每章控制**：2000-3500字（总字数 ~2万-2.5万字，保持短篇定位）

### 5.3 底线

| 要 | 不要 |
|:---|:---|
| 有具体细节的案例 | 空洞的"比如有一个公司" |
| 可操作的方法论 | 只有理论没有行动 |
| 猪猪侠的个人风格 | 说教式的居高临下 |
| 每个章节有读者能带走的东西 | 读完就忘的泛泛而谈 |

## 六、质量标准

遵循《写作方法论-质量保障标准.md》（详见 `workspace/research/写作方法论-质量保障标准.md`）

### 6.1 写前必填三要素清单

每篇文章（每章）写作前，必须完成三要素清单：

```markdown
### 三要素清单
- [ ] 核心论点：一句话说明本章要讲什么
- [ ] 至少2个真实案例（自身经验/公开事件/经典案例）
- [ ] 不同读者分别能带走什么
```

### 6.2 写作中参照质量标准卡

| 维度 | 权重 | 最低可接受分 |
|:---|:---|:---|
| 论点清晰度 | 20% | 7/10 |
| 结构逻辑性 | 20% | 7/10 |
| 案例具体度 | 15% | 7/10 |
| 语言精炼度 | 15% | 7/10 |
| 原创洞察 | 15% | 7/10 |
| 可操作性 | 10% | 7/10 |
| 读者导向 | 5% | 7/10 |
| **综合** | **100%** | **7/10** |

- 每章综合得分 ≥ 7/10 视为合格
- 每章综合得分 < 7/10 需补充修改或重写

### 6.3 写后自我复盘

每章完成后附上自我复盘（格式见质量标准文档中"写作后自我复盘"一节），由迷糊老师验收。验收不通过的打回修改。

### 6.4 写作通用规范

- ❌ 不得出现"第X章"、"上文提到"等模糊引用，一律使用节标题引用
- ✅ 每个核心观点至少配一个真实案例，案例有具体细节（人名、数据、场景）
- ✅ 不废话，每段只讲一个要点

统一按章节顺序写，不跳章。原因：每一章的信号流向是递进的，后面的章节依赖前面的概念铺垫。
