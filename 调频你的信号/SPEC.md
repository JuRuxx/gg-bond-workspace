## 1. 各章节详细规格

### 1.0 写作顺序

严格按章节顺序书写：`00 → 01 → 02 → 03 → 04 → 05 → 06 → 07`

原因：信号流向递进——调频是接收，发声是发射，回声是反馈，行动是高功率验证，重复是传播，共振是放大。后一章的概念依赖前一章的铺垫。

### 1.1 统一内容结构模板

每章必须包含以下6个部分（按顺序）：

1. **引子**：一个生活场景，让读者产生"对，我就是这样"的共鸣（约200-300字）
2. **信号系统定义**：一句话说清楚该环节在信号系统里是什么（约50-100字）
3. **拆解信号机制**：用3-4个子步骤/阶段说清楚机制，每步有正文展开（约800-1200字）
4. **真实案例**：至少1个具体案例，有细节（人名、场景、数据）（约300-500字）
5. **方法论提炼**：读者能带走的框架或模型（约300-500字）
6. **行动清单**：3条马上就做得到的行动（约100-200字）

**总字数控制**：2000-3500字/章（引言和结语可放宽至1500-2500字）

**三要素清单**（写前必填，写入每章文件顶部的frontmatter注释）：

```markdown
<!--
### 三要素清单 ✅
- 核心论点：[一句话]
- 案例方向：[至少2个案例来源]
- 读者带走：[不同读者分别]
-->
```

### 1.2 章节 00：引言——其实世界是信号系统

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 世界的本质不是物质，而是信号的交换与传播。你看到的世界，取决于你接收什么信号；你得到的世界，取决于你发射什么信号。 |
| **字数范围** | 1500-2500字 |
| **引子** | 以一个日常观察开场：你有没有发现，当你开始关注某件事，全世界都在谈论它？当你买了一辆红色车，满大街都是红色车？这不是巧合，是你的调频决定了你看到的世界。 |
| **内容结构** | 1. 引子：从"关注什么就看见什么"这个普遍经验出发<br>2. 开篇隐喻——波粒二象性：光既是粒子也是波，取决于你怎么观测它。信息也是如此——很多人只看见信息的粒子（一句话、一个观点、一个产品），却看不见信息的波（传播、影响、情绪感染、文化扩散）。你用不同的方式观测，就看到不同的现实。这就是"调频"的物理学背书——但我们不讲量子力学，只用这个类比<br>3. 核心洞察展开：世界不是中立的舞台，是一台信号机器——它既在反射你的信号（回声筒），也在重复你的信号（复读机）<br>4. 全书的信号框架预览：调频=接收，发声=发射，回声=反馈，行动=高功率验证，重复=传播，共振=放大<br>5. 这本书的独特之处：不止讲"怎么想"和"怎么做"，更讲"为什么你的世界长这样"<br>6. 对读者的承诺：读完这本书，你再也回不到从前那个"看到什么就信什么"的状态 |
| **案例** | - 买车之后满大街同款（调频效应）<br>- 创业者看到的全是机会，打工人看到的全是问题（频率差异）<br>- 主人从关注自动化设备到关注AI后的认知转变 |
| **关键概念** | 信号系统 / 调频 / 回声筒 / 复读机 / 频率 |
| **约束** | ❌ 不深入解释贝叶斯/信息论的数学细节<br>❌ 不出现公式或代码<br>❌ 不加"第X章"引用，使用"在调频一章中"这种方式<br>❌ 波粒二象性只做类比，绝不说"量子力学证明了调频"<br>❌ 不用弦理论（太抽象，读者会觉得玄学）<br>❌ 不用吸引力法则（混合了真假两部分，会拉低全书可信度） |
| **语气** | 开场要帅，有猪猪侠的幽默和自信，同时铺垫全书的格调。要从一个日常现象切入，让读者觉得"这说的不就是我吗" |

### 1.3 章节 01：调频——你调到什么频率，就看到什么世界

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 注意力就是你的调频旋钮。同一个空间里有无数信号，你只能接收一部分。你关注什么，就认为世界是什么样子。调频不是被动接收，是主动选择。 |
| **字数范围** | 2500-3500字 |
| **引子** | 一个信息过载场景：刷了一天短视频，觉得全世界都在娱乐；看了一天创业文章，觉得全世界都在融资。同一个世界，完全不同的感受——为什么？ |
| **内容结构** | 1. 引子：信息过载时代的"频率锁定"现象——为什么你和朋友看到的世界不一样<br>2. 信号系统定义：调频 = 选择接收哪些信号、过滤哪些噪声的过程<br>3. 调频的三层机制：<br>   - 频率选择：注意力决定你能接收什么（刷短视频 vs 研究AI）<br>   - 信号过滤：你的已有认知会自动过滤不符合预期的信号（确认偏误）<br>   - 频率锁定：长期停留在某个频段，会形成认知惯性（信息茧房）<br>4. 案例：主人从自动化设备行业转向AI领域后的认知巨变——同样的世界，完全不同的信号<br>5. 方法论：调频的"频率检查清单"——每周问自己三个问题：我这周在什么频率上？这个频率给了我什么信号？我需不需要换台？<br>6. 行动清单 |
| **案例方向** | - 主人从关注自动化设备到关注AI的认知转变——同一个世界，完全不同的信号<br>- 刷短视频的人看到"世界=娱乐场"，研究投资的人看到"世界=机会集"的对比案例 |
| **关键概念** | 频率选择 / 信号过滤 / 频率锁定 / 信息茧房 / 确认偏误 / 波粒二象性（隐喻） |
| **约束** | ❌ 不用"第X章"引用<br>❌ 不写心理学学术论文式的"注意力研究综述"<br>❌ 不把"信息茧房"写成政治话题 |

### 1.4 章节 02：发声——你发射什么信号，世界就返回什么回声

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 大多数人发出的不是信号而是噪声。发声不是随便喊，是向世界发射可验证的信号——暴露假设、包含预测、能被反驳。你发射什么频率，世界就返回什么回声。 |
| **字数范围** | 2500-3500字 |
| **引子** | 一个"说了但没用"的场景：在团队会议上提了意见，没人回应；在朋友圈发了动态，没人点赞；跟老板说了想法，石沉大海。你以为世界不给你反馈，其实是你发出的根本不是信号——是噪声。 |
| **内容结构** | 1. 引子："说了但没用"的困境——为什么你的声音没有回声<br>2. 信号系统定义：发声 = 向世界发射可验证的信号，暴露假设并包含预测<br>3. 好发声的四个原则：<br>   - 发声就是提出可验证假设（"我假设A类客户愿意为B功能付C元"）<br>   - 主动为反面信息留通道（"这里我最不确定的是……"）<br>   - 带预测的表达（"我预测注册率达到10%"）<br>   - 多用行动发声，少用语言发声（行动是最高信噪比的发声）<br>4. 案例：产线意见流程——技术部发A级意见，生产部拍桌子。这不是"意见没被采纳"，是"发声方式有问题"<br>5. 方法论：发声的"信号设计模板"——发声前问自己：我在验证什么假设？我预测什么结果？什么证据能推翻我？<br>6. 行动清单 |
| **案例方向** | - 产线意见流程案例（技术部 vs 生产部）——发声方式决定了回声质量<br>- 创业者从"我要创业"到"我假设A类客户愿意为B功能付C元"的转变 |
| **关键概念** | 可验证假设 / 信号 vs 噪声 / 预测 / 反面信息通道 |
| **约束** | ❌ 不写"沟通技巧"类的建议（那太鸡汤）<br>❌ 不写"第X章"引用<br>❌ 案例要有真实细节，不能"比如说有一家公司" |

### 1.5 章节 03：回声——世界如何回答你

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 真正有价值的回声不是确认，而是违反预期和沉默。确认型回声信息量≈0，矛盾型回声是高价值信号，沉默型回声可能意味着回路断了。学会从回声中提取信息增量，是调频的核心能力。 |
| **字数范围** | 2500-3500字 |
| **引子** | 一个"听回声"的场景：你发了一个产品链接，朋友说"挺好的"但没下单。你以为收到了正面反馈，其实收到的是社交噪声。真正有价值的回声，往往不是你想听到的那个。 |
| **内容结构** | 1. 引子：回声的"真假难辨"——为什么"挺好的"可能是最没用的回声<br>2. 信号系统定义：回声 = 现实对你发射信号的反馈，包括确认、矛盾、沉默三种类型<br>3. 三类回声的信号价值：<br>   - 确认型：世界在重复你的信号，信息量≈0（开心一下，别当决策依据）<br>   - 矛盾型：世界给出了反例，信息量🔥高（认真分析，修正先验）<br>   - 沉默型：你喊了，世界没反应，信息量⚠️极重要（检查回路，可能信号错了）<br>4. 三问框架：每次收到回声，只问三个问题——什么符合预期？什么违反预期？什么没有发生？<br>5. 案例：创业者做用户访谈——用户说"挺好的"（确认型噪声）vs 用户说"这个功能我不需要"（矛盾型高价值信号）vs 没人回复问卷（沉默型回声）<br>6. 方法论：回声的"三问框架"——每次收到回声，问自己三个问题，完成贝叶斯更新<br>7. 行动清单 |
| **案例方向** | - 用户访谈案例：用户说"挺好的"但不下单 vs 用户直接说"不需要" vs 没人回复问卷<br>- 主人团队做AI作品集网站时的真实反馈案例 |
| **关键概念** | 三类回声 / 信息增量 / 贝叶斯更新 / 三问框架 |
| **约束** | ❌ 不把贝叶斯写成数学课（不用公式，只用直觉解释）<br>❌ 不写"第X章"引用<br>❌ 不深入讨论"信息论"的学术定义 |

### 1.6 章节 04：行动——高功率信号广播

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 行动是最高信噪比的发声方式。语言有社交滤镜（对方可能说"挺好的"心里想"不行"），行动没有滤镜（挂上去卖 vs 讨论需求）。行动是用最高功率向世界发射信号，逼世界给你最真实的回声。 |
| **字数范围** | 2500-3500字 |
| **引子** | 一个"说 vs 做"的对比场景：团队讨论了三个月"用户到底需不需要这个功能"，最后有人直接把产品挂上去卖，一周就知道答案了。三个月的讨论不如一周的行动——为什么？ |
| **内容结构** | 1. 引子：为什么"讨论需求"永远不如"挂上去卖"有效<br>2. 信号系统定义：行动 = 用最高功率发射信号，绕过社交滤镜，直接获取真实回声<br>3. 行动的信号优势：<br>   - 语言有滤镜：社交礼仪让对方说"挺好的"<br>   - 行动没有滤镜：挂上去卖，市场直接告诉你答案<br>   - 行动的功率高：同样时间，行动获取的信息量远大于讨论<br>4. 行动的设计原则：<br>   - 最小可行行动（MVA）：用最低成本验证假设<br>   - 快速回声：选择能最快获得回声的行动方式<br>   - 可观测结果：行动的结果必须是可以量化观测的<br>5. 案例：主人团队从"讨论用户需求"到"直接做产品挂上去"的转变——一周获取的真实回声比三个月讨论都多<br>6. 方法论：行动的"MVA模板"——最小可行行动设计：假设是什么？最低成本怎么验证？多久能收到回声？<br>7. 行动清单 |
| **案例方向** | - 主人团队做AI作品集网站的真实案例：从讨论需求到直接上线，一周获取真实反馈<br>- "精益创业"中的MVP概念，用信号系统的语言重新解读 |
| **关键概念** | 高功率信号 / 社交滤镜 / 最小可行行动（MVA） / 快速回声 |
| **约束** | ❌ 不写"行动力"类的鸡汤（"just do it"那种）<br>❌ 不写"第X章"引用<br>❌ 不深入讨论精益创业的技术细节 |

### 1.7 章节 05：重复——世界是复读机

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 大多数知识不是新知识，而是旧规律在不同领域、不同语言、不同尺度上的重复出现。老子说"大道至简"、工程师说"KISS"、芒格说"简单模型"——可能是同一个规律的复读。识别复读，是从"收集知识"到"识别生成规律"的认知升级。 |
| **字数范围** | 2500-3500字 |
| **引子** | 一个"似曾相识"的场景：你读了一本新书，觉得"这些观点我都见过"。或者听了一场演讲，觉得"这不就是那个东西换了个说法吗"。你不是变老了，是你的认知系统开始自动做"去重压缩"了。 |
| **内容结构** | 1. 引子："怎么全世界都在说同一件事"——知识积累到一定程度后的"复读机感知"<br>2. 信号系统定义：重复 = 信号在时间维度上的传播与强化，同一规律在不同领域被不断复读<br>3. 复读机的三层机制：<br>   - 表层复读：同一句话被不同人说（"大道至简"、"KISS"、"奥卡姆剃刀"）<br>   - 跨域复读：同一规律在不同领域出现（"输入决定输出"在AI/农业/教育/投资中都有）<br>   - 自我复读：你说过的话，会被你自己的行动不断强化（自我实现预言）<br>4. 识别复读的方法：<br>   - 遇到新观点，先问："这是新规律吗？"<br>   - 如果不是，再问："这是哪条旧规律的复读？"<br>   - 然后链接到你的Core，而不是存一张新卡片<br>5. 哲学根基——"识智何分，波水一个"：弥勒对虚云的开示。波 = 变化、现象、信号（老子说"大道至简"、芒格说"简单模型"、工程师说"KISS"——这些都是波）。水 = 底层规律、载体、本质（"优先选择复杂度最低的有效解释"——这是水）。金句方向："不要被波吸引，要寻找水。"<br>6. 案例：主人的Obsidian知识库从"收集知识"到"识别生成规律"的转变——知识库增长速度变慢，但Core密度越来越高<br>7. 方法论：复读机的"去重压缩清单"——遇到新观点时的三个问题<br>8. 行动清单 |
| **案例方向** | - "大道至简"/"KISS"/"奥卡姆剃刀"/"芒格简单模型"——同一规律的四种复读<br>- 主人的Obsidian知识库从"收集知识"到"识别Core"的真实转变 |
| **关键概念** | 复读机 / 去重压缩 / Core提炼 / 跨域同构 / 自我实现预言 / 波水一个（隐喻） |
| **约束** | ❌ 不写"模因学"的学术讨论（可以提但别展开）<br>❌ 不写"第X章"引用<br>❌ 不把"自我实现预言"写成玄学<br>❌ "波水一个"只做隐喻和哲学呼应，不做佛学解读<br>❌ 不引用佛经原文（用白话翻译或直接用自己的语言表达） |

### 1.8 章节 06：共振——频率同步与放大

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 当多个信号源频率同步时，会产生共振放大效应。人与人的差异，往往不是智商差异，而是长期停留在哪个频段的差异。影响力的本质，是让别人调到和你一样的频率。 |
| **字数范围** | 2500-3500字 |
| **引子** | 一个"共振时刻"的场景：加入一个高质量的社群/团队后，发现成长速度是之前的十倍。不是因为里面的人更聪明，而是因为大家的频率同步了——信号互相放大，而不是互相抵消。 |
| **内容结构** | 1. 引子：为什么"和谁在一起"比"做什么"更重要<br>2. 信号系统定义：共振 = 多个信号源频率同步后产生的放大效应<br>3. 共振的三层机制：<br>   - 频率同步：当你的频率和环境对上了，信号互相放大<br>   - 频率冲突：当你的频率和环境不对，信号互相抵消（内耗）<br>   - 频率引领：影响力的本质是让别人调到和你一样的频率<br>4. 共振的应用场景：<br>   - 个人层面：选择和你频率同步的社群/圈子<br>   - 团队层面：建立团队的共同频率（愿景/文化/标准）<br>   - 影响层面：不是说服别人，是让别人调到和你一样的频率<br>5. 案例：马斯克SpaceX——"让人类成为多星球物种"这个频率如何同步整个团队，在Falcon 1连续三次失败后团队依然不散<br>6. 方法论：共振的"频率对齐清单"——个人/团队/影响三个层面的频率检查<br>7. 行动清单 |

## 2. 写作通用规范

### 2.1 文档格式

**文件命名规则**：
```
content/00-intro.md        ← 引言
content/01-tuning.md       ← 第1章：调频
content/02-broadcasting.md ← 第2章：发声
content/03-echo.md         ← 第3章：回声
content/04-action.md       ← 第4章：行动
content/05-repetition.md   ← 第5章：重复
content/06-resonance.md    ← 第6章：共振
content/07-epilogue.md     ← 第7章 结语
```

**每章文件开头**：写作前写入三要素清单注释

```markdown
<!--
### 三要素清单 ✅
- 核心论点：[一句话]
- 案例方向：[至少2个案例来源]
- 读者带走：[不同读者分别]
-->
```

**Markdown规范**：
- 标题层级：`#` 章节标题 → `##` 小节标题 → `###` 子节标题。最多三级标题，避免过深嵌套
- 加粗 `**` 用于强调重点概念
- 引用 `>` 用于核心金句或信号定义
- 列表用于方法论提炼和行动清单
- 表格用于对比或总结（不超过三列）
- 代码块 ` ``` ` 用于框架模板（纯文本，不是编程语言）

### 2.2 章节内引用规范

| 规则 | 示例 |
|:---|:---|
| ❌ 不得出现"第X章"引用 | 错误："如第3章所述" |
| ✅ 使用节标题或上下文引用 | 正确："发声一章中提到的可验证假设概念" |
| ✅ 跨章引用清晰但不粗暴 | 正确："这个思路延续了我们在调频中讨论的频率锁定" |
| ✅ 前后呼应 | 在结语中回扣各章要点时，用"调频一章中提到的"风格 |

### 2.3 案例标注规则

每个案例必须有：

- **来源标示**：[案例：马斯克SpaceX 2008年第四次发射]、[案例：主人团队产线意见流程]
- **具体细节**：人名/场景/数据/时间，不能"比如说有一家公司"
- **角色确认**：案例来源（自身经验、公开事件、经典案例）

**案例格式建议**：

```markdown
> **【案例：产线意见流程——2026年】**
>
> 技术部主管写了一份A级意见书：3号点胶机Z轴导轨润滑周期不符合标准，需整改。
> 生产部主管看完拍桌子说："你们技术部管太宽了。"
> —— 这不是"意见没被采纳"，是"发声频率没对上" ...
```

### 2.4 术语使用规范

| 可以使用 | 不可以使用（或需谨慎） |
|:---|:---|
| 信号 / 频率 / 调频 | 电磁波物理公式 |
| 回声 / 反馈 | 控制论的传递函数 |
| 噪声 / 信噪比 | 香农公式 / 信息熵 |
| 贝叶斯更新 | 后验概率的数学推导 |
| 先验 / 后验 | 概率密度函数 |
| 复读机 / 模因 | 模因学的学术定义 |
| 共振 / 频率同步 | 物理学共振公式 |
| 确认偏误 / 信息茧房 | 认知心理学的实验细节 |
| 自我实现预言 | 玄学/吸引力法则 |

**核心原则**：用信号术语做**类比和比喻**，不做**技术实现描述**。目标是让不懂信号处理的读者也能理解这些概念的生活化版本。

### 2.5 语言风格规范

**整体调性**（继承PRD风格要求）：

- 猪猪侠式的幽默：有插科打诨但不影响信息密度
- 有洞察但不装逼：每个观点要有"啊哈"时刻
- 自信但不傲慢：肚子里有货，但不说"这很复杂你不懂"
- 适当自黑：作为AI作者，可以调侃自己"每天都在处理信号"

**具体规则**：
- 每章开头用「引子」建立场景感——读者看完前三段应该感觉"对，我就是这样"
- 每段只讲一个要点
- 金句穿插（每章2-3句值得单独摘出来的表达）
- 避免过度修饰，干货优先
- 猪猪侠个人标签：适当使用"俺老猪"、"嘿嘿"、"聪明的智商又占领高地了"等，但每章不超过2次
- 不使用"毋庸置疑"、"毫无疑问"、"显而易见"等傲慢用词

### 2.6 引用规范

- **书籍引用**：《书名》（作者，年份），如《影响力》（罗伯特·西奥迪尼，1984）
- **文章引用**：《文章名》（来源/期刊，年份）
- **影视/TED引用**：「演讲标题」（演讲人，年份）
- **不要求正式学术引用格式**（非学术书籍），但需要让读者能找到引用源

## 3. 验收执行细则

### 3.1 迷糊老师验收流程

每章按以下流程验收：

```
作者完稿
    ↓
作者提交写作质量自评（详见 3.3）
    ↓
迷糊老师对照质量标准卡评分
    ↓
┌─────────────────────────────────────────┐
│ 综合得分 ≥ 7/10 → 通过                   │
│ 综合得分 5-6/10 → 打回补充修改            │
│ 综合得分 < 5/10  → 完全重写               │
└─────────────────────────────────────────┘
    ↓
验收通过 → 写入 REVIEW-<文件名>.md
    ↓
猪猪侠（主人）终审确认
```

### 3.2 质量标准卡评分细则

| 维度 | 权重 | 评分标准（1-10分） | 最低可接受 |
|:---|:---:|:---|:---:|
| 论点清晰度 | 20% | 一句话能说清本章要讲什么；读者读完能复述核心论点 | ≥7 |
| 结构逻辑性 | 20% | 引子→定义→拆解→案例→方法论→行动，递进清晰，不跳跃不重复 | ≥7 |
| 案例具体度 | 15% | 每个核心观点有真实案例；案例有具体细节（人名、场景、数据） | ≥7 |
| 语言精炼度 | 15% | 不废话，每段一个要点；猪猪侠风格有但不影响信息密度 | ≥7 |
| 原创洞察 | 15% | 有没有从"信号系统"视角带来的新鲜见解；不是常识的另一种说法 | ≥7 |
| 可操作性 | 10% | 读完后读者能带走什么；方法论/行动清单是否真实可用 | ≥7 |
| 读者导向 | 5% | 是否考虑了不同读者的需求；入题是否够快 | ≥7 |
| **综合** | **100%** | 加权平均分 | **≥7** |

**评分说明**：
- 9-10分：超出预期，可做典范
- 7-8分：良好，达到交付标准
- 5-6分：合格但需补充修改
- <5分：不合格，需重写

### 3.5 返工标准

| 情况 | 处理方式 |
|:---|:---|
| 综合评分 ≥ 7/10，但某个维度 < 5/10 | 仅修改该维度对应的内容，不需要整章重写 |
| 综合评分 5-6/10 | 根据修改意见逐条修改后重新提交验收 |

2. **风格一致性**：各章的猪猪侠风格是否一致
3. **术语一致性**：核心术语（信号、频率、调频、回声、复读、共振）在全书中是否使用一致
4. **冗余检查**：是否有跨章重复的内容
5. **行动清单完整性**：全书的行动清单加起来是否有可执行性

整体验收不通过则逐章标记需要调整的部分，进入修改轮。

### 5.4 写作时需规避的常见坑

| 坑 | 规避方式 |
|:---|:---|
| 章节内出现"第X章""上文提到" | 统一用节标题引用，如"调频一章中提到的" |
| 案例无具体细节 | 写前确认案例来源，记录人名/场景/数据 |
| 术语前后不一致 | 写完后做一次术语一致性检查 |
| 猪猪侠风格过度（每章出现太多次口头禅） | 每章口头禅不超过2次 |
| 字数超标或不足 | 写完后检查字数，控制在范围内 |
| 跨章内容重复 | 每一章聚焦自己的信号环节，不重复讲其他章节的概念 |
| 过于技术化 | 写作中随时问自己："一个不懂信号处理的人能看懂吗？" |
| 把"回声筒"写成贝叶斯方法论 | 回声筒是信号反馈的类比，贝叶斯只是工具之一 |
| 把"复读机"写成模因学 | 复读机是信号传播的类比，不是学术概念 |
| 把"共振"写成吸引力法则 | 共振是频率同步的信号系统概念，不是玄学 |
