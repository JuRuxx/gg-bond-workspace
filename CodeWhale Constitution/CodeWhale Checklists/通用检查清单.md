# 通用验收检查清单

> CodeWhale Constitution 附属文档 · 独立版本迭代 · 当前版本 2.1
> 每次代码合并前逐项检查，缺一项不得合并。

---

## 目录标准

- [ ] 目录名称使用 `kebab-case`
- [ ] 目录结构符合项目类型的标准骨架
- [ ] 目录名称清晰反映职责

## 文件标准

- [ ] Python 文件使用 `snake_case`
- [ ] JavaScript / TypeScript 文件使用 `kebab-case`
- [ ] React 组件文件使用 `PascalCase` + `.tsx`
- [ ] 每个文件只做一件事
- [ ] 不存在职责模糊的通用文件

## 函数标准

- [ ] 没有使用万能动词命名（`handle` / `process` / `execute` / `run`）
- [ ] 所有函数名自解释
- [ ] 函数不超过 50 行
- [ ] 布尔参数已拆分为独立函数

## 格式规范

- [ ] 没有使用 Tab 缩进
- [ ] 没有超过 100 字符的行
- [ ] 文件末尾有空行
- [ ] 没有行尾空格
- [ ] 布尔变量使用标准前缀
- [ ] 导入按规范分组排序

## 注释与异常

- [ ] 所有公开函数有文档注释
- [ ] 注释说明"为什么"而非"是什么"
- [ ] TODO 标注了责任人
- [ ] 没有空的异常捕获块

## 代码红线

- [ ] 没有硬编码密钥或敏感信息
- [ ] 没有硬编码魔数
- [ ] 没有使用打印语句作为生产日志
- [ ] 没有超过 100 行的函数
- [ ] 没有超过 500 行的文件
- [ ] 没有直接拼接 SQL
- [ ] 没有使用 `eval()` / `exec()` 执行用户输入

## README

- [ ] 有且仅有 6 个章节（项目名 > 一句话描述 > 这是什么 > 快速开始 > 目录结构 > 开发 > LICENSE）
- [ ] 一句话描述不超过 20 字
- [ ] 

- [ ] 提交信息格式正确
- [ ] 类型在允许列表中
- [ ] 主题不超过 50 字
- [ ] 分支命名符合规范
