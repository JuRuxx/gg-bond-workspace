# README 标准

> CodeWhale Constitution 附属文档 · 独立版本迭代 · 当前版本 2.0

---

## 核心原则

README 是项目的入口上下文。它服务于两类读者：

- **人类**——快速判断"这是什么项目，我要不要深入"
- **AI / Agent**——从零状态进入项目时建立初始化上下文

因此 README 的目标是在 **60 秒内**帮助人类或 Agent 回答以下四个问题：

1. 这是什么？
2. 如何运行？
3. 代码在哪里？
4. 下一步该做什么？

回答不了的 README 不合格。

---

## 标准结构

```
# 项目名
> 一句话描述

## 这是什么

## 快速开始

## 目录结构

## 开发

## LICENSE

## [可选] 配置
## [可选] 架构
## [可选] FAQ
```

### 必需章节（6 个）

不能少。少了任何一个，Agent 或人类无法回答四个核心问题。

### 可选章节

项目需要才加。例如需要环境变量配置的项目加 `## 配置`，体系复杂的项目加 `## 架构`。

---

### # 项目名

项目全称。不用加副标题，不用加版本号。

### > 一句话描述

不超过 20 个字的定位。不是功能介绍，是一句话让人知道这个项目解决什么问题。

```
# bilibili-subtitle
> B 站字幕批量下载与清洗工具

# codewhale-llm
> LLM 驱动的代码审查 Agent
```

### ## 这是什么

3~5 句。说清楚：

- 这个项目解决什么问题
- 核心技术栈
- 适用场景（选填）

不要写长篇概述，Agent 和人类只需要快速确认"这是我要找的东西吗"。

```
## 这是什么

bilibili-subtitle 是一个从 B 站视频批量下载并清洗字幕的工具。
基于 Python + requests，输出 SRT 格式字幕文件。
适用于字幕翻译、语料收集、内容检索等场景。
```

### ## 快速开始

最小可运行示例。一行命令启动优先。

```
## 快速开始

```bash
pip install bilibili-subtitle
bili-sub --url https://www.bilibili.com/video/BV1xx411c7mD
```

或者给出三步操作：安装 → 配置 → 运行。

### ## Agent Summary（推荐）

为 Agent 提供结构化摘要。人类可以跳过。

```
## Agent Summary

Language: Python 3.12
Framework: FastAPI
Entry: src/main.py
Test: pytest
Build: uv run
Purpose: 批量下载并清洗 B 站字幕
```

Agent 扫描此章节可直接获取上下文关键字段，无需全文推理。

### ## 目录结构

呈现项目的物理骨架。只展示核心目录和文件，不要展开所有细节。

```
## 目录结构

project/
├── src/
│   ├── core/        # 核心业务逻辑
│   ├── tools/       # 工具函数
│   └── config/      # 配置
├── tests/
├── docs/
└── README.md
```

### ## 开发

告诉开发者下一步做什么。

```
## 开发

# 安装依赖
poetry install

# 运行测试
pytest

# 提交代码前运行
pre-commit run --all-files
```

### ## LICENSE

显式标明许可证。不标 LICENSE 等于告诉别人"别用这个项目"。

```
## LICENSE

MIT
```

---

## AI / Agent 优化建议

Agent 扫描 README 时优先提取以下信息：

1. 语言和框架（从"这是什么"和目录结构推断）
2. 启动方式（"快速开始"）
3. 目录骨架（"目录结构"）
4. 测试命令（"开发"）

如果四个信息都能从 README 直接提取，Agent 可以零人工干预进入开发状态。

---

## 禁止

- ❌ 缺少任一必需章节
- ❌ 章节约名：不用 "一种高效的……"、"关于本项目的……" 等绕弯表述
- ❌ 在 README 里放发行说明或更新日志（另开 CHANGELOG.md）
- ❌ 功能说明书式的大段文字（3~5 句说清楚，说不清楚的说明项目定位不够明确）
- ❌ 未标 LICENSE
