# 目录与文件标准

> CodeWhale Constitution 附属文档 · 独立版本迭代 · 当前版本 2.0

## 目录命名

所有目录统一使用 `kebab-case`（小写字母与连字符）。目录不参与语言导入解析，可跨语言统一。

项目骨架按项目类型统一。

### Python 项目

```
project/
├── src/
│   ├── __init__.py
│   ├── main.py          # 入口
│   ├── core/            # 核心业务逻辑
│   ├── tools/           # 工具函数
│   ├── models/          # 数据模型
│   └── config/          # 配置
├── tests/
├── docs/
├── pyproject.toml
└── README.md
```

### JavaScript / TypeScript 项目

```
project/
├── src/
│   ├── index.ts         # 入口
│   ├── core/
│   ├── tools/
│   ├── types/
│   └── config/
├── tests/
├── docs/
├── package.json
└── README.md
```

### 嵌入式 / 硬件项目

```
project/
├── src/
│   ├── main.py
│   ├── drivers/         # 硬件驱动
│   ├── sensors/         # 传感器逻辑
│   ├── navigation/      # 导航模块
│   ├── tasks/           # 任务调度
│   └── config/
├── tests/
├── docs/
└── README.md
```

目录结构是为"看懂项目怎么跑"服务的。如果业务特殊，可以在核心骨架基础上调整。

---

| 正确 | 错误 |
| --- | --- |
| `user-service/` | `user_service/` |
| `memory-store/` | `memoryStore/` |
| `bilibili-subtitle/` | `bilibili_subtitle/` |

## 文件命名

文件命名遵循语言生态，不强行跨语言统一。

### Python 文件

使用 `snake_case`（PEP 8 社区惯例）。

```
# 正确
memory_index.py
user_service.py
temperature_sensor.py

# 错误
memory-index.py         # kebab-case 在 Python 中非法
memoryIndex.py          # Python 不使用驼峰
```

### JavaScript / TypeScript 文件

使用 `kebab-case`（社区主流实践）。

```
# 正确
user-service.ts
memory-index.ts
api-client.ts

# 错误
UserService.ts          # 文件名不使用驼峰
memory_index.ts         # JS/TS 不使用 snake_case
```

### React 组件文件

使用 `PascalCase` + `.tsx`（框架惯例）。

```
# 正确
UserOrderList.tsx
DashboardPanel.tsx

# 错误
user-order-list.tsx     # 组件文件不应使用 kebab-case
userOrderList.tsx       # 组件文件不应使用 camelCase
```

### 测试文件

各语言遵循测试框架惯例。

| 语言 | 规则 | 示例 |
| --- | --- | --- |
| Python | `test_<模块>.py` | `test_user_service.py` |
| JavaScript / TypeScript | `<模块>.test.ts` 或 `<模块>.spec.ts` | `user-service.test.ts` |
