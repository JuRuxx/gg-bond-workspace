# 工作流标准

> CodeWhale Constitution 附属文档 · 独立版本迭代 · 当前版本 1.0

---

## 开发流程

```
需求分析 → 规格定义 → 实现编码 → 代码审查 → 测试验证 → 发布上线
```

各阶段交付物：

| 阶段 | 交付物 | 标准 |
| --- | --- | --- |
| 需求分析 | 需求文档 | 明确目标与范围 |
| 规格定义 | 接口设计 / 技术方案 | 明确实现路径 |
| 实现编码 | 代码 | 符合 CodeWhale Standards |
| 代码审查 | 审查记录 | 验收清单全部通过 |
| 测试验证 | 测试通过报告 | 单元测试与集成测试覆盖 |
| 发布上线 | 上线记录 | 含回滚方案 |

---

## Git 分支规范

```
main  ←  develop  ←  feature/<任务标识>
```

- `main`：生产分支，仅接收经过审查与测试的合并
- `develop`：开发基线分支
- `feature/<任务标识>`：功能分支，每个独立任务一条分支

---

## 提交信息格式

```
<type>(<scope>): <subject>
```

允许的类型：

| 类型 | 含义 |
| --- | --- |
| `feat` | 新功能 |
| `fix` | 修复缺陷 |
| `docs` | 文档变更 |
| `style` | 代码格式 |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试 |
| `chore` | 杂项 |
| `ci` | CI/CD 配置 |

```
# 正确示例
feat(订单模块): 新增批量导出功能
fix(用户中心): 修复手机号格式校验失败
refactor(支付模块): 抽离支付策略为策略模式

# 错误示例
update code           # 未说明变更内容
fix bug               # 描述过于模糊
```
