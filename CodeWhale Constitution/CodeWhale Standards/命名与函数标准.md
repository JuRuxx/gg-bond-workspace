## 函数命名

### 好命名与坏命名

| 正确的命名 | 错误的命名 | 说明 |
| --- | --- | --- |
| `createMemory()` | `handleMemory()` | "处理"含义模糊 |
| `retrieveUserByEmail()` | `getUser()` | 检索条件不明确 |
| `deleteOrder()` | `processOrder()` | 操作语义不明确 |
| `calculateDiscount()` | `run()` | 无法推断功能 |
| `sendEmail()` | `execute()` | 无法推断功能 |

### 函数职责

- 一个函数只做一件事
- 函数不应超过 50 行
- 布尔参数应优先拆分为两个独立函数

```python
# 不推荐
def send_email(recipient: str, body: str, is_batch: bool):
    if is_batch:
        ...
    else:
        ...

# 推荐
def send_email(recipient: str, body: str):
    """发送单封邮件。"""

def send_batch_email(recipients: list[str], body: str):
    """批量发送邮件。"""
```

### 参数命名

参数名称应当自解释，避免歧义缩写和单字母命名。

```python
# 不推荐
def process(a, b):
    ...

# 推荐
def create_memory_index(documents: list[str], options: IndexOptions) -> IndexResult:
    ...
```

## 语言级命名规则

| 场景 | Python | JavaScript / TypeScript |
| --- | --- | --- |
| 模块 / 文件 | `snake_case` | `kebab-case` |
| 类 | `PascalCase` | `PascalCase` |
| 函数 | `snake_case` | `camelCase` |
| 变量 | `snake_case` | `camelCase` |
| 常量 | `UPPER_SNAKE_CASE` | `UPPER_SNAKE_CASE` |
| 接口 | — | `PascalCase` |
| React 组件 | — | `PascalCase` + `.tsx` |

### 布尔变量

使用以下前缀：

- `is_` / `is`
- `has_` / `has`
- `should_` / `should`
- `can_` / `can`

禁止否定式命名：

```python
# 不推荐
is_not_disabled = True
no_errors = True

# 推荐
is_enabled = True
has_error = False
```
