## 1. 写作顺序

严格按章节顺序：`01 → 02 → 03 → 04 → 05 → 06 → 07 → 99`

原因：每一章的概念依赖前一章的铺垫：
- 01 先回答"为什么是微积分"（立框架）
- 02 微分（最基础的运算）
- 03 积分（微分的逆/补充）
- 04 迭代+收敛（在前两者基础上的时间维度）
- 05 泛函优化（把所有活动纳入统一框架）
- 06 演示（把前五章概念实操一遍）
- 07 结语（收束）

**注意**：与母书不同，这里各章是**数学概念的递进**，不是信息流的递进。所以第二章微分必须在前。

### 2.3 章节02：语义微分——文字世界的梯度

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 语义梯度是语言微积分最基本的运算——一段文字的语义梯度决定了它在"哪个方向、以多大力道"推拉读者的认知。 |
| **字数** | 3000-4000字 |
| **引子** | 对比两句话："今天天气不错" vs "但我不看好这个方案"——前者的梯度"零或弱"，后者的梯度"强且方向明确"。给出"每句话都有语义梯度"的直觉。 |
| **概念展开** | ① **梯度方向**：文字在把语义空间往哪个方向推——"但是"、"然而"是梯度反转指示器，"更重要的是"是梯度强化。每段话读完，读者的认知被朝某个方向拉了一点。<br>② **梯度大小**：文字推的力度——"不得不承认，你让我开始动摇了"的梯度 > "这个你可以考虑一下"的梯度。强烈语言、具体数据、情感锚点→大梯度。<br>③ **偏导数/维度敏感度**：一篇文章不只在单一维度产生梯度——它在"事实维度"的偏导可能很大，在"情感维度"的偏导可能很小。好的表达知道在哪些维度拉、哪些维度放。<br>④ **多层梯度**：一篇文章的梯度≠句子梯度的简单和——段落有段落梯度，篇章有篇章梯度。梯度是分层级的。 |
| **母书链接** | 链接母书"阅读：世界模型重建与更新"——阅读就是把作者的语义梯度解压缩到读者的世界模型中。梯度是"解压缩"的运算单元。 |
| **案例** | 拆解一段真实的文字（可选：某篇公众号爆文/乔布斯发布会一段话/罗振宇演讲一段），逐句标注梯度方向和梯度大小。展示"整段的梯度 ≠ 句子梯度之和"。 |
| **行动/启示** | ① 下次读完一段话问自己"这段话在把我往哪个认知方向推？"<br>② 写东西前先确定"我想让人往哪个方向走"（目标梯度）<br>③ 识别"高梯度文字"的写作技巧——用数据、用对比、用锚定 |
| **约束** | ✅ 本章正式引入∇符号（作为概念符号）<br>❌ 不写数学的梯度公式（∇f(x)不要展开） |

### 2.4 章节03：语义积分——从碎片到整体

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 语义积分把分散的文字输入累积成一个完整的认知结构。积分不是把碎片拼在一起——是把碎片沿着一条"语义路径"积累到读者的世界模型中。 |
| **字数** | 3000-4000字 |
| **引子** | "你读十本关于同一个话题的书——你的认知变化不是加法，是积分。"引出"读了很多但整合不了"的困境 = 积分路径没设计好。 |
| **概念展开** | ① **路径积分**：阅读不是"点积"（一篇一篇独立读）而是"路径积分"（顺序影响结果）。读A→读B 和 读B→读A 的语义积分不一样。先用什么建立框架（先验），再用什么填充细节（更新），路径决定认知。<br>② **积分步长**：每篇文字提供的信息量不是平均的——有的文章步长大（颠覆认知），有的步长小（提供细节）。步长大≠积分质量高——步长太大可能"超调"（读一篇文章就完全倒向某方）。步长太小可能"不收敛"（读了很多但认知没动）。<br>③ **积分边界**：研究一个领域的时候，积分区间（该读哪些文章）和积分密度（每个方向读多少）是核心问题。不是"要把所有东西读完"而是"积分到收敛就停"。<br>④ **多路径积分**：同一件事从不同角度读（技术文章+案例报道+专家评论）相当于在不同的"坐标系"中积分——多个路径积分的平均值比单路径积分更接近真理。 |
| **母书链接** | 链接母书"阅读"——阅读六层递进中的"压缩解压缩"和"可迁移表示"可以用积分的语言重新描述：解压缩是逆向积分，迁移是路径再参数化。 |
| **案例** | "学习一个全新领域（比如量化投资），给出推荐的阅读路径积分策略"——第一本读什么（定框架），第二本读什么（填充维度），第三本读什么（换视角积分）。比较"好的路径"和"坏的路径"的积分效率差异。 |
| **行动/启示** | ① 进入新领域前，先设计"积分路径"而不是随意找书单<br>② 读不同视角的文章 = 多路径积分，比强化一个方向更接近真理<br>③ 感觉"读了但没进展"时，检查是否积分边界太窄（总在同一个维度积分） |
| **约束** | ✅ 引入∫符号（概念性符号）<br>❌ 不写路径积分的数学公式 |

### 2.5 章节04：迭代收敛——走向真理的贝叶斯之路

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 学习/理解/接近真理的本质是"世界模型参数序列的收敛"。P₀ → P₁ → P₂ → ... → P*，其中P*是"基于现有所有信息的理性认知极限"——这既是贝叶斯更新链，也是迭代收敛过程。 |
| **字数** | 3000-4000字 |
| **引子** | "你是什么时候真正'想通'一件事的？"——那种"之前觉得对的事突然不认同了"的时刻，就是模型参数的跳跃。引出"认知不是跳跃，是迭代——只是你不知道中间步"的感觉。 |
| **概念展开** | ① **迭代方程**：Pₙ₊₁ = f(Pₙ, Eₙ)——P是认知状态，E是新信息，f是更新规则。每一轮更新都依赖当前认知状态——所以"同一个消息在十年前和今天的冲击完全不同"。<br>② **收敛条件**：什么情况下认知会收敛？信息充足、信源可信、评估函数对齐、无系统性偏误。什么情况下不收敛？missing data（缺关键信息）+ 认知偏误（损失函数歪了）+ 争议未决（边界在震荡）。<br>③ **收敛速度**：几何收敛（每次迭代都能减少一半差距）vs 线性收敛（每次迭代都减少固定比例差距）vs 震荡不收敛。好老师/好书＝加速收敛（高信息密度+高可信度）。坏信息源＝延迟收敛/误导收敛方向。<br>④ **真理作为固定点**：P*的定义——"如果所有相关信息都给了你，你仍然不改变这个认知，这就是P*"。不是绝对真理，是条件真理——在给定信息集下的最优认知。<br>⑤ **信息茧房的收敛陷阱**：只读同一立场的文章→在错误的吸引子附近收敛（局部最优≠全局最优）。打破的方法：引入对抗训练（读对立观点），迫使模型跳出局部最优。 |
| **母书链接** | 链接"说服：世界模型参数更新"——说服的本质是给对方的世界模型提供训练数据，目标是"让对方的Pₙ收敛到你想要的方向"。 |
| **案例** | "三年的认知转变"——取一个真实话题（如"AI会不会取代工作"），模拟三年前的P₀→经历各种信息→三年后的P_current。标注每个节点：哪件事引起了梯度反转，哪件事加速了收敛。 |
| **行动/启示** | ① 定期检查你的认知有没有收敛（还是停滞/震荡）<br>② 感觉到"三观被刷新"时，记录刷新你的信息是什么<br>③ 主动搜索对立观点 = 做对抗训练，避免局部最优收敛 |
| **约束** | ✅ 引入lim和→符号<br>❌ 不展开贝叶斯公式（用"先验→后验"的人话版本） |

### 2.6 章节05：泛函优化——六合一的统一视角

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 阅读、思考、表达、演讲、辩论、说服——这六个认知活动都可以统一为一个数学框架：**在某个空间上找使目标泛函取极值的路径或参数**。理解这一点，你就拥有了"元认知的数学直觉"。 |
| **字数** | 2000-3000字 |
| **引子** | 回看母书的六活动信息流：阅读→思考→表达→演讲→辩论→说服。现在用泛函优化的语言重新描述它们——你会发现它们是同一类优化问题的不同实例。 |
| **概念展开** | ① **泛函是什么**（人话版）："泛函就是答案的答案的答案"——输入一个东西，输出一个好坏分数。你的大脑一直在默默计算各种泛函：这个方案好（分数高）、那句话不合适（分数低）。<br>② **六活动的泛函统一表述**：<br>  - 阅读：找P* = argmin d(P, P_author) —— 让世界模型最接近作者的世界模型<br>  - 思考：找解 = argmin 路径长度 s.t. 评估函数约束<br>  - 表达：找文本 = argmin d(intent, decoder(text)) —— 让读者解码最接近你的意图<br>  - 演讲：找迁移轨迹 = argmin 迁移时间 s.t. 状态可达<br>  - 辩论：找证据 = argmax 边界偏移量 s.t. 证据可信度<br>  - 说服：找训练数据 = argmax 参数更新量 s.t. 修改步长限制<br>③ **为什么这有用**：当你把一个问题翻译成"在什么空间找什么样的极值"的时候，你自动回答了——搜索空间、评估函数、约束条件、停止条件。这是母书每个活动方法论的第一性原理基础。 |
| **母书链接** | 母书每个活动的方法论可以映射到泛函优化的每一步：方法论 = 对特定泛函问题的求解策略。这是母书的"元解释"。 |
| **案例** | 选一个"说服"的场景（特斯拉销售），展示它本质上是"在对方世界模型参数空间找一个可行的更新方向"——这个表述让销售策略的每一步都有了数学意义：选训练数据→往目标方向∇→控制步长→验证是否收敛。 |
| **行动/启示** | ① 下次遇到沟通难题，问自己"这是哪个空间+哪个泛函的优化问题"<br>② 明确泛函 = 明确目标——你表达（写/说）的时候，"好"的标准是什么？<br>③ 所有方法论的底层都是这个：定义空间→定义泛函→求解。 |
| **约束** | ✅ 使用argmin/argmax符号（概念性）<br>❌ 不写变分法的数学公式<br>❌ 不出现泛函的严格数学定义（Lebesgue积分、变分导数等） |

### 2.7 章节06：推导演示——从模糊到真理的完整运算

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 用一个完整案例展示"语言微积分"的全程运算——从模糊想法出发，经过语义微分→积分→迭代收敛的每个步骤，最终输出一个经过计算的结论。这是一个"像解微积分题一样运算语言"的实操演示。 |
| **字数** | 3000-4000字 |
| **引子** | "你遇到过这种决策吗——想了很久但没想明白。不是你智商问题，是你没有把思考过程当成运算。今天俺老猪带你算一遍。" |
| **案例选择** | **"是否应该跳槽"**——这是一个普通人熟悉的、有多个维度、有情感因素、有信息不足的典型决策。适合展示完整运算过程。 |
| **运算步骤** | 每步标注对应的数学运算：<br><br>**第1步：初始状态P₀**（积分初始值）<br>  - 你的原始状态：对当前工作"还行但有点烦"，对跳槽"想但怕"<br>  - 数学意义：在"工作价值"空间中的初始状态向量<br><br>**第2步：语义微分∇**（读相关文字，求每一篇的语义梯度）<br>  - 读一篇《工作幸福感研究报告》→ 梯度方向：工作满意度 ≈ 收入×文化×成长，你的当前值偏低<br>  - 读一篇《跳槽失败案例》→ 梯度方向：小心未知风险，保守为上<br>  - 读一篇《某个朋友跳槽成功的案例》→ 梯度方向：机会是留给行动的人<br>  - 计算各篇的梯度方向和大小，标注矛盾（保守vs冒险两个方向在打架）<br><br>**第3步：语义积分∫**（多路径积分，累积认知）<br>  - 从三个领域积分：主观感受路径 + 客观市场路径 + 他人经验路径<br>  - 对比不同积分路径的收敛速度：哪条路径提供的信息量最大（梯度最大）<br>  - 发现"主观感受"路径积分已收敛（你其实已经知道答案，只是不敢承认）<br><br>**第4步：迭代更新P₀→P₁→P₂**（每次思考后的认知收敛情况）<br>  - P₀：模糊（知道方向不知道具体）<br>  - P₁：有结构了（把这个决策拆成8个维度）<br>  - P₂：收敛到一个方向（在5个维度上有明确倾向，3个维度存疑）<br>  - 标注每次迭代的"信息增量"：量化和质化新信息<br><br>**第5步：检查收敛条件** <br>  - 是否满足收敛条件？信息来源是否充分且可靠？<br>  - 存疑的三个维度是否可以通过"一次快速实验"（面试其他公司）获得新数据？<br>  - 设计"收敛加速器"：去面试但不要答应——获取市场信息<br><br>**第6步：输出结论** <br>  - 当前最优解：可以开始准备跳槽，先面试拿数据而不是直接辞职<br>  - 数学意义：这是一个"局部最优，但不是全局最优"的结论——因为信息集还不完整<br>  - 留了一个"参数更新接口"：去面试回来后P₂→P₃，再做一次迭代 |
| **每步格式** | 每步用表格展示：<br>```<br>**运算** | **输入** | **操作** | **输出** |<br>:---|:---|:---|:---|<br>微分∇ | 工资对比文章 | 计算梯度方向和大小 | 梯度强但方向单一 → 信任权重偏低 |<br>``` |
| **约束** | ❌ 案例不能假大空——必须是有真实感的决策<br>❌ 不能只展示"完美案例"——要展示有矛盾、有不确定性的运算过程<br>❌ 运算不是严格的数值计算，是"概念化运算"——每一步都要让读者看懂"在算什么" |

### 2.8 章节07：结语——语言微积分的意义与边界

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 语言微积分框架能做什么、不能做什么。它不是数学——它是"数学感"的思维方式。它的价值在于让你用微积分的眼光看文字，而不是真的去算向量。 |
| **字数** | 1000-1500字 |
| **内容** | ① **回顾**：我们从母书的"语言是计算"一路走到"语言可以像微积分一样运算"<br>② **这个框架能做什么**：<br>  - 给母书的方法论一个更底层的解释——不是"怎么做"，是"为什么这么做有效"<br>  - 把模糊的沟通直觉变成可说的"我这个梯度方向不对"<br>  - 当你感觉"信息不够"或"决策没收敛"时，你有一个检查框架<br>③ **这个框架不能做什么**（诚实面对边界）：<br>  - 不能替代真正的数学建模——这是思维框架，不是科学工具<br>  - 不能帮你算精确的"梯度值"——语义空间目前还没法数值化<br>  - 不是NLP技术的简化版——它面向的是人，不是机器<br>  - 当一个问题涉及太多不确定变量时，框架只能帮你"看清"不能帮你"算清"<br>④ **真正的价值**：框架最大的价值不是"算准确"，是"让你意识到语言活动有底层结构"。就像你学了物理之后看世界不一样——学了这个框架之后看文字不一样。<br>⑤ **引母书的回归**：框架给你"为什么"，母书给你"怎么做"——两者结合：知道底层在算什么，才能更好地决定怎么输出。 |

| **收尾** | 猪猪侠式收尾：有力量、不鸡汤、留一个让人合上书后想东西的句子 |

### 3.4 猪猪侠语言风格

- 不装逼：数学概念是为表达服务的，不是为展示"我懂微积分"
