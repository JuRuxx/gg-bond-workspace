## 一、缩进与格式

### ✅ 必须遵守
| 规则 | 要求 | 示例 |
|:----|:-----|:-----|
| 缩进 | **4个空格**，严禁Tab | `def foo():␣␣␣␣pass` |
| 行长 | 最大 **88字符**（Black默认标准） | 超长需换行 |
| 换行 | 括号内隐式对齐或悬挂缩进 | 见PEP 8 |
| 空行 | 顶级函数/类之间**2个空行**，方法之间**1个空行** | |
| 编码 | **UTF-8**，文件头不用声明 `# -*- coding: utf-8 -*-` | Python 3默认 |

### 换行示例
```python
# ✅ 正确：括号内对齐
long_function_name(var_one, var_two,
                   var_three, var_four)

# ✅ 正确：悬挂缩进 + 4空格
foo = long_function_name(
    var_one, var_two,
    var_three, var_four
)
```

## 二、命名规范

| 类型 | 规范 | 示例 |
|:----|:-----|:-----|
| **模块名** | 全小写+下划线 | `ir_temp.py`, `patroller.py` |
| **类名** | 驼峰(CamelCase) | `Patroller`, `TemperatureSensor` |
| **函数/方法** | 全小写+下划线 | `read_temperature()`, `start_patrol()` |
| **变量** | 全小写+下划线 | `current_temp`, `battery_level` |
| **常量** | 全大写+下划线 | `MAX_TEMP_THRESHOLD = 85.0` |
| **私有属性** | 前导单下划线 | `_internal_state` |
| **私有方法** | 前导单下划线 | `_calibrate_sensor()` |
| **避免的名字** | 不要用单字母 | 循环变量可用 `i`, `j`，其他必须起有意义的名字 |

## 三、注释与文档

### Docstring（PEP 257标准）

每个**公开函数/类/模块**必须有docstring：

```python
def read_temperature(sensor_id: int) -> float:
    """读取MLX90614红外温度传感器的物体温度。

    Args:
        sensor_id: 传感器I2C地址（默认0x5A）

    Returns:
        物体温度，单位摄氏度

    Raises:
        IOError: I2C通信失败时抛出
    """
    pass
```

### 行内注释
- 只在**代码意图不够清晰**时加
- 注释和代码之间至少**2个空格**
```python
x = x + 1  # 边界补偿（因为传感器有1°C偏差）
```

### 块注释
```python
# 以下代码处理低电量自动回桩逻辑
# 电量阈值可在配置文件中修改
if battery_level < CHARGE_THRESHOLD:
    start_docking()
```

### ❌ 禁止的无意义注释
```python
# ❌ 废的
x = x + 1  # 给x加1

# ✅ 好的
x = x + 1  # 温度累计补偿（每10次采样+1°C）
```

## 五、代码质量红线

### 🚫 绝对禁止
1. **没有异常处理的硬件调用**
   ```python
   # ❌ 禁止
   temp = sensor.read()

   # ✅ 必须
   try:
       temp = sensor.read()
   except IOError as e:
       logging.error(f"传感器读取失败: {e}")
       temp = None
   ```

2. **硬编码魔数**
   ```python
   # ❌ 禁止
   if distance < 20:  # 这是啥？
       stop()

   # ✅ 必须
   MIN_SAFE_DISTANCE_CM = 20  # 最小安全距离
   if distance < MIN_SAFE_DISTANCE_CM:
       stop()
   ```

3. **裸print（用logging替代）**
   ```python
   # ❌ 禁止
   print("温度:", temp)

   # ✅ 必须
   import logging
   logging.info(f"温度: {temp}°C")
   ```

4. **超过50行的函数（拆！）**
5. **没有类型注解的关键函数参数**

### ✅ 必须做到的
- 每个函数不超过50行
- 每个文件不超过300行
- 异常链用 `raise ... from e` 保持上下文
- 配置文件使用YAML/JSON，不用硬编码常量
- 所有外部资源（GPIO/I2C/文件）使用 `with` 语句或确保释放

## 六、Git提交规范

### 提交信息格式
```
<类型>: <简短描述>

<详细描述（可选）>
```

### 类型
| 类型 | 何时使用 |
|:----|:---------|
| `feat` | 新功能 |
| `fix` | 修复bug |
| `docs` | 文档更新 |
| `style` | 代码格式（不影响功能） |
| `refactor` | 重构 |
| `test` | 测试 |
| `chore` | 构建/工具 |

### 示例
```
feat: 添加MLX90614红外测温驱动

- 支持I2C通信
- 读取物体温度和环温
- 含错误重试和日志
```
