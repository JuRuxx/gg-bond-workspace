# 主页技术规格文档

## 1. 目录结构

现有 `novel-web/` 不变，新增文件：

```
novel-web/
├── index.html             # ← 重构为：个人简介 + 双卡片导航
├── novel/                 # 小说章节（现有）
├── 天兵科技投资分析报告.md  # 投资报告（现有）
├── Caddyfile              # 不变
├── server.crt / server.key  # 不变
```

无需路由框架，单 HTML 页。

## 2. HTML 结构

```
<body>
  <header>        → 昵称 + 简短标语
  <section.hero>  → 头像 / 个人简介文案（1-2行）
  <section.cards> → .card × 2，平铺居中
  <footer>        → 版权 / 社交链接
</body>
```

卡片 `<a>` 包裹，`href` 直跳：

| 卡片 | 跳转目标 | 说明 |
|------|---------|------|
| 📖 小说入口 | `./novel/` 或 `./novel/index.html` | 当前小说阅读页，若重构则新建 `novel/index.html`（与原 index.html 同内容，仅目录迁移） |
| 🚀 投资分析 | `./天兵科技投资分析报告.html` | Caddy 可渲染 Markdown，故直接链接 `.md` 也行；后续新增入口在 `./reports/` 下追加卡片 |

## 3. 卡片设计规范

| 属性 | 值 |
|------|-----|
| 尺寸 | `width: 280px; height: 180px` |
| 圆角 | `border-radius: 16px` |
| 背景 | `background: var(--card-bg)`，半透明磨砂（可选 `backdrop-filter: blur(8px)`） |
| 边框 | `border: 1px solid var(--card-border)` |
| hover | 上移 `translateY(-4px)` + 阴影放大 `box-shadow: 0 8px 24px rgba(...)`，过渡 `0.2s ease` |
| 图标 | Emoji 或 SVG inline，居中上方 |
| 文字 | 标题 18px bold，描述 13px secondary color |
| 点击 | `<a>` 原生跳转，无需 JS |

## 4. CSS 方案

CSS 变量挂载在 `:root`，支持暗色主题通过 `prefers-color-scheme: dark` 切换：

```css
:root {
  --bg: #f5f5f7;
  --text: #1d1d1f;
  --text-secondary: #6e6e73;
  --card-bg: rgba(255,255,255,0.85);
  --card-border: rgba(0,0,0,0.06);
  --accent: #0071e3;
  --max-width: 720px;
}

@media (prefers-color-scheme: dark) {
  :root {
    --bg: #1d1d1f;
    --text: #f5f5f7;
    --text-secondary: #98989d;
    --card-bg: rgba(44,44,46,0.85);
    --card-border: rgba(255,255,255,0.08);
    --accent: #2997ff;
  }
}
```

布局使用 Flexbox 居中，无需第三方 CSS 库。Transition 统一 `0.2s ease`。Font stack：`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`。

## 5. 兼容性要求

- 现代浏览器：Chrome 90+ / Firefox 90+ / Safari 15+ / Edge 90+
- 无 polyfill，无 JS 依赖
- 移动端：卡片 `flex-wrap: wrap` 适配 <480px 单列

## 6. Caddy 配置

**不变。** 当前配置已托管 `novel-web/` 为静态文件服务器，新 `index.html` 自动生效。若跳转 `.md` 文件，Caddy 默认渲染为 HTML（`file_server` 内置 markdown 渲染）。无需增加 rewrite 或反向代理规则。
