> 从《Toyota Production System》第二章提炼的六把实操工具。
> 不是我翻译的，是我**剥开皮看里面的肉**。

## 工具一：5Why — 问5次为什么

### 大野原版的演示

**问题**：机器停了。

| 层数 | 追问 | 答案 |
|-----|------|------|
| 1 | 为什么机器停了？ | 保险丝烧了。 |
| 2 | 为什么会超负荷？ | 轴承润滑不够。 |
| 3 | 为什么润滑不够？ | 油泵没抽上油。 |
| 4 | 为什么油泵没抽上油？ | 泵轴磨损松了。 |
| 5 | 为什么泵轴磨损了？ | **没装滤网，铁屑进去了。** ← 根因 |

**两种处理方式对比：**

```
普通做法：换保险丝 → 3天后又烧 → 再换 → 恶性循环
大野做法：装滤网 → 解决问题 → 永不复发
```

### 这个工具到底怎么用（不是问5次就好）

**大野的5Why不止是追问技巧，而是：**

**1. 它是"科学思维"在现场的应用**

> "说实话，丰田生产体系就是建立在这种科学方法的实践和进化之上的。"

5Why本质上在做一件事：**从现象推到原因 → 再从原因推到更深的原因 → 直到触及系统层面**。

**2. 它不是一次性的，是一整套思维方式的投射**

大野演示了5Why如何帮他设计整个TPS：

| 问题 | 追问链条 | 答案 → 体系级方案 |
|-----|---------|------------------|
| 为什么1个汽车工人看1台机器，而纺织厂女工看40台？ | →为什么汽车厂不能？ | →因为机器没有判断能力→ **自治化（Autonomation）** |
| 为什么不能准时制？ | →为什么前工序做太快？ | →因为我们不知道生产节奏→ **平准化** |
| 为什么做太多零件？ | →为什么没法阻止超产？ | →因为没有信息反馈→ **看板（可视化控制）** |

**5Why不只是修机器的时候用，还是设计整个生产体系的时候用。**

**3. 在实际工作中怎么用：**

```
发生故障 →
    ① 拍下现场照片/记录数据 →
    ② 召集相关的人（不要自己写） →
    ③ 在白板上写"为什么？" →
    ④ 逼大家回答，每回答一个再追问"为什么？" →
    ⑤ 直到找到系统层面的根因 →
    ⑥ 改了系统，而不是改了症状
```

### 你们的厂怎么用

**场景一：设备售后问题**

> 客户说"你们的点胶机又漏胶了"

```
1问：为什么漏胶了？ → 密封圈磨损
2问：为什么密封圈这么快就磨损了？ → 胶水有硬颗粒
3问：为什么胶水有硬颗粒？ → 胶水供应商批次问题
4问：为什么之前的批次没问题？ → 供应商换了原料配方没通知
5问：为什么供应商换配方不通知？ → 没有对供应商做工艺变更管控
```

结果：不是多配密封圈给客户，是**建立供应商变更管理流程**。

**场景二：产线效率问题**

> "今天又没完成产能目标"

```
1问：为什么没完成？ → 某个工序卡了
2问：为什么工序卡了？ → 操作工换了好几次料
3问：为什么需要换这么多次料？ → 料箱太小
4问：为什么料箱这么小？ → 没考虑换料时间
5问：为什么设计时没考虑？ → 设计部门和产线没沟通
```

结果：不是加人加班，是**在设计和产能之间建立反馈机制**。

## 工具二：7大浪费（7 Wastes）— 大野的原版定义

### 大野的原话

> "应用丰田生产体系的第一步，就是完整识别浪费：
> 1. 过生产的浪费
> 2. 等待的浪费
> 3. 搬运的浪费
> 4. 加工本身的浪费
> 5. 库存的浪费
> 6. 动作的浪费
> 7. 制造次品的浪费"

### 公式

```
现有能力 = 工作 + 浪费

真正的效率提升 = 消灭浪费 → 让"工作"达到100%
```

**关键**：效率提升必须跟成本挂钩。不是产量越高越好，是只生产需要的东西。

### 逐个拆解（带自动化设备厂视角）

#### 1. 过生产的浪费 — 最可怕的浪费

**大野说：** "没有比过生产更可怕的浪费。"
**为什么可怕？** 不只是做了多余的东西——过生产**掩盖了其他6种浪费**。

```
做过生产的车间：
  ┌→ 看起来忙忙碌碌
  ├→ 产生了大量库存
  ├→ 搬运/动作/等待都被掩盖
  └→ 管理层觉得"产能很满"

真正的状况：
  需求只有100件，做出来200件
  → 100件压着 → 空间/资金/管理成本
  → 出错时200件全废
```

**在自动化设备厂的体现：**
- 备品备件做一堆（"反正以后用得上"）
- 机架标准化做得太过头（"60种机型共用一种机架"→浪费）
- 功能堆叠（"反正PLC加个功能不花钱"→实际上维护成本翻倍）
- 「做了再说」心态

**如何判断是否过生产浪费：**

```
问自己三个问题：
① 这批做了谁会马上用？
② 如果现在不做，客户会等死吗？
③ 做完后，库存放哪？（要交仓库费=浪费）
```

#### 2. 等待的浪费

**大野版本：** 人等机器、机器等人、前工序等后工序。
**在自动化设备厂的体现：**
- 调试期：技术员A等机械装完 → 机械装完了等电气
- 生产期：产线等物料 → 物料到了等换型
- 售后：工程师等客户确认 → 客户确认了等配件到
- 最扎心的：**项目经理等各部门汇报**→然后才发现进度延误

**一句话：** 只要有"干等"的地方，就是改进点。

#### 3. 搬运的浪费

**大野版本：** 零件搬来搬去不产生价值。
**在自动化设备厂的体现：**
- 车间布局不合理：机架做了要搬很远去组装，组装好了又搬去调试
- 仓库物料搬运：领料→搬运到工位→多余的还要搬回去
- 产线之间：半成品从产线A搬到产线B（如果有空间，这一条不存在？恰恰是有空间才有搬运浪费）

**大野的解法——建立生产流：**
把机器按工序顺序排列，而不是按机器类型排列。这样零件就是在流水线上走，而不是被搬来搬去。

#### 4. 加工本身的浪费

**大野版本：** 加工本身带来的浪费。
**在自动化设备厂的体现：**
- 公差不必要地严：客户要求±0.1，内控标准±0.01（3倍的精度=3倍的成本+3倍的时间和管控难度）
- 表面处理过度：明明没人看的地方也抛光喷漆
- 功能过剩：本来只测5个参数，机器具备测20个参数的能力

**关键问题：** 这些多余的加工是谁要求的？技术部门自己定的标准？客户明确要求的？还是"保险起见"?

#### 5. 库存的浪费

**大野版本：** 原材料/半成品/成品的积压。
**大野的洞见：**

> "我们从农耕社会继承了囤积的本能。但低速增长期，囤积就是浪费。"

**库存的隐藏成本（不是只有资金占用）：**
- 管理成本：库房、搬运、盘点、防损
- 质量风险：库存期长了会变化（电子元器件有保质期）
- 掩盖问题：库存多→生产线停了也能撑→没人关心停线原因
- 信息失真：库存多→判断不出真实的客户需求

**在自动化设备厂的体现：**
- 标准件囤积（"常用件多备点"→结果变更了用不上）
- 半成品堆积（"先做好放着"→客户改方案了全部重来）
- **最致命：** 库存多了，管理者看不到现场的真实问题

**大野的名言（后来自成一派）：**
> "库存就是水面，石头就是问题。水面高的时候看不到石头。水位降了，石头露出来了，你才能去搬走它们。"

#### 6. 动作的浪费

**大野版本：** 多余的动作——工人绕很远去拿工具、弯腰、转身。
**在自动化设备厂的体现：**
- 工作台面：工具、零件、图纸散落到处
- 装配流程：拿螺丝→拧螺丝→放扳手→走几步拿另一把扳手
- 调试过程：PLC程序改一下→到现场跑一圈→回来再改

**大野的思路：** 不是"工人动作不够快"，是**设计上有问题**导致工人不得不做多余动作。

**解法：** 标准作业 + 5S（整理整顿清扫清洁素养），减少找东西、走路、弯腰的时间。

#### 7. 制造次品的浪费

**大野版本：** 返工、报废。
**跟别的浪费的关系：**
- 过生产 + 次品 = 做了一堆废品
- 等待 + 次品 = 做了半天发现要返工
- 库存 + 次品 = 一批货全废

**大野的解法不是靠检查：**

> 不是靠QC最后来挑次品，而是**从根源上防止产生次品**。

**自治化（Autonomation）：**
- 机器出问题自己停
- 问题暴露→改进→不再发生

**5S + 标准化：**
- 操作步骤标准化 → 减少人为失误
- 现场整洁 → 减少误判

**在自动化设备厂的体现：**
- 视觉检测方案本身不准 → 漏检/误检 → 次品流出
- 装配时错装/漏装 → 要返工拆开重装
- 设计问题 → 到客户现场才发现 → 售后大修

### 对这7种浪费，一个大野式的核心理念

**不要一次性消除所有浪费，先聚焦过生产（#1）和次品（#7）。**

为什么？
- 过生产是"母浪费"——消除了它，库存/搬运/等待都会跟着消失
- 次品是"最花钱的浪费"——一个次品出去，损失不只是零件成本

## 工具三：标准作业三要素

### 大野原版的定义

三要素：
1. **节拍时间（Cycle Time）** — 做一个零件/单位的时间
2. **作业顺序（Work Sequence）** — 工人操作的正确顺序
3. **标准手持（Standard Inventory）** — 工序间的最小在制品

### 节拍时间怎么算

```
节拍时间 = 操作时间 ÷ 每天需求量
         = 480分钟 ÷ 40件/天
         = 12分钟/件
```

**关键：不是机器最快速度，是客户需要的速度。**

如果机器一个零件3分钟就能做出来，但客户只要12分钟一件——你开快机器不是好事，是过生产。

### 为什么大野强调"你自己写"

> "标准作业不能坐在办公桌前写。必须到车间现场试了又试改了又改。"

**更深的原因：**
- 只有写标准作业的人真正理解了工序，才能教别人
- 写的过程本身就是一次完整的改善机会
- 如果你没能力写——说明你不够了解这个工位

**大野的训练方法：**

> "培训一个新工人只需要3天。但培训者必须亲手握住工人的手来教。这才能建立信任。"

### 在自动化设备厂怎么用

**场景一：装配作业标准**

```
节拍时间：客户要求每天交付2台 → 480 ÷ 2 = 240分钟/台
作业顺序：
  ① 领取机架及零件（5分钟）
  ② 安装X轴模组（40分钟）
  ③ 安装Y轴模组（35分钟）
  ④ 电气接线（60分钟）
  ⑤ 视觉安装调试（50分钟）
  ⑥ 通检（30分钟）
  ⑦ 包装（20分钟）
  总计：240分钟 → 正好符合节拍

标准手持：1台（装配台上） + 1台（检验区）= 2台
```

**场景二：调试作业标准**

```
节拍时间：调试周期=客户合同约定的交付日
作业顺序：
  ① 上电自检（15分钟）
  ② 手动对各轴运动（30分钟）
  ③ 执行全自动程序（2小时）
  ④ 检查各工位动作时序（1小时）
  ⑤ 检查次品分拣逻辑（30分钟）
  ⑥ 连续运行10min验证（2小时）

标准手持：0台（调试完成就走，不停留）
```

### 标准作业的大野式进阶用法

标准作业不是"刻在墙上的，不准改"，而是：
- 今天写下 → 看到更好的做法就改
- 改一次 → 产量提升/质量提升/劳动强度降低
- **改到所有人都认同为止**

**标准作业的目的是什么？**
> 不是为了限制工人，是为了**发现问题**。

如果有标准作业，工人不按标准做→原因是什么？
- 标准写错了→改标准
- 工人没学会→再培训
- 工具/机器有问题→维修

**没有标准化，就没有改进的基准。**

## 工具四：互助运动 — 接力赛模式

### 大野原版的场景

> "工作场所就像田径接力——如果接力棒传得好，总成绩可以比四个运动员各自的最佳成绩加起来还要好。"

**具体做法：**
- 一个后工序的工人被耽误了 → 其他人帮他设机器
- 恢复正常后 → 被帮的人拿回接力棒 → 其他人回原位
- 目标是 **每个人都能熟练地交接棒**

**大野叫它 "互助运动"（Mutual Assistance Campaign）。**

### 为什么这很重要

**大野对日本传统工作方式的观察：**

> "在日本，传统上竞争是个人的——相扑、剑道、柔道。我们不叫'竞争'，而是'求道'。但在现代工业中，**团队协作比个人技艺更重要**。"

### 在自动化设备厂的体现

**你们厂的三组人：**

```
   技术部                   生产部                  售后部
┌──────────┐          ┌──────────┐           ┌──────────┐
│ 机械设计  │          │ 装配组   │           │ 安装调试  │
│ 电气设计  │          │ 调试组   │           │ 维保     │
│ 软件     │          │ 质量检   │           │ 客户沟通  │
└──────────┘          └──────────┘           └──────────┘

现实：三个部门之间是「扔过墙」——设计完扔给生产，生产完了扔给售后
```

**大野会怎么改：**

```
不是三个独立的部门，而是一个接力队

设计 ← 装配 → 调试 → 安装

信息流  实物流  信息流
 ↓        ↓       ↓
装配反馈设计问题
调试反馈装配问题
售后反馈调试问题

每个环节收到"上一棒传递来的信息"，就是改进点
```

**实操例子：**

装配工发现某个螺丝很难装 → 不是抱怨，而是：
1. 标记这个问题
2. 设计组接收到 → 改设计
3. 一个月后检查 → 修复了吗？

这要求什么？**信息闭环**。

### 大野谈"训练"（这本书最动人的一段之一）

> "用脑子理解理论很容易；问题是**用身体记住它**。目标是做到本能地知道和去做。有毅力忍受训练是第一步。"

**这段话的分量：**
- 大野不是在说"去培训"
- 是在说"**把正确的做法练到肌肉记忆里**"
- 就像运动员的接棒练习——理论上一遍就懂，但不练100遍就是会在赛场上掉棒

## 工具五：看板 — 6条规则的实操解读

### 看板到底是什么

大野的原始定义：一张装在乙烯基信封里的纸。

**信息类型：**
1. **领料信息** — 取什么、取多少
2. **转移信息** — 从哪里到哪里
3. **生产信息** — 前工序要做什么

**两个最基本的看板类型：**

```
取货看板（Withdrawal Kanban）：
  后工序拿着它去前工序 → "我要这些"

生产看板（Production Kanban）：
  前工序根据它生产 → "你要的我做了"
```

### 6条规则逐条拆解

#### 第1条：后工序去前工序取产品

**反常识的地方：**

```
传统：前工序做了 → 推到后工序
看板：后工序需要 → 去前工序取
```

**这个反常识有什么用？**

- 前工序不用猜测后工序要什么（不用做预测）
- 后工序只拿需要的东西（不多拿）
- **信息流的方向和物料流的方向正好相反**

**类比：**
超市模式——你（后工序）去货架（前工序）拿需要的商品。货架空了→超市补货（前工序生产）。超市不会在你还没拿的时候就把东西送你家里。

#### 第2条：前工序只生产被取走的数量

**看起来简单，实际上影响巨大：**

```
限制：
  - 前工序想多做？→ 没有看板 → 不能多做
  - 前工序想提前做？→ 没有看板 → 不能提前
  - 前工序做错了？→ 看板不匹配 → 立即发现
```

**效果：**
- 过生产彻底被阻止
- 库存自然降到最低
- 每个零件都有"身份证"（看板）

#### 第3条：没有看板不准取也不准生产

**这是一条铁律。**

大野自己说的：
> "在开始时我不得不强制推行第3条规则。尤其是第3条——没有看板就不准取——是那些不熟练的工人最常违反的。"

**为什么必须强制执行？**
- 因为人的本能：先拿再说、先做再说
- 看板就是**把"反本能"制度化**
- 如果没有铁律，看板体系就会崩溃

**在自动化设备厂：**
类似地——比如**没有工单不准开工**。看起来简单，但管理层的直觉往往是"先做着再说，工单回头补"。但一旦开了这个口子，整个计划体系就废了。

#### 第4条：看板必须附着在商品上

**看板就是商品的信息标签——看板在哪，信息在哪。**

**效果：**
- 一眼就知道这是什么、从哪里来、去哪里
- 不需要纸质单据
- 不需要ERP查询
- **可视化控制**

**在自动化设备厂：**
- 零件架上有看板/标签 → 永远不会拿错零件
- 机器上有看板/标签 → 知道这台机器什么状态

#### 第5条：100%合格品

**大野的狠招：**

> "如果前工序产生了次品，后工序必须停线。次品退回前工序。这是有针对性的难堪，目的是防止再发生。"

**不是靠检查体系保证质量，而是靠倒逼机制：**
- 次品一旦产生 → 后工序停线 → 所有人都知道 → 压力和羞耻感 → 彻底解决
- 不是"出了次品赶紧检查发现"——是**根本不让次品流出来**

**在自动化设备厂：**
- 设备出厂前的全检不是目的，过程控制才是
- 如果一台设备必须靠最后全检才能过关→说明过程控制有漏洞

#### 第6条：减少看板的数量

**这是最狠的一条——持续改进的机制化。**

```
看板多 = 库存多 = 浪费多
  ↓
要求减少看板数量
  ↓
必须减少库存
  ↓
暴露问题（石头露出水面）
  ↓
必须解决问题
  ↓
才能继续减少看板
  ↓
循环……
```

**本质上，第6条把"持续改进"变成了一个自动循环的机制：**

```
减少看板 → 暴露问题 → 必须改进 → 改进完成 → 再减少看板 → 再暴露新问题……
```

**不是靠领导推动，是靠体系自己推动。**

### 大野的警告

> "一知半解地引入看板，既不遵守规则，也不会带来预期的控制或成本降低。看板只会带来百害而无一利。"

**看板的正确引入顺序：**
1. 在公司内部建立生产流（先别搞外协厂）
2. 内部实验看板
3. 3-5年后稳定了
4. 才扩展到外协厂

**大野用了20年（1940s→1963）才把看板推到所有外协厂。**

## 工具六：平准化 — 山低谷浅

### 大野原版的命题

> "山应该低，谷应该浅"（Mountains Should Be Low and Valleys Should Be Shallow）

**为什么平准化是必需的：**

```
后工序领取得不均匀（峰谷波动）
          ↓
前工序必须备余力应对高峰
          ↓
余力在低谷期闲置 → 浪费
          ↓
加上外协厂也被牵连
          ↓
必须让最终总装线的波动最小化
```

### 核心定义

**生产平准化（Production Leveling, Load Smoothing）** = 降低峰值、填平谷底，让流量表面平滑。

### 最震撼的数字：换模时间

```
1940年代：换模要 2-3 小时
  ↓ 大野+新乡重夫持续改进
1950年代：15分钟到1小时
  ↓
1960年代末：3分钟
```

**怎么做到的？** 不是机械改进，是**整个思维方式的转变**：
- 不把换模看成"停机时间"→ 而是看成"必须做的事情"
- 预先准备好一切（工具、模具、设置）
- 多人在同一时间做不同的事（像F1进站）
- **问题不是"怎么快点换好"，而是"出厂就设计好快速换模"**

### 混流生产——花冠和卡利那同线跑

**传统工厂：**
```
周一周二：全部花冠
周四周五：全部卡利那
          ↓
物料波峰波谷巨大
```

**丰田的混流生产：**
```
花冠 → 卡利那 → 花冠 → 旅行车 → 花冠 → 卡利那……
          ↓
每台车都不同，但流量是平稳的
```

### 在自动化设备厂的平准化

**场景：你们厂的订单往往是几种机型在一起**

传统做法：
```
1-10日：集中做视觉拔磁规机
11-20日：集中做三轴点胶机
21-30日：集中做六工位点胶机

问题：
  - 工人前半段做熟练了→又换机型→重新熟悉
  - 物料集中到货→仓库爆满
  - 一个客户等整套设备→但同一客户的不同设备分时段做→客户要等齐才能验收
```

**平准化做法：**

```
每天各机型都做一些：
  周一：拔磁规*2 + 三轴*1 + 六工位*2
  周二：拔磁规*2 + 三轴*1 + 六工位*2
  ...

效果：
  - 工人持续接触各机型（技能保持）
  - 物料均匀到货（仓库压力小）
  - 每个客户的设备同步推进（验收时间缩短）
```

### 大野的核心理念

> "丰田生产体系具有无与伦比的适应变化的弹性。这种能力是它在低速增长期的力量源泉。"

**这句话的潜台词：**
- 传统大批量生产在高速增长期很强大
- 一旦进入低速增长/多样化需求，它就僵硬了
- 丰田的体系从设计之初就是为了**多品种小批量**
- 所以它在变化面前是最有弹性的

**对你们厂的延伸：** 自动化设备本身就是非标、小批量的。大野的体系天然适合非标自动化设备厂——但你们自己有没有意识到这一点？还是仍然在用"大批量思维"做非标？
