# 管理文化比较研究：日欧美 vs 中国

> **一句话主题**：把日本、欧洲、美国的管理文化吃透，看哪些能用到中国人的组织和公司里。

---

## 🎯 项目目标

1. **吃透** — 深入研读日欧美三国/地区自己推崇的管理经典著作，理解其思想源头和核心逻辑
2. **升华** — 抽象出可迁移的底层原理，不被术语和表面形式困住
3. **运用** — 结合中国国情和本公司（自动化设备/制造领域）实际，转化出可落地的管理实践

## 🧠 研读方法论

本项目运用 **《计算你的语言》解压缩阅读法** 来啃每一本书。

### 阅读的本质

> **阅读 = 把作者压缩在文本中的向量表示，解压缩到读者的世界模型中，并选择性更新参数。**

一本书是作者几十年经验的高效压缩包。读的过程就是把它解压开，重建作者的认知结构，然后更新我们自己的世界模型。

### 三层递进：读到哪一层才算"吃透"

| 层次 | 阶段 | 你要做什么 |
|------|------|-----------|
| **第一层：他在说什么？** | ① 解码 | 认识字不算什么，要把文字转成语义 |
| | ② 模型重建 | 在脑子里重建作者的逻辑结构、因果链条 |
| **第二层：这改变了我什么？** | ③ 先验更新 | 读之前我怎么想 → 读之后我怎么想？变化在哪？ |
| | ④ 压缩与解压缩 | 识破作者的压缩策略——他省略了什么？放大了什么？ |
| **第三层：这能用来干嘛？** | ⑤ 共同推理 | 读到一半能预判后半段走向，跟作者并行思考 |
| | ⑥ 可迁移表示 | 读完了坐标系变了，这个框架能用来解释其他领域的问题 |

### 解压缩检查清单

每读完一本书，必须问自己这三个问题：

| # | 问题 | 回答标准 |
|---|------|---------|
| 1 | **作者的原始模型是什么？** | 一句核心论点 + 一条因果链 + 支撑材料 |
| 2 | **我的认知有什么更新？** | 读前prior → 读后posterior，变化在哪？ |
| 3 | **这个框架能迁移到哪？** | 能不能解释另一个领域？能不能实际用到公司？ |

**三个问题答不出来，等于没读。**

### 研读产出格式

每本书产出独立笔记：`notes/国别/序号_书名_作者.md`

包含：
- **基本信息**（作者、年代、类型）
- **作者的模型**（核心思想，用自己的话复现，3-5条）
- **解压缩分析**（他用什么压缩策略？省略了什么？）
- **先验更新记录**（我的prior→posterior变化）
- **经典摘录**（原文金句）
- **与中国文化对照**（共鸣点/冲突点/水土不服的地方）
- **可迁移的底层原理**（脱掉形式，留住本质）
- **运用方向**（可能的管理实践转化）
