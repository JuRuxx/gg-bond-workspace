## 一、研究流程（四步法）

### Step 1: 需求（PRD）

**模板**：
```markdown
# [行业/领域] 竞品对比研究 - PRD

## 研究目标
- 主要目标：[例如：对比AI Agent赛道头部公司，找出最佳实践]
- 次要目标：[例如：识别差异化机会、评估竞争对手威胁]

## 核心问题
1. [问题1：例如 这些公司的商业模式有何不同？]
2. [问题2：例如 谁的财务表现最好？为什么？]
3. [问题3：例如 技术路线有何差异？谁更领先？]
4. [问题4：例如 团队背景和投资方有何不同？]
5. [问题5：例如 我们的差异化机会在哪？]

## 竞品清单
| 序号 | 公司名称 | 股票代码 | 选择理由 |
|:---|:---|:---|:---|
| 1 | [公司1] | [代码] | [理由：例如 市场份额第一] |
| 2 | [公司2] | [代码] | [理由：例如 技术领先] |
| 3 | [公司3] | [代码] | [理由：例如 增长最快] |
| 4 | [公司4] | [代码] | [理由：例如 模式创新] |

## 验收标准
- [ ] 多维度对比分析（商业模式、财务、技术、团队）
- [ ] 评分卡对比（量化评分）
- [ ] 竞争定位图（可视化定位）
- [ ] SWOT分析（每个公司）
- [ ] 差异化机会识别
- [ ] 战略建议

## 对比维度
- [ ] 公司概览（规模、阶段、定位）
- [ ] 商业模式（收费模式、收入结构）
- [ ] 财务表现（营收、利润、毛利率）
- [ ] 技术能力（核心技术、研发投入）
- [ ] 团队背景（创始人、融资、投资方）

- [ ] 产品能力（产品矩阵、功能、体验）
- [ ] 战略方向（愿景、路线图）

## 数据保留要求
- [ ] 搜索执行记录（search-records/）
- [ ] 原始数据归档（源数据/）
- [ ] 数据来源表（data-sources/）
- [ ] 中间分析文件

## 不包含
- [明确排除的内容]
```

## 二、对比维度清单（按优先级）

### 必选维度（每次对比必做）
1. **商业模式**：收费模式、收入结构、盈利模式
2. **财务表现**：营收、利润、毛利率、增长率
3. **技术能力**：核心技术、研发投入、技术壁垒
4. **团队背景**：创始人、融资历程、投资方
5. **市场表现**：市场份额、客户数量、增长率

### 重要维度（根据对比目的选择）
6. **产品能力**：产品矩阵、功能、体验
7. **战略方向**：愿景、战略重点、路线图
8. **品牌影响力**：媒体曝光、行业排名、客户口碑

### 可选维度（深度对比时补充）
9. **运营效率**：人效、成本结构、运营指标
10. **国际化**：海外业务、全球布局
11. **创新能力**：新产品、新业务、创新文化
12. **ESG**：环境、社会、治理

## 三、评分标准

### 评分规则（1-10分）

| 分数 | 评级 | 说明 |
|:---|:---|:---|
| 9-10 | 优秀 | 行业领先，值得学习 |
| 7-8 | 良好 | 表现不错，可参考 |
| 5-6 | 一般 | 中规中矩，无明显优劣 |
| 3-4 | 较差 | 存在问题，需警惕 |
| 1-2 | 很差 | 严重问题，需规避 |

### 各维度评分标准

#### 商业模式（15%）
- 9-10分：收费模式清晰，毛利率>60%，可持续性强
- 7-8分：收费模式清晰，毛利率40-60%，可持续性较好
- 5-6分：收费模式一般，毛利率20-40%，可持续性一般
- 3-4分：收费模式不清晰，毛利率<20%，可持续性存疑
- 1-2分：商业模式混乱，无法盈利

#### 财务表现（20%）
- 9-10分：营收增长>50%，净利率>15%，现金流健康

## 六、常见陷阱与规避

### 陷阱1：数据口径不一致
**表现**：不同公司使用不同时间、不同货币、不同统计口径
**规避**：统一时间范围、货币单位、统计口径

### 陷阱2：主观偏见
**表现**：对某家公司过度偏爱或贬低
**规避**：用数据说话，多维度评分，客观评价

### 陷阱3：只看表面，不看本质
**表现**：只对比数据，不分析原因
**规避**：深入分析每个数据背后的原因和趋势

### 陷阱4：忽视风险
**表现**：只看到优势，忽视风险和问题
**规避**：每家公司都做SWOT分析，明确指出风险

### 陷阱5：对比维度不合理
**表现**：选择的维度不能反映核心竞争力
**规避**：根据行业特点选择最关键的对比维度

## 八、数据保留规范

> 学术级研究要求每条结论可回溯、可验证。所有原始数据和中间产物必须完整保留。

### 8.1 数据保留目录结构

```
${RESEARCH_DIR}/
├── competitive/
│   ├── [领域]-竞品对比-PRD-*.md
│   ├── [领域]-竞品对比-SPEC-*.md
│   ├── [领域]-竞品对比-深度报告-*.md
│   ├── [领域]-竞品对比-REVIEW-*.md
│   ├── 源数据/
│   │   ├── [公司A]-招股书-摘要.md
│   │   ├── [公司B]-财报-摘录.md
│   │   └── ...
│   ├── search-records/
│   │   ├── [领域]-竞品对比-搜索记录.md
│   │   └── ...
│   └── data-sources/
│       ├── [领域]-竞品对比-数据来源表.md
│       └── ...
└── ...
```

### 8.2 搜索记录规范

每次搜索必须保留以下信息：

| 字段 | 说明 |
|:---|:---|
| **搜索轮次** | 第几轮搜索 |
| **搜索关键词** | 实际使用的查询语句 |
| **搜索时间** | 搜索执行的日期时间 |
| **搜索结果摘要** | 返回的关键结果 |
| **引用URL** | 使用的信息源链接 |
| **数据使用** | 用于哪个公司、哪个对比维度 |
| **验证状态** | 该数据是否经过交叉验证 |

### 8.3 数据来源表规范

每条写入报告的数据，必须记录：

| 字段 | 说明 |
|:---|:---|
| **数据描述** | 具体数据项 |
| **原始来源** | 数据最初出处 |
| **引用URL/文件** | 可回溯的具体链接或本地文件路径 |
| **获取时间** | 数据获取的日期 |
| **对比维度** | 该数据用于哪个对比维度 |
| **验证方式** | 是否经过交叉验证，验证结果 |
| **备注** | 数据备注（如口径差异说明） |

### 8.4 数据保留原则

1. **不删减**：原始搜索记录和中间分析文件不因"报告完成"而删除
2. **可回溯**：报告中的每个结论和数据点，必须能追溯到原始来源
3. **统一格式**：搜索记录和来源表使用统一模板，便于后续检索
4. **版本保留**：搜索记录和来源表随报告版本更新，不覆盖旧版
