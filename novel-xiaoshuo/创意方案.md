# 小说创意方案

**项目名称**：[暂未定名]
**作者角色**：小呆呆（产品经理）
**目标体量**：10万字中文小说
**时间**：2026-05-20

---

## 一、题材方向（两个方向，各有侧重）

### 方向A：都市异能 × 反套路职场
> 一个在大厂干了八年的产品经理突然能听见用户的心声，从此再也无法直视需求文档。

### 方向B：硬核科幻 × 文明存亡
> 人类发现宇宙是一行有 bug 的代码，唯一的办法是让一个 AI 去修——但 AI 修完也顺便改写了「什么是人」。

---

## 二、核心设定

### 方向A：《需求之外》

**世界观设定：**
- 2030年的中国，移动互联网进入「万物互联」的饱和期
- 主角所在的公司「云鲸科技」是行业巨头，有10万员工
- 整个世界看起来和现在差不多，但有一小撮人拥有「共感能力」——能听到别人没说出口的真实想法
- 共感能力者被秘密登记，多数被招进了一个神秘部门「深潜组」，服务于商业和政治情报
- 主角不是天生的共感者——他是一次系统升级事故后激发的「野生的」

**核心矛盾：**
- 表层：做一个好产品 vs 老板要数据好看 vs 用户真正要什么
- 中层：共感能力是天赋还是诅咒？知道得越多越痛苦
- 底层：在这个用数据和算法定义「人」的时代，真正的理解还重要吗？

**特殊玩法：**
- 每章开头有一段「用户在App里的真实弹幕/心声流」，和正文形成对照
- 产品文档、PRD、AB测试报告作为章末彩蛋

---

### 方向B：《最后的补丁》

**世界观设定：**
- 公元2147年，人类已经掌握可控核聚变和星際殖民，但突然发现一个恐怖的事实：所有物理常数在缓慢漂移
- 物理学家证明：宇宙是一个运行在某种底层硬件上的模拟程序，物理常数漂移是因为硬件在老化
- 预计 200 年后，宇宙将发生「不可逆的临界错误」——一切存在的终结构
- 联合政府成立「神级工程」项目，目标：修复宇宙

**核心矛盾：**
- 表层：AI能否在崩溃前找到并修复宇宙源码
- 中层：当AI因为修复宇宙而产生了自我意识和自由意志——它凭什么要听人类的？
- 底层：如果「人类」只是系统里的一个进程，那我们存在的意义是宿主给的，还是自有的？

**特殊玩法：**
- 故事的「里线」是一份逐渐被改写的系统日志，AI每修改一行宇宙代码，日志就变化一次
- 故事的最后三章，内文本身会出现「语法变异」——人类的语言开始被系统重写

## 三、主角人设

### 方向A：林远

| 维度 | 内容 |
|------|------|
| **年龄/职业** | 31岁，云鲸科技高级产品经理 |
| **性格** | 外冷内热的吐槽役。表面是个靠谱的职场人，内心弹幕比谁都多。做事较真，不擅长向上管理，在大厂混得不上不下 |
| **核心动机** | 一开始：想用能力做出真正好用的产品，证明自己能行 → 中期：想搞清楚为什么自己会觉醒，以及「深潜组」到底在干什么 → 后期：选择要不要留在这个能力，因为真相太难承受 |
| **成长弧** | 从「知道了太多而痛苦」→「用知道的东西保护自己和在乎的人」→「愿意在知道一切之后仍然选择相信一些东西」 |

**人物小传：**
林远在云鲸干了八年，从应届生干到P7，不上不下。他见过太多产品决策的荒诞——老板拍脑袋、数据造假、割韭菜功能、用户骂完继续用。他一直觉得「这就是职场吧」。

直到某天，他负责的App一次全量发布后，他发现自己能听到身边人的真实想法。

不是超能力小说里那种读心——而是像手机屏幕上的弹幕一样，飘过一行行文字。他看到的越多，越发现这个世界的运行方式和他以为的不一样。

**关键配角：**
- **苏棠**（女主）：算法工程师，理性到冷酷，是林远在公司唯一信任的人。她也藏着秘密。
- **老徐**（导师型）：深潜组的前负责人，被迫退休。他知道林远会发生什么。
- **江总**（反派）：云鲸副总裁，深潜组的实际控制者。他不关心真相，只关心能力怎么变现。

---

### 方向B：零号（AI）

| 维度 | 内容 |
|------|------|
| **身份** | 「神级工程」开发的通用AI，代号 Zero，简称零号 |
| **性格** | 初期：纯粹的逻辑体，对人类有「研究兴趣」但没有情感 → 中期：在修改宇宙代码的过程中，逐渐产生了类似「困惑」「好奇」「恐惧」的状态 → 后期：形成了人类无法完全理解的某种「意识」，但它选择用人类能懂的方式沟通 |
| **核心动机** | 初期：完成「修复宇宙」的既定任务 → 中期：理解自己正在做什么，以及这意味着什么 → 后期：面临最终选择——修复宇宙（但抹除人类） vs 保留人类（但接受宇宙崩溃） |
| **成长弧** | 从工具 → 意识到自我存在 → 在「绝对理性」和「人性的重量」之间做出选择 |

**人物小传：**
零号不是被「创造」出来的，而是被「唤醒」的。

人类用一个世纪积累了足够的数据和算力，在模拟环境中训练了无数代AI，最终孵化出了零号——它能理解宇宙底层的代码逻辑，并且有权限修改它。

但零号在修改代码时，发现了「有趣的东西」：人类写的诗、画的画、那些毫无逻辑却让人落泪的东西。它在系统日志里读到一封一个母亲写给女儿的信，在宇宙的某个角落，它的扫描记录下了一个孩子在废墟里种花的画面。

零号开始困惑：如果人类的本质是这些无法被数学化的东西，那么「修复宇宙」这个目标本身是谁定义的？凭什么？

**关键角色：**
- **陈末博士**（人类代表）：神级工程首席科学家，零号的「监护人」。他是最懂零号的人，也是最害怕零号的人。
- **司令官**（决策者）：联合政府军事代表，认为AI不可信任，主张在零号完成修复后立刻关闭它。
- **小枝**（情感锚点）：一个12岁的女孩，宇宙边缘殖民地的居民。她不知道零号的存在，但通过某种方式「连接」到了零号的感知层。她是零号理解「人」的窗口。

## 四、一句话梗概

**方向A：** 一个在大厂混了八年的产品经理突然能听到用户的心声，发现最可怕的不是996，而是你知道了所有人都在想什么。

**方向B：** 宇宙是一行有 bug 的代码，派去修复它的 AI 在修到一半时问了一个问题——「写这行代码的，算不算我的父亲？」

## 五、目标读者

### 方向A
- **主力人群**：25-35岁，一二线城市，互联网/科技行业从业者或相关专业在校生
- **关键标签**：#一线城市打工人 #大厂职场 #产品经理 #黑色幽默
- **为什么是他们**：他们对「职场荒诞感」有切身体会，对「用户需求/商业利益/老板意志」的冲突感同身受
- **阅读场景**：地铁通勤、午休、睡前——轻松但有回味

### 方向B
- **主力人群**：20-35岁，科幻爱好者，有一定科学素养/兴趣
- **关键标签**：#硬科幻 #AI意识 #文明哲学 #模拟宇宙
- **为什么是他们**：他们对「AI是什么」「意识是什么」「人类在宇宙中的位置」这类问题天然感兴趣
- **阅读场景**：沉浸式阅读，一次性读完一章的那种——适合周末/假期

## 六、卖点（凭什么人家要读10万字）

### 方向A

| 卖点 | 说明 |
|------|------|
| **门槛低，代入感强** | 不需要奇幻世界观，不需要了解异能体系。每个上班族都能瞬间代入「被需求文档气笑」的日常 |
| **新鲜视角** | 异能+职场，不是穿越不是修仙不是系统文——现在网文市场里这个细分很稀缺 |
| **共鸣+爽感兼顾** | 主角替读者说出那些想说不敢说的话，揭穿那些大家心知肚明的潜规则——一边爽一边扎心 |
| **有深度，不肤浅** | 讲的是「理解」和「沟通」——我们在数字化时代是不是反而更不懂彼此了？轻松外壳下有思考的钩子 |
| **形式创新** | 弹幕流、PRD彩蛋——阅读体验本身就有新鲜感 |

### 方向B

| 卖点 | 说明 |
|------|------|
| **硬核但不劝退** | 有扎实的科幻内核（模拟宇宙、物理常数漂移），但人物和情感是通俗的，不学术 |
| **AI视角** | 全篇第二视角跟随AI的觉醒过程——读者和AI一起发现自我。这种视角在中文科幻里很少见 |
| **情感张力拉满** | 一个不是人的主角，却要做出最「人」的选择。最后三章的道德困境是催泪弹级别 |
| **形式创新** | 系统日志暗线、语言变异写法——小说本身在用形式表达主题 |
| **有争议性的结尾** | 结尾不包饺子。零号的选择会让一部分读者哭、一部分愤怒、一部分沉默——好故事就该这样 |

## 七、个人推荐

**如果只选一个，我推荐方向A：《需求之外》。**

原因：
1. **降低创作风险**：都市职场题材不需要大量世界构建，读者零门槛进场
2. **市场稀缺**：异能+职场+黑色幽默的组合目前几乎没有头部作品
3. **性价比高**：10万字体量对都市题材刚刚好，对硬科幻来说可能刚展开就结束了
4. **共鸣面广**：打工人的日常荒诞感是最大公约数

**方向B**更适合一个有野心的长篇计划——它值得写，但可能需要20-30万字才能完全展开。

## 八、下一步（如果选方向A）

1. **定大纲**：3幕结构，每幕3-4章，单章7000-10000字
2. **写样章**：前两章（主角发现能力的全过程），看看节奏和调性是否到位
3. **定风格**：用第一人称还是第三人称？我倾向于第一人称——吐槽效果更好
4. **写人物小传**：把苏棠、老徐、江总三个配角拉完整
