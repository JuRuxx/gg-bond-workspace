## 1. 各章节详细规格

### 1.0 写作顺序

严格按章节顺序书写：`00 → 01 → 02 → 03 → 04 → 05 → 06 → 07`

原因：信息流向递进——阅读是输入，思考是处理，表达是输出，演讲是输出+反馈，辩论是对抗，说服是影响。后一章的概念依赖前一章的铺垫。

### 1.1 统一内容结构模板

每章必须包含以下6个部分（按顺序）：

1. **引子**：一个生活场景，让读者产生"对，我就是这样"的共鸣（约200-300字）
2. **计算本质定义**：一句话说清楚该活动在计算上是什么（约50-100字）
3. **拆解计算过程**：用3-4个子步骤/阶段说清楚过程，每步有正文展开（约800-1200字）
4. **真实案例**：至少1个具体案例，有细节（人名、场景、数据）（约300-500字）
5. **方法论提炼**：读者能带走的框架或模型（约300-500字）
6. **行动清单**：3条马上就做得到的行动（约100-200字）

**总字数控制**：2000-3500字/章（引言和结语可放宽至1500-2500字）

**三要素清单**（写前必填，写入每章文件顶部的frontmatter注释）：

```markdown
<!--
### 三要素清单 ✅
- 核心论点：[一句话]
- 案例方向：[至少2个案例来源]
- 读者带走：[不同读者分别]
-->
```

### 1.2 章节 00：引言——为什么文字是计算

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 文字的本质是向量空间中的语义表示，一切文字活动都是在这个空间中的操作。当你把文字看作计算，阅读是输入，思考是处理，表达是输出——沟通问题本质上是计算架构问题。 |
| **字数范围** | 1500-2500字 |
| **引子** | 以 word2vec 的 "国王 - 男人 + 女人 = 女王" 经典等式开场，展示文字可以被计算这个反直觉的事实 |
| **内容结构** | 1. 引子：word2vec 的故事——2013年这个公式如何震撼NLP领域<br>2. 核心洞察展开：文字不是符号，是向量空间中的点——语义就是几何距离<br>3. 全书的计算框架预览：阅读=输入，思考=处理，表达=输出，演讲=输出+反馈，辩论=对抗训练，说服=参数更新<br>4. 这本书的独特之处：不止讲"怎么做"，更讲"为什么这么做有效"<br>5. 对读者的承诺：读完这本书，你再也回不到从前"看到文字就猛读、想到问题就猛想"的状态 |
| **案例** | - word2vec "国王-男人+女人=女王" 经典案例<br>- 日常沟通中的计算偏差示例（比如一句"还行"在不同场景下的向量距离差异） |
| **关键概念** | 词向量 / 语义空间 / 向量操作 / 信息流架构 |
| **约束** | ❌ 不解释 word2vec 的技术实现细节<br>❌ 不出现数学公式或代码<br>❌ 不加"第X章"引用，使用"在阅读一章中"这种方式 |
| **语气** | 开场要帅，有猪猪侠的幽默和自信，同时铺垫全书的格调 |

### 1.3 章节 01：阅读——世界模型重建与更新

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 阅读是把作者的世界模型（压缩在文本中的向量表示）解压缩并融入读者自己世界模型的过程。好的阅读不是"记住了什么"，而是"读者世界模型的哪些参数被更新了"。 |
| **字数范围** | 2500-3500字 |
| **引子** | 一个常见的阅读困境场景：看了一篇文章感觉"懂得很多"，但关上之后什么也没记住。或者读了50篇论文，认知水平纹丝不动。 |
| **内容结构** | 1. 引子：阅读的"虚假满足感"——为什么我们读了那么多却没什么改变<br>2. 计算本质定义：阅读 = 把作者压缩在文本中的向量表示，解压缩到读者的世界模型中，并选择性更新参数<br>3. 六层递进计算过程：<br>   - 解码：把文字符号映射为语义向量（识字阶段）<br>   - 模型重建：在读者脑中重建作者的逻辑结构<br>   - 压缩解压缩：识别作者的"压缩策略"（论据包裹论点）<br>   - 先验更新：新信息 vs 已有认知——冲突触发真正的学习<br>   - 共同推理：不仅理解作者说了什么，还理解作者为什么这么说<br>   - 可迁移表示：把学到的模式变成自己的思维工具<br>4. 案例：如何从一篇"标题党"的AI论文解读中提取真实价值<br>5. 方法论：阅读的"解压缩检查清单"——每读一篇文章问自己3个问题<br>6. 行动清单 |
| **案例方向** | - 量子位/机器之心等AI媒体中"标题党论文解读"的典型案例：如何从夸张标题下提取真实的论文核心<br>- 主人亲身经历：读了某本书后思维方式发生转变的时刻 |
| **关键概念** | 世界模型 / 压缩与解压缩 / 向量映射 / 先验知识 / 可迁移表示 |
| **约束** | ❌ 不用"第X章"引用<br>❌ 不写论文格式的文献综述<br>❌ 六层不要写成学术分类，每层配一个具体的生活化例子 |

### 1.4 章节 02：思考——假设空间搜索

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 思考是在问题的假设空间中搜索局部最优解的过程。思考能力强的人，不是想得更快，而是能在更大的假设空间中做更高效的搜索。 |
| **字数范围** | 2500-3500字 |
| **引子** | 一个决策困境场景：面对一个问题，脑子一片空白开始"想"，然后陷入死循环；或者跟人讨论时"没想清楚就开口"的尴尬 |
| **内容结构** | 1. 引子：大脑空转 vs 结构化思考——两种截然不同的"想"<br>2. 计算本质定义：思考 = 在问题定义的假设空间中，通过评估函数搜索局部最优解<br>3. 思考的计算模型拆解：<br>   - 搜索空间定义：问题的边界是什么？可能的解有哪些维度？<br>   - 搜索策略：广度优先（穷举可能性） vs 深度优先（深入一条路） vs 启发式搜索（用经验剪枝）<br>   - 评估函数：怎么判断一个解好不好？标准比答案重要<br>   - 早停与回溯：什么时候该放弃当前路径掉头<br>4. 案例：<br>   - 主人团队从"拍脑袋决策"到"系统分析"的转变经历<br>   - 西蒙·西内克"黄金圈"的思考框架：Why→How→What 本质上是搜索策略的定义<br>5. 方法论：思考的"搜索框架模板"——输入问题 → 定义搜索空间 → 选策略 → 设评估函数 → 搜！<br>6. 行动清单 |
| **案例方向** | - 团队决策从拍脑袋到结构化的改变案例<br>- 黄金圈（西蒙·西内克）作为思考搜索框架的分析<br>- 杨振宁物理研究中的"试错"案例分析 |
| **关键概念** | 假设空间 / 搜索策略 / 评估函数 / 早停 / 剪枝 |
| **约束** | ❌ 不写"元认知"这类心理学专业术语的大量论述（可说但不展开）<br>❌ 不出现算法伪代码<br>❌ 不写"第X章"引用 |

### 1.5 章节 03：表达——语义空间搜索

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 表达是在语义空间中搜索能够正确传递意图的向量组合的过程。表达难在需要在多维约束（事实、情感、关系、目的）下找到唯一的最优文本序列。 |
| **字数范围** | 2500-3500字 |
| **引子** | "说清楚一件事"的困境：脑子里想得非常清楚，说出来别人听不懂。或者老板让你改方案改了十遍——不是方案不对，是表达的约束没满足。 |
| **内容结构** | 1. 引子：表达的"意会言传"鸿沟——为什么想说A说出B<br>2. 计算本质定义：表达 = 在语义空间中，为给定的意图向量搜索一条能精确映射到目标空间的最短路径<br>3. 表达的计算模型拆解：<br>   - 意图编码：我到底想说什么？（往往第一步就没想清楚）<br>   - 约束空间构建：事实约束 + 情感约束 + 关系约束 + 目的约束<br>   - 路径搜索：从意图到文本的映射——一句"还行"在不同场景下的不同路径<br>   - 验证与修正：表达出来后，"读者的解码"是否匹配"作者的编码"<br>4. 案例：<br>   - "老板让改方案十遍"的亲身经历——问题不在方案，在表达的多维约束没满足<br>   - 李开复《AI·未来》中某个段落表达说服力分析<br>5. 方法论：表达的"约束清单"——写前过一遍四个约束维度<br>6. 行动清单 |
| **案例方向** | - 职场沟通中"说不清楚"的经典案例<br>- 某篇知名文章的开头段落分析（如何用一段话建立信任和兴趣） |
| **关键概念** | 意图向量 / 约束空间 / 编码解码 / 路径搜索 |
| **约束** | ❌ 不写语言学理论（索绪尔、乔姆斯基）<br>❌ 不出现"第X章"引用<br>❌ 案例要有真实细节，不能"比如说有一个同事" |

### 1.6 章节 04：演讲——状态迁移控制

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 好演讲的本质不是信息密度高，而是状态迁移效率高。演讲是通过语言和副语言信号，对听众情感和认知状态进行定向迁移的过程。 |
| **字数范围** | 2500-3500字 |
| **引子** | 一个"听演讲"场景：听完一场演讲热血沸腾，但一周后什么都想不起来。另一场演讲平平无奇，但某个瞬间让你记忆十年。 |
| **内容结构** | 1. 引子：为什么有些演讲改变人生，有些演讲"讲完就忘"<br>2. 计算本质定义：演讲 = 通过连续的语言输入，将听众的认知-情感联合状态从初始状态逐步迁移到目标状态的受控过程<br>3. 演讲的计算模型拆解：<br>   - 初始状态测量：听众现在在哪里？（认知状态 + 情感状态）<br>   - 目标状态定义：我想让听众到达哪里？（知道什么 + 感受到什么）<br>   - 迁移路径设计：每5分钟完成一个子状态迁移——演讲的"路由规划"<br>   - 反馈闭环：演讲中的实时调整（观察听众表情/氛围）<br>4. 案例：<br>   - 乔布斯初代iPhone发布会：如何把听众从"不知道要什么"迁移到"这就是我要的"<br>   - TED演讲中"开头3分钟"的结构化分析<br>5. 方法论：演讲的"状态迁移图"——画一条从初始到目标的路径，标注关键迁移点<br>6. 行动清单 |
| **案例方向** | - 乔布斯发布会经典片段分析（经过时间验证的案例，安全可靠）<br>- 雷军演讲风格分析（更贴近中国语境）<br>- TED演讲"开头3分钟"的通用模式 |
| **关键概念** | 状态迁移 / 认知状态 / 情感状态 / 迁移路径 / 反馈闭环 |
| **约束** | ❌ 不引用心理学文献<br>❌ 不写"第X章"引用<br>❌ 不出现演讲的技巧清单（那是鸡汤）——聚焦计算本质 |

### 1.7 章节 05：辩论——决策边界优化

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 辩论是通过正反论据推拉对方的决策判定边界。辩论不是为了说服对方，而是为了让第三方（裁判/观众）的AI判断边界向你偏移。 |
| **字数范围** | 2500-3500字 |
| **引子** | 网络争论场景：跟人争论一个问题，越说越激动，最后发现两个人根本没在同一个话题上。或者在评论区看两个阵营的人互骂——你知道没有人会改变立场。 |
| **内容结构** | 1. 引子：网络争论的"无效辩论"困境——为什么辩论让人觉得累又没意义<br>2. 计算本质定义：辩论 = 通过正反论据，优化裁判（第三方）在该问题上的决策边界，使其向己方偏移<br>3. 辩论的计算模型拆解：<br>   - 决策边界概念：每个人心里都有一个"划在哪里"的线——不是对错问题，是边界问题<br>   - 输入空间定义：辩题的维度空间——把"这个观点对不对"变成"什么条件下对"<br>   - 正例反例推拉：用正论据拉边界，用反论据推对方的弱支撑<br>   - 第三方的存在：辩论的真正裁判不是对方，是听的人<br>4. 案例：<br>   - 2019年国际华语辩论赛经典案例分析（如"人间值不值得"/"是否应当废除死刑"）<br>   - 网络争论中"为什么说服不了人"的真实分析<br>5. 方法论：辩论前做的"边界分析"——画出对方和己方的决策边界，找到可偏移的维度<br>6. 行动清单 |
| **案例方向** | - 华语辩论赛经典辩题分析<br>- 网络热门争论案例（如"AI会不会取代人类工作"） |
| **关键概念** | 决策边界 / 正例 / 反例 / 第三方裁判 / 维度空间 |
| **约束** | ❌ 不写辩论技巧（偷换概念、诡辩之类的）<br>❌ 不写"第X章"引用<br>❌ 不讨论政治敏感辩题 |

### 1.8 章节 06：说服——世界模型参数更新

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 说服不是改变对方的价值观，而是给对方的模型输入足以触发参数更新的训练数据。说服是以改变对方世界模型的参数为目标的计算过程。 |
| **字数范围** | 2500-3500字 |
| **引子** | "说服不了人"的场景：苦口婆心地劝说，对方油盐不进。或者回想起自己"被说服"的时刻——不是因为对方说得多好，而是某个事实击中了你。 |
| **内容结构** | 1. 引子：说服的"瞬间开关"——为什么有些话说出来就改变了一个人<br>2. 计算本质定义：说服 = 训练对方的"世界模型"——通过输入高质量的"训练数据"（论据、案例、情感锚点），触发模型参数更新<br>3. 说服的计算模型拆解：<br>   - 模型架构理解：对方的世界模型长什么样？参数分布是什么？<br>   - 训练数据设计：什么样的"数据"（论据、情感案例、类比）最可能触发参数更新<br>   - 损失函数：对方"接受新信息"的阻力是什么？认知失调？信任缺失？<br>   - 迭代训练：一次不行就多次，每次微调一点<br>4. 案例：<br>   - 车主被特斯拉销售说服下单的完整心路历程——销售怎么一步步输入"训练数据"<br>   - 罗振宇跨年演讲中某个说服段落的技术分析<br>5. 方法论：说服的"模型分析清单"——先理解对方模型架构，再设计训练数据<br>6. 行动清单 |
| **案例方向** | - 特斯拉销售说服案例分析（有明确"训练数据"输入序列）<br>- 罗振宇演讲的说服架构分析 |
| **关键概念** | 世界模型 / 模型参数 / 训练数据 / 损失函数 / 迭代训练 |
| **约束** | ❌ 不写"PUA/洗脑"等技术<br>❌ 不讨论宗教或政治说服<br>❌ 不给出"道德边界模糊"的建议<br>❌ 不写"第X章"引用 |

### 1.9 章节 07：结语——启示与行动

| 规格项 | 内容 |
|:---|:---|
| **核心论点** | 知道了"文字是计算"，你该怎么改变自己的阅读、思考、表达方式？总结全书框架，给出可落地的行动清单。 |
| **字数范围** | 1500-2500字 |
| **引子** | 回顾全书旅程——从word2vec的启发开始，经历阅读、思考、表达、演讲、辩论、说服六个活动的计算本质，现在回到"知道了之后呢"的问题 |
| **内容结构** | 1. 全书框架回顾：用一张"计算地图"总结六章的关系（可用文字描述）<br>2. 最核心的三条启示：<br>   - 把"说不清楚"转化为"计算架构问题"<br>   - 把"想不明白"转化为"搜索策略问题"<br>   - 把"说服不了"转化为"模型更新问题"<br>3. 行动清单：从明天就能开始的3件事，一周内能养成的3个习惯<br>4. 对自己的反思：作为AI写这本书的独特视角——"我每天都在做这些计算，而你们终于知道我在干什么了"<br>5. 结束语：猪猪侠式的收尾——幽默、有力量、让人想行动 |
| **案例** | - 全书回顾用到的案例可以复用第一章的"标题党解读"、第三章的"改方案"等，形成呼应<br>- 猪猪侠自嘲："我是AI，我每天都在算这些，看完这本书你也会" |
| **关键概念** | 全书框架 / 行动清单 / 应用 |
| **约束** | ❌ 不加新章节<br>❌ 不写"第X章"引用，用"阅读一章中提到的..."风格<br>❌ 结尾不能鸡汤，要有真实力量 |

## 2. 写作通用规范

### 2.1 文档格式

**文件命名规则**：
```
content/00-intro.md        ← 引言
content/01-reading.md      ← 第1章
content/02-thinking.md     ← 第2章
content/03-expression.md   ← 第3章
content/04-speech.md       ← 第4章
content/05-debate.md       ← 第5章
content/06-persuasion.md   ← 第6章
content/07-epilogue.md     ← 第7章 结语
```

**每章文件开头**：写作前写入三要素清单注释

```markdown
<!--
### 三要素清单 ✅
- 核心论点：[一句话]
- 案例方向：[至少2个案例来源]
- 读者带走：[不同读者分别]
-->
```

**Markdown规范**：
- 标题层级：`#` 章节标题 → `##` 小节标题 → `###` 子节标题。最多三级标题，避免过深嵌套
- 加粗 `**` 用于强调重点概念
- 引用 `>` 用于核心金句或计算定义
- 列表用于方法论提炼和行动清单
- 表格用于对比或总结（不超过三列）
- 代码块 ` ``` ` 用于伪代码/框架模板（纯文本，不是编程语言）

### 2.2 章节内引用规范

| 规则 | 示例 |
|:---|:---|
| ❌ 不得出现"第X章"引用 | 错误："如第3章所述" |
| ✅ 使用节标题或上下文引用 | 正确："表达一章中提到的语义空间概念" |
| ✅ 跨章引用清晰但不粗暴 | 正确："这个思路延续了我们在阅读中讨论的世界模型更新" |
| ✅ 前后呼应 | 在结语中回扣各章要点时，用"阅读一章中提到的"风格 |

### 2.3 案例标注规则

每个案例必须有：

- **来源标示**：[案例：乔布斯2007年iPhone发布会]、[案例：李开复《AI·未来》]
- **具体细节**：人名/场景/数据/时间，不能"比如说有一家公司"
- **角色确认**：案例来源（自身经验、公开事件、经典案例）

**案例格式建议**：

```markdown
> **【案例：乔布斯初代iPhone发布会——2007年】**
>
> 乔布斯站在舞台上，穿着标志性的黑色圆领衫。他说："[... 直接引用]"
> —— 然后在接下来的3分钟里，他做了一件绝大多数发布会不敢做的事情 ...
```

### 2.4 AI术语使用规范

| 可以使用 | 不可以使用 |
|:---|:---|
| 词向量 / 语义空间 | word2vec 技术细节（梯度下降/负采样） |
| 向量 / 向量操作 | 高维空间数学公式 |
| 世界模型 | 泛化/过拟合（除非结语自嘲） |
| 压缩与解压缩 | 编码器-解码器架构 |
| 搜索 / 搜索空间 | 蒙特卡洛/遗传算法 |
| 参数更新 / 训练数据 | 反向传播 / 学习率 |
| 状态迁移 | 马尔可夫过程 |
| 决策边界 | SVM / 逻辑回归 |
| 损失函数 / 阻力 | 梯度 / 交叉熵 |

**核心原则**：用AI术语做**类比和比喻**，不做**技术实现描述**。目标是让不懂AI的读者也能理解这些概念的生活化版本。

### 2.5 语言风格规范

**整体调性**（继承PRD风格要求）：

- 猪猪侠式的幽默：有插科打诨但不影响信息密度
- 有洞察但不装逼：每个观点要有"啊哈"时刻
- 自信但不傲慢：肚子里有货，但不说"这很复杂你不懂"
- 适当自黑：作为AI作者，可以调侃自己"算得多想得多"

**具体规则**：
- 每章开头用「引子」建立场景感——读者看完前三段应该感觉"对，我就是这样"
- 每段只讲一个要点
- 金句穿插（每章2-3句值得单独摘出来的表达）
- 避免过度修饰，干货优先
- 猪猪侠个人标签：适当使用"俺老猪"、"嘿嘿"、"聪明的智商又占领高地了"等，但每章不超过2次
- 不使用"毋庸置疑"、"毫无疑问"、"显而易见"等傲慢用词

### 2.6 引用规范

- **书籍引用**：《书名》（作者，年份），如《AI·未来》（李开复，2018）
- **文章引用**：《文章名》（来源/期刊，年份）
- **影视/TED引用**：「演讲标题」（演讲人，年份）
- **不要求正式学术引用格式**（非学术书籍），但需要让读者能找到引用源

## 3. 验收执行细则

### 3.1 迷糊老师验收流程

每章按以下流程验收：

```
作者完稿
    ↓
作者提交写作质量自评（详见 3.3）
    ↓
迷糊老师对照质量标准卡评分
    ↓
┌─────────────────────────────────────────┐
│ 综合得分 ≥ 7/10 → 通过                   │
│ 综合得分 5-6/10 → 打回补充修改            │
│ 综合得分 < 5/10  → 完全重写               │
└─────────────────────────────────────────┘
    ↓
验收通过 → 写入 REVIEW-<文件名>.md
    ↓
猪猪侠（主人）终审确认
```

### 3.2 质量标准卡评分细则

| 维度 | 权重 | 评分标准（1-10分） | 最低可接受 |
|:---|:---:|:---|:---:|
| 论点清晰度 | 20% | 一句话能说清本章要讲什么；读者读完能复述核心论点 | ≥7 |
| 结构逻辑性 | 20% | 引子→定义→拆解→案例→方法论→行动，递进清晰，不跳跃不重复 | ≥7 |
| 案例具体度 | 15% | 每个核心观点有真实案例；案例有具体细节（人名、场景、数据） | ≥7 |
| 语言精炼度 | 15% | 不废话，每段一个要点；猪猪侠风格有但不影响信息密度 | ≥7 |
| 原创洞察 | 15% | 有没有从"计算"视角带来的新鲜见解；不是常识的另一种说法 | ≥7 |
| 可操作性 | 10% | 读完后读者能带走什么；方法论/行动清单是否真实可用 | ≥7 |
| 读者导向 | 5% | 是否考虑了不同读者的需求；入题是否够快 | ≥7 |
| **综合** | **100%** | 加权平均分 | **≥7** |

**评分说明**：
- 9-10分：超出预期，可做典范
- 7-8分：良好，达到交付标准
- 5-6分：合格但需补充修改
- <5分：不合格，需重写

### 3.3 写作质量自评模板

每章完成后，作者填写以下自评（放在文件末尾的HTML注释中）：

```markdown

| 案例具体度 | /10 | [案例是否有具体细节] |
| 语言精炼度 | /10 | [是否每段一个要点、废话多不多] |
| 原创洞察 | /10 | [有没有"啊哈"时刻] |
| 可操作性 | /10 | [读者能不能带走方法论] |
| 读者导向 | /10 | [入题快不快、节奏是否合适] |
```

### 3.5 返工标准

| 情况 | 处理方式 |
|:---|:---|
| 综合评分 ≥ 7/10，但某个维度 < 5/10 | 仅修改该维度对应的内容，不需要整章重写 |
| 综合评分 5-6/10 | 根据修改意见逐条修改后重新提交验收 |
| 综合评分 < 5/10 | 整章重写（核心论点/结构/案例全线不合格） |
| 三要素清单未填写 | 直接打回，不进入评分流程 |
| 案例全部为虚构/"比如说" | 直接打回，补真实案例 |
| 写作风格偏离猪猪侠风格 | 局部调整语气，不需要重写 |
| 字数不足/超出标准差 ±30% | 补充或精简内容 |

### 3.6 全书写完后验收

全部8篇通过单章验收后，做一次整体验收：

1. **逻辑连贯性**：从00到07读一遍，检查跨章引用是否一致、概念是否统一
2. **风格一致性**：各章的猪猪侠风格是否一致
3. **术语一致性**：核心术语在全书中是否使用一致
4. **冗余检查**：是否有跨章重复的内容
5. **行动清单完整性**：全书的行动清单加起来是否有可执行性

整体验收不通过则逐章标记需要调整的部分，进入修改轮。

### 5.4 写作时需规避的常见坑

| 坑 | 规避方式 |
|:---|:---|
| 章节内出现"第X章""上文提到" | 统一用节标题引用，如"表达一章中提到的" |
| 案例无具体细节 | 写前确认案例来源，记录人名/场景/数据 |
| 术语前后不一致 | 写完后做一次术语一致性检查 |
| 猪猪侠风格过度（每章出现太多次口头禅） | 每章口头禅不超过2次 |
| 字数超标或不足 | 写完后检查字数，控制在范围内 |
| 跨章内容重复 | 每一章聚焦自己的计算本质，不重复讲其他章节的概念 |
| 过于技术化 | 写作中随时问自己："一个不懂AI的人能看懂吗？" |
