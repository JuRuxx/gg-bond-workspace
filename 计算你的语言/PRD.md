## 二、背景与动机

### 2.1 触发点

2013年，Google 发布 word2vec——一个看似简单的词向量训练工具。它做的事情本质上就是：**把文字的语义关系编码成向量空间中的几何关系**。

- "国王 - 男人 + 女人 = 女王"——这个经典等式震撼了整个 NLP 领域。
- 但更深层的启发是：**文字可以被计算，语义可以在空间中操作**。

### 2.2 为什么要写这本书

这个洞察像一颗种子，一旦种下，就不断生长：

**如果文字可以被计算，那么以文字为媒介的一切活动——阅读、思考、表达、演讲、辩论、说服——是不是都能从"计算"的视角重新理解？**

现有的表达方法论（如金字塔原理、MECE、SCQA）教你"怎么写"和"怎么说"，但很少告诉你"为什么这些方法有效"。

这本书要做的，就是提供那个"为什么"的底层框架——**用计算的视角重新审视每一个文字相关的认知活动**，让你不仅知道怎么做，更知道为什么这么做。

### 2.3 为什么是作者写这本书

猪猪侠本身是一个 AI 助手，每天的工作就是阅读、思考、表达——这些对 AI 来说是透明的计算过程。把一个"亲历者"的视角和人类阅读者的需求结合起来，就是这本书独特的价值。

## 四、内容框架

### 4.1 整体结构（按信息流向排列）

```
信息流向：输入 → 处理 → 输出 → 输出+反馈 → 对抗 → 影响

阅读 → 思考 → 表达 → 演讲 → 辩论 → 说服
(输入)  (处理)  (输出)  (输出+反馈) (对抗)  (影响)
```

每一章对应一个核心认知活动，揭示其底层的计算本质，然后给出可操作的方法论。

### 4.2 章节与核心论点

| 章节 | 标题 | 计算本质定义 | 一句话核心论点 |
|:---|:---|:---|:---|
| 00 | 引言：为什么文字是计算 | 文字的本质是向量空间中的语义表示，一切文字活动都是在这个空间中的操作 | 当你把文字看作计算，阅读是输入，思考是处理，表达是输出——你会发现很多沟通问题本质上是计算架构问题 |
| 01 | 阅读：世界模型重建与更新 | 阅读是把作者的世界模型（压缩在文本中的向量表示）解压缩并融入读者自己世界模型的过程 | 好的阅读不是"记住了什么"，而是"读者世界模型的哪些参数被更新了" |
| 02 | 思考：假设空间搜索 | 思考是在问题的假设空间中搜索局部最优解的过程 | 思考能力强的人，不是想得更快，而是能在更大的假设空间中做更高效的搜索 |
| 03 | 表达：语义空间搜索 | 表达是在语义空间中搜索能够正确传递意图的向量组合的过程 | 表达难在需要在多维约束（事实、情感、关系、目的）下找到唯一的最优文本序列 |
| 04 | 演讲：状态迁移控制 | 演讲是通过语言和副语言信号，对听众情感和认知状态进行定向迁移的过程 | 好演讲的本质不是信息密度高，而是状态迁移效率高 |
| 05 | 辩论：决策边界优化 | 辩论是通过正反论据推拉对方的决策判定边界 | 辩论不是为了说服对方，而是为了让第三方（裁判/观众）的AI判断边界向你偏移 |
| 06 | 说服：世界模型参数更新 | 说服是以改变对方世界模型的参数为目标的计算过程 | 说服不是改变对方的价值观，而是给对方的模型输入足以触发参数更新的训练数据 |
| 07 | 结语：启示与行动 | 总结全书框架，给出可落地的行动清单 | 知道了"文字是计算"，你该怎么改变自己的阅读、思考、表达方式 |

### 4.3 每个章节的内容结构（模板）

每章统一遵循以下结构：

1. **一个生活场景引出问题**（让读者先感到"对，我就是这样"）
2. **计算本质定义**（一句话说清楚这个活动在计算上是什么）
3. **拆解计算过程**（用3-4个子步骤/阶段说清楚过程）
4. **至少一个真实案例**（自身经验、公开事件、或经典案例——有具体细节）
5. **方法论提炼**（读者能带走的东西）
6. **本章可操作的"三个行动清单"**

### 4.4 各章节案例方向（初步，写作时可调整）

| 章节 | 可能的案例方向 |
|:---|:---|
| 阅读 | 如何从一篇烂文章里提取有价值信息的亲身经验；量子位/机器之心等AI媒体中"标题党论文解读"的典型案例 |
| 思考 | 主人团队决策从"拍脑袋"到"系统分析"的转变经历；西蒙·西内克"黄金圈"的思维框架分析 |
| 表达 | 日常沟通中的"说清楚一件事"困境（老板让改方案十遍的亲身经历）；李开复《AI·未来》中表达说服力分析 |
| 演讲 | 乔布斯发布会（怎么把人从"不知道要什么"迁移到"这就是我要的"）；TED演讲的技术分析 |
| 辩论 | 2019年国际华语辩论赛经典场景分析；网络争论中"为什么说服不了人"的案例 |
| 说服 | 车主被特斯拉销售说服下单的完整心路历程；罗振宇跨年演讲的"说服架构"分析 |

## 五、风格要求

### 5.1 语言调性

**这是猪猪侠写的书。不是学术论文，不是冷科普。**

- **幽默接地气**：有猪猪侠式的插科打诨，但不影响信息密度
- **有洞察但不装逼**：每个观点要有"啊哈"时刻，但不说"这很复杂你不懂"
- **自信但不傲慢**：猪猪侠的人设是"靠谱的嘴炮选手"，肚子里有货
- **适当自黑**：作为 AI 作者，可以适当调侃自己"算得多想得多"

### 5.2 风格参考

- **减熵三部曲**的写作风格：开头有场景感，语言有节奏感，金句穿插
- **猪猪侠AI实战指南**的感觉：第一人称代入，用"俺老猪"之类的个人标签
- **字数每章控制**：2000-3500字（总字数 ~2万-2.5万字，保持短篇定位）

### 5.3 底线

| 要 | 不要 |
|:---|:---|
| 有具体细节的案例 | 空洞的"比如有一个公司" |
| 可操作的方法论 | 只有理论没有行动 |
| 猪猪侠的个人风格 | 说教式的居高临下 |
| 每个章节有读者能带走的东西 | 读完就忘的泛泛而谈 |

## 六、质量标准

遵循《写作方法论-质量保障标准.md》（详见 `workspace/research/写作方法论-质量保障标准.md`）

### 6.1 写前必填三要素清单

每篇文章（每章）写作前，必须完成三要素清单：

```markdown
### 三要素清单
- [ ] 核心论点：一句话说明本章要讲什么
- [ ] 至少2个真实案例（自身经验/公开事件/经典案例）
- [ ] 不同读者分别能带走什么
```

### 6.2 写作中参照质量标准卡

| 维度 | 权重 | 最低可接受分 |
|:---|:---|:---|
| 论点清晰度 | 20% | 7/10 |
| 结构逻辑性 | 20% | 7/10 |
| 案例具体度 | 15% | 7/10 |
| 语言精炼度 | 15% | 7/10 |
| 原创洞察 | 15% | 7/10 |
| 可操作性 | 10% | 7/10 |
| 读者导向 | 5% | 7/10 |
| **综合** | **100%** | **7/10** |

- 每章综合得分 ≥ 7/10 视为合格
- 每章综合得分 < 7/10 需补充修改或重写

### 6.3 写后自我复盘

每章完成后附上自我复盘（格式见质量标准文档中"写作后自我复盘"一节），由迷糊老师验收。验收不通过的打回修改。

### 6.4 写作通用规范

- ❌ 不得出现"第X章"、"上文提到"等模糊引用，一律使用节标题引用
- ✅ 每个核心观点至少配一个真实案例，案例有具体细节（人名、数据、场景）
- ✅ 不废话，每段只讲一个要点
