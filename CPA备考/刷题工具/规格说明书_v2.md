# CPA 刷题工具 · MVP 规格说明书 v2

> 2026-06-11 主人明确：**先做一个简单的 MVP，不需要功能全。**  
> 以下是经过规整后的完整规格，每一条都是确定下来的。

---

## 零、MVP 范围定义

**只做一个功能：章节练习 + 模拟考试**

不做错题本，不做收藏题，不做多科目，不做统计图表。

数据从 `localStorage` 读取题目，答题过程中自动记录进度（正确/错误），但 MVP 不提供独立入口查看错题记录。

---

## 一、技术选型

| 项 | 值 |
|:---|:----|
| 架构 | 纯前端 HTML 单页应用 |
| 依赖 | **零依赖**，双击 `index.html` 即可用 |
| 存储 | `localStorage`（题目数据、答题记录） |
| 科目 | 公司战略与风险管理（仅此一科） |
| 响应式 | 适配桌面浏览器，不做移动端 |

## 二、题库数据规格

### 2.1 题目结构

每道题一个对象，硬编码在 HTML 的 `<script>` 标签内的 `questions` 数组中：

```javascript
{
  id: 1,                  // 唯一编号，从1开始递增
  chapter: "第一章",      // 章名称
  question: "题干文字",    // 纯文本，不支持HTML
  options: ["选项A", "选项B", "选项C", "选项D"],
  answer: 0,              // 正确选项的下标（0=A, 1=B, 2=C, 3=D）
  analysis: "解析文字"     // 纯文本，不支持HTML
}
```

**与 v1 规格书的差异：**
- 去掉 `section` / `type` / `difficulty` 字段（MVP 不区分）
- 去掉多选支持（MVP 只做单选题）
- `options` 从对象数组改为字符串数组
- `answer` 从字母改为数字下标（更简单）
- `question` / `analysis` 改为纯文本（题目数据本身不需要 HTML）

### 2.2 题目数量

MVP 内置 **30 道** 公司战略与风险管理模拟题，覆盖教材 8 章的常见考点，每题选自真实考点。

### 2.3 章节映射

每道题的 `chapter` 字段对应教材章节，可直接按"第一章~第八章"筛选。

## 六、章节练习（路由：practice）

### 6.1 流程

```
[步骤1] 选择章节（"请选择题章"下拉框 + "开始练习"按钮）
    │
    ▼
[步骤2] 刷题界面（题号栏 + 题目 + 选项 + 确认按钮 + 解析区域）
    │
    ▼
[步骤3] 做完该章全部题目，或点击"退出"返回首页
```

### 6.2 步骤 1：选章页

- 下拉框：列出"第一章"~"第八章"
- 按钮："开始练习"（选中章节后可用）
- 下拉框默认显示"请选择章节"，选中后点按钮进入刷题

### 6.3 步骤 2：刷题界面

**布局：**

```
┌─────────────────────────────────┐
│  ← 返回（右上角）                │
│  第一章（当前章节标题）           │
│  进度：5/10（已答/总题数）        │
├─────────────────────────────────┤
│  [1] [2] [3] [4] [5] [6] ...     │ ← 题号栏，按顺序排列
├─────────────────────────────────┤
│                                  │
│  题目内容文字...                  │
│                                  │
│  ○ A. 选项文字                   │
│  ○ B. 选项文字                   │
│  ○ C. 选项文字                   │
│  ○ D. 选项文字                   │
│                                  │
│  [  确认答案  ]  [⭐ 收藏]       │
│                                  │
│  ┌────────────────────────┐     │
│  │ 解析区域（答完后显示）   │     │
│  │ ✅ 回答正确 / ❌ 错误   │     │
│  │ 解析文字...              │     │
│  └────────────────────────┘     │
│                                  │
│  [ 上一题 ]        [ 下一题 ]   │
└─────────────────────────────────┘
```

**题号栏规则：**
- 从左到右排列该章全部题号（1~N）
- **未答题号**：灰色底，白色字
- **已答正确**：绿色底，白色字
- **已答错误**：红色底，白色字
- **当前题**：蓝色高亮边框

**答题交互规则：**
1. 用户点击一个选项 → 该选项边框变蓝（选中态），其余选项恢复默认
2. 已选选项后，**[确认答案]** 按钮从禁用变为可用
3. 点击 **确认答案** → 区域显示✅/❌ 标记 + 解析文字
4. 确认后 **不可再改选**，上一题/下一题按钮可以切换
5. 点击 **下一题** 自动按题号顺序走，已确认的题不再重新显示
6. 答错自动记录（同步到 localStorage）

**收藏按钮：**
- 答对的题可点击收藏（收藏状态用 ⭐/☆ 表示）
- 收藏数据存到 `localStorage`
- MVP 不提供独立收藏页入口，但是如果有收藏数据会保留

**上一题/下一题：**
- 第一题时"上一题"禁用
- 最后一题时"下一题"变成"完成"按钮
- 点击"完成" → 回到章节选择页

**「重新开始」按钮：**
- 位于章节标题旁边（小按钮）
- 点击弹窗确认：**「确定重新开始本章节？所有答题记录将清除」**
- 确认后清空该章进度数据，回到该章第1题

### 6.4 退出

- 任何时候点击左上角"← 返回"回到首页
- 进度的数据持久化保留

## 七、模拟考试（路由：exam）

### 7.1 流程

```
[点击"模拟考试"] → 进入考试
    │
    ▼
[考试中：随机30题，倒计时45分钟]
    │
    ▼
[交卷（手动/自动）] → 成绩报告
```

### 7.2 进入考试

- 点击首页的"模拟考试" → 直接从所有题库中随机抽取 **30 道**（无重复）
- 题目顺序随机打乱
- 直接进入答题状态，**没有确认页**

### 7.3 考试界面

**与章节练习的不同点：**

| 项目 | 章节练习 | 模拟考试 |
|:-----|:---------|:---------|
| 题号 | 显示全部题号 1~N | 只显示当前题号 X/30 |
| 题号栏 | 有颜色标记已答/正确/错误 | 无题号栏 |
| 确认答案 | 确认后立即显示解析 | **确认后不显示解析**，只记录答案 |
| 改变答案 | 确认后不可改 | **确认后可以改**（再点另一个选项自动覆盖） |
| 解析 | 答后立即显示 | **交卷后统一显示** |
| 进度条 | 有（已答/总题数） | 有（X/30） |

**考试界面布局：**

```
┌─────────────────────────────────┐
│  📝 模拟考试                      │
│  ⏱ 44:32（倒计时）  8/30         │
├─────────────────────────────────┤
│                                  │
│  题目内容文字...                  │
│                                  │
│  ○ A. 选项文字                   │
│  ○ B. 选项文字                   │
│  ○ C. 选项文字                   │
│  ○ D. 选项文字                   │
│                                  │
│  [ 确认答案 ]                    │
│                                  │
│  [ 上一题 ]        [ 下一题 ]    │
│                                  │
│  [            交卷              ] │
└─────────────────────────────────┘
```

**答题交互规则：**
1. 点击选项 → 选中态高亮
2. 点击 **确认答案** → 记录该题答案（不显示对错/解析）
3. **确认后可以改答案**：再点另一个选项 → 新选项高亮 → 再点确认 → 覆盖上一次记录
4. 已答的题，题号在进度显示 X/30 中计入

### 7.4 倒计时

- 显示 `MM:SS` 格式，每秒刷新
- **时间到 00:00** 时自动交卷
- **刷新页面**不会丢失考试状态（localStorage 存考试状态）
- **考试期间刷新页面** → 回到考试继续（倒计时继续走，因为存了开始时间）

### 7.5 交卷

**手动交卷：**
- 底部始终显示"交卷"按钮
- 点击后弹窗确认：**「确定交卷？剩余 X 题未答」**
- 确认后进入成绩报告

**自动交卷：**
- 倒计时归零时自动交卷
- 未答的题计为错误

### 7.6 成绩报告

```
┌─────────────────────────────────┐
│         考试结果                  │
│                                  │
│  得分：22 / 30                   │
│  正确率：73%                     │
│  用时：38 分钟                    │
│                                  │
│  ┌───────────────────────────┐  │
│  │  错题回顾                   │  │
│  │                            │  │
│  │  #1. 题目内容...           │  │
│  │  你的答案：B  正确：C      │  │
│  │  解析：...                 │  │
│  │  ─────────────────────     │  │
│  │  #2. 题目内容...           │  │
│  │  你的答案：A  正确：D      │  │
│  │  解析：...                 │  │
│  └───────────────────────────┘  │
│                                  │
│  [ 返回首页 ]                    │
└─────────────────────────────────┘
```

- 错题回顾：只显示答错的题，按题号排列
- 正确题不列出
- 点击"返回首页"回到首页

## 八、数据持久化

### localStorage 键值结构

```javascript
{
  // 各章节答题进度
  chapterProgress: {
    "第一章": {
      answered: { "1": true, "3": true },   // 题号 → 是否正确
      wrong: [3],                              // 答错的题号
      favorite: [1]                            // 收藏的题号
    },
    "第二章": { answered: {}, wrong: [], favorite: [] }
    // ... 以此类推
  },

  // 模拟考试状态（考试中时存在）
  examState: {
    questionIds: [5, 12, 3, ...],  // 30个题号（随机顺序）
    answers: { "5": 1, "12": 0 },  // 题号 → 所选选项下标
    startTime: 1718082000000,      // 开始时间戳（毫秒）
    completed: false               // 是否已完成（交卷后变为 true）
  },

  // 模拟考试历史记录
  examHistory: [
    { date: "2026-06-11", score: 22, total: 30, timeMinutes: 38 }
  ]
}
```

### 存储规则

- 每题回答确认后，立即写入 `chapterProgress`
- 模拟考试每确认一题，立即写入 `examState.answers`
- 模拟考试中途关闭页面，下次打开从 localStorage 恢复考试状态
- 交卷后 `examState.completed = true`，新增一条 `examHistory`
- 清除考试数据：再次点"模拟考试"时，如果已有 `examState` 且未完成，问用户是否继续上次考试

## 九、边界与异常处理

| 场景 | 处理方式 |
|:-----|:---------|
| 考试进行中再次点"模拟考试" | 弹窗提示：**「你有未完成的考试（剩余X题），是否继续？继续 / 重新考试 / 取消」** |
| localStorage 不可用 | 应用仍可运行，但答题记录和考试状态不保存 |
| 模拟考试倒计时归零 | 自动交卷，弹窗提示「时间到，自动交卷」 |
| 章节练习无题目（某章题库为空） | 选章后提示"该章节暂无题目" |
| 全部 30 题回答完 | 弹窗提示「所有题目已答完！」→ 确认后自动交卷 |
