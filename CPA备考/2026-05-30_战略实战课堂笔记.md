## 考点详解与案例对照

### 1️⃣ PEST分析（宏观环境分析）

| 维度 | 分析内容 | 对奥普新的影响 |
|------|---------|--------------|
| **P** 政治/法律 | 制造业合规要求越来越严 | 设备维护记录可能被检查 |
| **E** 经济 | 喇叭行业竞争激烈 | 停线=直接烧钱 |
| **S** 社会/文化 | 员工对公平性敏感 | 技术部不愿意"随叫随到" |
| **T** 技术 | 设备越来越智能 | 技术部知识壁垒在提高 |

> **核心结论：外部环境在推动"必须建立规范的服务流程"**

### 2️⃣ SWOT分析（内部+外部综合）

#### 以技术部为主体：

|  | **优势(S)** | **劣势(W)** |
|--|-----------|-----------|
| | ✅ 懂设备、懂技术 | ❌ 人手有限，项目压力大 |
| | ✅ 掌握数据和维护记录 | ❌ 跨部门话语权不足 |
| | ✅ 可以制定标准和流程 | ❌ 被随叫随到，无法做长期规划 |

|  | **机会(O)** | **威胁(T)** |
|--|-----------|-----------|
| | 🔥 产线数据→预防性维护 | ⚠️ 生产部不配合→流程形同虚设 |
| | 🔥 建流程→建立专业权威 | ⚠️ 内部有人不适应→抵触 |
| | 🔥 规范化→更多时间做创新 | ⚠️ 承诺的响应时间做不到→信誉受损 |

#### SWOT矩阵（四种策略）：

```
SO策略：技术部懂设备(优势)→趁机构建服务流程标准(机会) ✅
WO策略：人手少(劣势)→用流程减少被打断(机会) ✅
ST策略：有知识壁垒(优势)→防止被生产部牵着走(威胁) ✅
WT策略：话语权弱(劣势)→先管好自己，避免直接冲突(威胁) ✅
```

### 3️⃣ 五力模型（波特的行业竞争分析）

#### 标准五力：

```
潜在进入者
    ↓
供应商议价能力 → 现有企业间竞争 ← 买方议价能力
    ↑
替代产品
```

#### 公司内部版五力：

```
共同老板（相当于"买方/决策者"）
    ↓
生产部 ←→ 技术部（内部竞争资源）
    ↑
供应商（外部替代资源）
一线员工自己修（替代方案）
```

> **关键洞察：生产部议价能力强，是因为技术部没有正式的服务通道。流程建立→数据生成→权力结构变化。**

### 4️⃣ 战略的三个层次

| 层次 | 管什么 | 谁管 | 我们的案例 |
|:----:|-------|------|-----------|
| **公司层** | 公司往哪走 | CEO/董事会 | 奥普新深耕喇叭自动化（不归技术部管） |
| **业务层** | 怎么打赢对手 | 部门主管 | 技术部靠专业+速度建口碑 |
| **职能层** | 日常怎么做 | 主管/组长 | **产线服务流程（我们今天做的）** |

> **速记口诀：公司定方向、业务定打法、职能定干活**

### 5️⃣ 基本竞争战略（波特的三种通用战略）

```
三条路（只能选一条）：
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ 成本领先       │  │ 差异化       │  │ 集中战略      │
│ 靠便宜取胜     │  │ 靠特色取胜   │  │ 专注细分市场  │
└──────────────┘  └──────────────┘  └──────────────┘
```

> **技术部不能选成本领先（不能靠"便宜"竞争），选的是差异化：**
> - 专业能力 — 别人修不了的机器我们能修
> - 响应速度 — 流程明确，承诺可信
> - 数据能力 — 设备维护记录、故障分析
> - 流程规范 — 这不是每个技术部都有的

### 6️⃣ 价值链分析

#### 基本框架：

```
支持活动：
  企业基础设施 / 人力资源 / 技术开发 / 采购管理
基本活动：
  内部后勤 → 生产运营 → 外部后勤 → 市场营销 → 服务
```

> **技术部属于"支持活动"中的"技术开发"，但直接影响"基本活动"中的"生产运营"效率。**
> 产线停10分钟=基本活动停10分钟=直接损失产量。

### 7️⃣ 波士顿矩阵 (BCG Matrix)

```
            市场增长率
         高 ←──────→ 低
      ┌────────────┬────────────┐
   高 │ 明星业务    │ 金牛业务    │
      │ 自动化的产线 │ 传统喇叭    │
      │ (需持续投资) │ (稳定收钱)  │
      ├────────────┼────────────┤
   低 │ 问题业务    │ 瘦狗业务    │
      │ 新客户定制  │ 老旧手工线  │
      │ (培育/放弃) │ (淘汰/剥离) │
      └────────────┴────────────┘
```

> **冲突根源：技术部在"明星业务"(自动化)需要时间投入，生产部在"金牛业务"(传统产线)需要稳定运行。金牛业务的频繁打断影响了明星业务的发展。**
> 建立服务流程就是在"金牛"和"明星"之间划清界限。

### 8️⃣ 发展战略（公司层战略）

#### 三大类：

```
发展战略（扩张）
├── 一体化
│   ├── 横向（收购同行）
│   └── 纵向（收购上下游）
├── 密集型
│   ├── 市场渗透 / 市场开发 / 产品开发
└── 多元化
    ├── 相关多元化 / 不相关多元化
收缩战略（瘦身）
稳定战略（不动）
```

> **产线服务流程=运营层面的优化，支撑公司的效率提升方向，属于职能层战略。**

### 9️⃣ 战略落地的三层结构

```
┌──────────────────────────────────────┐
│  制度层（规则体系）                    │
│  故障分级P0-P3、意见分级A-C          │
├──────────────────────────────────────┤
│  执行层（流程机制）                    │
│  报修流程、值班制、升级链             │
├──────────────────────────────────────┤
│  支撑层（工具/数据/人员）              │
│  腾讯文档表格、排班表、备件流程       │
└──────────────────────────────────────┘
```

> **三层缺一不可：有制度没执行=废纸，有执行没支撑=人肉硬扛，有支撑没制度=乱搞。**

### 🔟 内部治理

#### 定义：公司内部各部门之间怎么协调、怎么制衡、怎么决策。

> **技术部产线意见提出流程 = 典型的内部治理机制**
>
> 解决了：
> - 谁可以提意见 → 所有技术部同事
> - 怎么提 → A/B/C三级
> - 提到哪 → 不同级别对口不同人
> - 怎么闭环 → 跟踪→验证→归档
>
> **核心原则："内部先过一道，再对外发声"**

## 学习方式总结

今天用的是 **"案例教学法"** ——用自己公司的真实问题去套CPA知识点。

优势：
- ✅ 比死记硬背强10倍
- ✅ 考试时遇到类似的案例题，脑子会自动调用"技术部和生产部"这个例子
- ✅ 学完了马上能用在工作里

后续计划：
- 每天工作的事猪猪侠加速处理 → 主人省出时间学CPA
- 每学一个考点，用公司实际案例串一遍
- 需要时拉迷糊老师/波比等灵魂团队分工协作
